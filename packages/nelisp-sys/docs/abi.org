#+TITLE: nelisp-sys ABI and platform model
#+OPTIONS: toc:2

* Overview

Package-local ABI contract (Doc 130 extraction criterion 8, derived from
Doc 132).  =nelisp-sys= has its own C-replacement ABI; it does NOT reuse or
blur the NeLisp Sexp runtime ABI.

* Supported targets

| Triple | Arch | OS | Object | C ABI |
|--------+------+----+--------+-------|
| =x86_64-unknown-linux-gnu= | x86_64 | linux | ELF64 | SysV AMD64 |
| =aarch64-unknown-linux-gnu= | aarch64 | linux | ELF64 | AAPCS64 |
| =x86_64-apple-darwin= | x86_64 | macos | Mach-O 64 | SysV AMD64 / Darwin |
| =aarch64-apple-darwin= | aarch64 | macos | Mach-O 64 | AAPCS64 / Darwin |
| =x86_64-pc-windows-msvc= | x86_64 | windows | PE/COFF | Microsoft x64 |

* Scalar layout

| Type | Size | Alignment |
|------+------+-----------|
| =i8/u8/bool= | 1 | 1 |
| =i16/u16= | 2 | 2 |
| =i32/u32= | 4 | 4 |
| =i64/u64= | 8 | 8 |
| =isize/usize/(ptr T)= | pointer width | pointer alignment |

* Struct layout (=:repr c=)

1. fields laid out in source order;
2. each field starts at the next offset satisfying its alignment;
3. struct alignment is the max field alignment;
4. final size rounded up to struct alignment.

=(sys:layout NAME)= returns =(:size N :align A :fields ((F :offset O :size S :align A)...))=.

* Slice ABI (=(slice T)= / =(slice-mut T)=)

A slice value is a *pointer to a two-word header* laid out as a =:repr c=
struct of =data= (a =(ptr T)=) followed by =len= (a =usize=):

| Field | Offset | Size |
|-------+--------+------|
| =data= (=(ptr T)=) | 0 | pointer width |
| =len= (=usize=) | pointer width | pointer width |

So =sizeof(slice T)= = 2 x pointer width and =alignof= = pointer alignment.
A function taking =(slice T)= receives, in one integer register, a pointer to
this header -- it is *not* split into two registers.  The equivalent C surface
on lp64 is:

#+begin_src c
struct nl_slice_T { T *data; size_t len; };
RET f(struct nl_slice_T *s, ...);
#+end_src

Access lowering (Stage 130.5):
- =(sys:slice-len S)= -> read =usize= at offset = pointer width;
- =(sys:slice-ref S I)= -> read =data= (offset 0), then read =sizeof(T)= bytes
  at =data + I*sizeof(T)=;
- =(sys:slice-set! S I V)= -> same address, write =sizeof(T)= bytes.

MVP gaps (clear errors or documented behaviour, never silent miscompiles):
- no runtime bounds check yet; the checked and =sys:slice-ref-raw= forms lower
  identically (static safety only);
- signed element reads sign-extend via the Phase 47 =ptr-read-sN= primitive
  (which desugars to an unsigned read + =shl=/=sar= pair); unsigned reads
  zero-extend;
- the slice operand must be a bare slice-typed variable -- a slice op through a
  non-var place is a clear =nelisp-sys-backend-error=.

* Calling convention (MVP)

=:abi c= maps to the target's C ABI.  MVP lowers integer/pointer args and
returns on linux/x86_64 first; other targets are object-inspection gates
until a C harness verifies them.

* Object metadata

Emitted ABI summary (sidecar or custom section):
