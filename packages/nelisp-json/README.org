#+TITLE: nelisp-json Package

* Summary

=nelisp-json= is the pilot extracted library package for the NeLisp
package split.  It provides the pure Lisp JSON parser and encoder used
by NeLisp components that need a =json.el=-compatible surface without
depending on host Emacs =json.el=.

* Layout

#+begin_src text
packages/nelisp-json/
├── README.org
├── src/
│   └── nelisp-json.el
└── test/
    └── nelisp-json-test.el
#+end_src

* Public API

- =nelisp-json-encode-string=
- =nelisp-json-encode=
- =nelisp-json-parse-string=
- =nelisp-json-serialize=
- =nelisp-json-read-from-string=
- =nelisp-json-pretty-print-string=

* Dependencies

- Runtime dependency: NeLisp core =nelisp= v1.0 load-path environment
- Elisp dependency declared by the library: =cl-lib=

* Extraction Notes

- The library source moved from =src/nelisp-json.el= to
  =packages/nelisp-json/src/nelisp-json.el= without code changes.
- The ERT suite moved from =test/nelisp-json-test.el= to
  =packages/nelisp-json/test/nelisp-json-test.el=.
- Existing callers continue using =(require 'nelisp-json)=; load-path
  wiring in the root build is responsible for resolving the package.

* Template Use

Future extracted libraries should copy this structure and document:

- what the package provides
- the exported feature name and public API

- any migration notes needed by maintainers
