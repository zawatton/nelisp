#+TITLE: nelisp-state Package

* Summary

=nelisp-state= provides a persistent key-value store on top of Emacs'
built-in SQLite support.  It is the extracted NeLisp library package
for cacheable state, namespaces, TTL expiry, and DB lifecycle helpers.

* Dependencies

- Runtime dependency: NeLisp core =nelisp= v1.0 load-path environment
- Elisp dependency declared by the library: =subr-x=
- Host capability dependency: built-in Emacs SQLite support
- Internal NeLisp package dependencies declared by the library: none

* Dependents

- Runtime dependents in =src/=: =nelisp-http=; downstream consumer = anvil.el's =anvil-state= via fboundp delegation (architecture α; Wave 1 deleted =nelisp-tools=)
- Test dependents in =test/=: =nelisp-state-test.el=
- Existing callers continue using =(require 'nelisp-state)=; the root
  build load-path resolves the extracted package location.
