#+TITLE: nelisp-http Package

* Summary

=nelisp-http= is the extracted NeLisp HTTP library package following
the Doc 45 package-split layout.  It provides the higher-level HTTP
fetch/cache surface layered on top of host Emacs URL primitives while
preserving the existing =(require 'nelisp-http)= feature name.

* Status

- Extraction status: package split applied on =2026-04-27=
- Package status: active optional library under =packages/=
- Source compatibility: existing callers still load via root build
  load-path wiring

* Layout

#+begin_src text
packages/nelisp-http/
├── README.org
├── src/
│   └── nelisp-http.el
└── test/
    └── nelisp-http-test.el
#+end_src

* Public API

- =nelisp-http-fetch=
- =nelisp-http-fetch-batch=
- =nelisp-http-fetch-post=
- =nelisp-http-fetch-head=
- =nelisp-http-robots-check=
- =nelisp-http-fetch-cache-clear=
- =nelisp-http-fetch-cache-get=
- =nelisp-http-fetch-cache-list=

* Dependencies

- Runtime dependency: NeLisp core =nelisp= v1.0 load-path environment
- Internal NeLisp dependency declared by the library: =nelisp-state=
- Host Elisp dependencies declared by the library: =cl-lib=, =url=,
  =url-http=, =url-parse=, =dom=

* Dependents

- Source dependent: =src/nelisp-tools.el=
- Test dependent: =packages/nelisp-http/test/nelisp-http-test.el=
- Existing callers continue using =(require 'nelisp-http)=; load-path
  wiring in the root build resolves the package path

* Extraction Notes

- The library source moved from =src/nelisp-http.el= to
  =packages/nelisp-http/src/nelisp-http.el= without code changes.
- The ERT suite moved from =test/nelisp-http-test.el= to
  =packages/nelisp-http/test/nelisp-http-test.el=.
- The root =Makefile= already auto-discovers =packages/*/src= and
  =packages/*/test= for compile and test targets, so no build rule
  change was required for this extraction.

* API Notes

- Primary surface: synchronous GET/HEAD/POST helpers with cache-aware
  fetch support
- Cache backend: =nelisp-state= namespace =\"http\"=
- Robots support: cached =robots.txt= evaluation via
  =nelisp-http-robots-check=
- Extraction helpers: HTML selector and dotted JSON-path extraction
