* What changed in stage-d-v3.0?

stage-d-v3.0 ships a zero-Rust standalone NeLisp reader/evaluator
tarball.  The executable is short and platform-native:

| Platform        | Command          |
|-----------------+------------------|
| Linux x86_64    | =bin/nelisp=     |
| macOS arm64     | =bin/nelisp=     |
| Windows x86_64  | =bin/nelisp.exe= |

The archive does not contain an Emacs binary, =anvil-runtime=,
=libnelisp_runtime=, cargo output, or a Rust runtime dependency.

* Install

For local iteration:

#+begin_src shell
make standalone-tarball PLATFORM=linux-x86_64
make standalone-tarball-verify PLATFORM=linux-x86_64
./release/stage-d-v3.0/install-v3.sh --from "$(pwd)/dist"
$HOME/.local/share/anvil-stage-d-v3.0/bin/nelisp eval "(+ 40 2)"
#+end_src

On Apple Silicon:

#+begin_src shell
make standalone-tarball PLATFORM=macos-aarch64
make standalone-tarball-verify PLATFORM=macos-aarch64
./release/stage-d-v3.0/install-v3.sh --from "$(pwd)/dist"
$HOME/.local/share/anvil-stage-d-v3.0/bin/nelisp eval "(+ 40 2)"
#+end_src

The macOS arm64 tarball is ad-hoc signed at build time with =codesign -s -=.
The verifier checks that signature before running =bin/nelisp=; it does not
repair or re-sign the extracted binary.  =install-v3.sh= also verifies the
installed =bin/nelisp= signature on Apple Silicon before reporting success.

On Windows x86_64 the tarball contains =bin/nelisp.exe=.

For local Windows PowerShell iteration:

#+begin_src powershell
.\tools\build-standalone-tarball.ps1 -Version stage-d-v3.0 -Target windows-x86_64
.\tools\verify-standalone-tarball.ps1 -Version stage-d-v3.0 -Target windows-x86_64
.\release\stage-d-v3.0\install-v3.ps1 -From .\dist -Prefix "$env:TEMP\nelisp-install-smoke"
& "$env:TEMP\nelisp-install-smoke\bin\nelisp.exe" eval "(+ 40 2)"
#+end_src

* Verify the CLI

#+begin_src shell
nelisp eval "(+ 40 2)"
printf '%s\n' "(+ 40 2)" "(vector 1 \"a\" nil t)" "(exit)" | nelisp repl --no-prompt
#+end_src

Expected output:

#+begin_src text
42
42
[1 "a" nil t]
#+end_src

* Full native verification

For Linux and Apple Silicon source checkouts:

#+begin_src shell
./scripts/verify-cross-platform.sh --parallel-jobs 2 --include-tarball
#+end_src

For Windows x86_64 PowerShell:

#+begin_src powershell
.\scripts\verify-cross-platform.ps1 -IncludeTarball
#+end_src

Expected output ends with =Cross-platform verify PASS=.
With =--include-tarball=, verification also installs from the freshly built
=dist= tarball and runs the installed =bin/nelisp= or =bin/nelisp.exe=.

* Build Artifacts

The standalone bundle contains:

- =bin/nelisp= or =bin/nelisp.exe=
- =src/nelisp*.el=
- =scripts/*.el=
- =lisp/*.el=
- =README.org=
- =VERSION=
- =PLATFORM=
- =MANIFEST.txt=

* Notes

- Tarball cap: < 15 MB compressed.
- The verifier checks SHA-256, layout, and native =eval= / REPL execution
  when the platform matches the host.
- macOS arm64 native verification also checks the existing code signature
  before executing =bin/nelisp=.
- Cross-built non-native tarballs are verified layout-only.
