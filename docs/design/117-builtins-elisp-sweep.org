* §0 TL;DR

eval/builtins.rs is the biggest single Rust file in the project
(excluding tests.rs).  It hosts 49 `bi_*` builtin handlers
dispatched by `eval::builtins::dispatch'.  Each handler is a small
function (~20-80 LOC) doing arg-validation + algorithm + result
wrap.  Doc 100 §100.C + Doc 101 §101.B + Doc 111 §111.B already
demonstrated that individual handlers can swap to AOT elisp
(`bi_truncate', `bi_length_cons_cc', `bi_recordp_cc' precedent).

Doc 117 sweeps the remaining ~46 handlers (= 49 minus 3 already
elisp-dispatching), targeting Tier A *first* (= pure-logic
handlers with no Env mutation, no syscall, no reader callout).
Tier B (syscalls) waits for Doc 122 (AOT syscall grammar) +
Doc 125 (libc binding).  Tier C (Env-touching) waits for Doc 119.

* §2 Implementation plan (Tier A pure subset, 4 handlers)

Each handler swap follows the established pattern (Doc 111 §111.B
recordp / §111.C aref-vector / §111.D cell-ops shipped examples):

1. Write `lisp/nelisp-cc-bi-<name>.el' with `defconst …--source'
   holding the AOT elisp.
2. Add `scripts/compile-elisp-objects.el' manifest entry +
   `build.rs' source list entry.
3. Add extern decl + safe wrapper in `build-tool/src/lib.rs'.
4. Replace the `bi_<name>' Rust body with a thin AOT dispatch
   (= same pattern as `bi_truncate' / `bi_length_cons_cc' on main).
5. Add probe test `build-tool/tests/elisp_cc_bi_<name>_probe.rs'
   with ≥ 3/3 cases.
6. Verify: lib tests still pass, probe ≥ 3/3, no regression.

### Per-handler

*** §117.A.1 — bi_make_vector

Algorithm: `(make-vector N INIT)` → fresh `Sexp::Vector` of N
copies of INIT.

AOT ops: `vector-make N slot' (§115.1) + N-element fill via
`vector-slot-set' (§111.E) in a recursive helper.

Estimate: ~20 LOC elisp.  Single commit.

*** §117.A.2 — bi_string_bytes

Algorithm: `(string-bytes STR)` → `Vector` of `Sexp::Int` (one per
byte).

AOT ops: `str-len' + `str-byte-at' + `vector-make' +
recursive copy.

Estimate: ~30 LOC elisp.  Single commit.

*** §117.A.3 — bi_signal

Algorithm: `(signal CONDITION DATA)` — build cons of CONDITION and
DATA, throw via `EvalError::Lisp(_)`.

The "throw" part needs control flow back to the dispatcher.  In
the Rust version this is a `Err(EvalError::Lisp(...))` return; in
elisp it'd need either:
- A throw / catch primitive (= already shipped? check AOT
  grammar).
- A sentinel Sexp tag the dispatcher recognises as "this is a
  signal".

For Doc 117.A scope, defer signal to Doc 117.D or after
condition-case primitives ship.  Skip from initial sweep.

*** §117.A.4 — bi_f64_trunc

Algorithm: f64 → i64 truncation.  Doc 110 §110.F covers most of
the f64 dispatch already.

AOT ops: f64 → int conversion grammar (Doc 110 §110.E.2.b if
present, else add).

Estimate: ~25 LOC elisp.  Single commit, requires Doc 110 §110.E
audit first.

* §4 Risks + open questions

** §4.1 Throw / signal primitive

`bi_signal' and friends need a way to non-locally return from
elisp.  Options:
- Add AOT `throw' grammar op that returns a sentinel tag the
  dispatcher recognises.
- Use the existing `condition-case' machinery in Rust (= keep
  signal Rust-side for now).

*Decision*: defer signal to Doc 117.D, ship Tier A.1+A.2 first to
establish the swap pattern.

** §4.2 Per-handler size

Tier A.1 and A.2 are tiny (~50 LOC each).  Bundling them in one
commit OR two commits is acceptable.  Recommend one commit per
handler for clean bisect history.

** §4.3 Existing test coverage

Each handler has ERT + Rust integration tests in `eval/tests.rs'.
The elisp swap must preserve all existing test outcomes.  Don't
add new behaviour; pure migration.

** §4.4 Tier B + C interactions

After Tier A ships, Tier B (syscalls) needs Doc 122 (AOT
syscall + atomic grammar) + Doc 125 (libc binding) before
proceeding.  Tier C waits for Doc 119 (env.rs elisp化).

Following the §117.B / §117.B+ quit-flag + write-stderr-line +
write-stdout-bytes + read-stdin-bytes sweeps (commits 5162c7b7,

** §117.D.gaps.1 — Bool-reader Rust-LOC math (= why thin readers are net-positive)

A typical bool-reader handler in =builtins.rs= is ~11 lines:

#+begin_src rust
fn bi_termios_saved_p(args: &[Sexp]) -> Result<Sexp, EvalError> {
    require_arity("_termios-saved-p", args, 0, Some(0))?;
    #[cfg(unix)]
    {
        return Ok(if tty_raw::termios_saved_p() { Sexp::T } else { Sexp::Nil });
    }
    #[cfg(not(unix))]
    {
        Ok(Sexp::Nil)
    }
}
#+end_src

To sweep to elisp dispatch (= the §117.B quit-flag pattern) one
needs *all* of:

- =nl_*_ptr() -> *const u8= extern in the owning module (~3-5 lines).
- =extern "C"= decl inside the =extern "C" {}= block in =lib.rs=
  (1 line).
- safe wrapper =pub unsafe fn= inside =elisp_cc_spike= (~5 lines
  with mandatory doc).
- new entry in =scripts/compile-elisp-objects.el= manifest (~3 lines).
- new entry in =build-tool/build.rs= rerun-if-changed list (~1
  line, counted on the elisp side but still review surface).
- the handler body itself shrinks to ~8 lines (still has
  =require_arity= + =#[cfg(unix)]= gate + dispatch + 0/non-zero
  → T/Nil map).

Net Rust LOC per swept bool-reader = ~+5 to +10.  For the entire
batch (= 5 readers) that's +25 to +50 Rust LOC for zero
algorithmic gain (= a single =load(SeqCst)= moves into elisp).

The §117.B quit-flag sweep paid this cost (+152 net per commit
=a849cec2= message) because:

  1. =QUIT_FLAG= read/write happens from elisp-side polling (= the
     quit-flag-pending-p path is now end-to-end elisp from the
     eval loop's perspective once Doc 119 lands), so the Rust
     plumbing dies when the Rust eval loop dies.
  2. The set / clear ops are *write* paths that need to live in
     elisp anyway for the AOT cutover.

Neither condition holds for the bool readers below — they are
consumed by either Rust-side test probes or by other Rust shims,
not by an elisp polling loop that's already migrating.

** §117.D.gaps.2 — Deferred handlers + their gap classification

*** Class 1: AtomicBool readers, plumbing > body savings

The following handlers each read a single =AtomicBool::load=
through a thin =*_p()= helper.  All ship-able with §122.E
=ptr-read-u8= (already shipped via =nelisp-cc-atomic-raw-mem.el=),
but the per-handler Rust-LOC math (§117.D.gaps.1) says +5..+10
Rust LOC per sweep.

| Handler                              | Storage                                  | Body LOC | Sweep verdict |
|--------------------------------------+------------------------------------------+----------+---------------|
| =bi_sigint_handler_installed_p=      | =quit::SIGINT_INSTALLED: AtomicBool=     | 3        | DEFER         |
| =bi_termios_saved_p=                 | =tty_raw::TERMIOS_SAVED: AtomicBool=     | 11       | DEFER         |

Re-evaluate when *both* sides of the call (= the eval-loop Rust
caller AND the body) have migrated to elisp, at which point the
ptr accessor + extern decl can also be deleted (= net-negative
Rust LOC).

*** Class 2: Once::is_completed readers, no atomic to expose

The following handlers read =std::sync::Once::is_completed()=.
The =Once= struct has no stable in-memory representation that
AOT elisp can poke via =ptr-read-u8= — it's an opaque
synchronization primitive.  Sweeping requires either:

  (a) Adding a parallel =AtomicBool COMPLETED= flag set after
      =call_once=, then dispatching the reader through it (cost:
      ~3 Rust LOC for the parallel atomic + the sweep plumbing
      cost above = +8..+13 per handler).
  (b) Replacing =Once::call_once= with a hand-rolled CAS-guarded
      init using =atomic-compare-exchange= grammar so the whole
      install path moves to elisp.  Blocked on Doc 122.I/J
      (CString) + Doc 122.K (sigaction-struct marshalling) for
      the install bodies themselves.

| Handler                              | Storage                                  | Body LOC | Sweep verdict |
|--------------------------------------+------------------------------------------+----------+---------------|
| =bi_raw_mode_hooks_installed_p=      | =tty_raw::HOOKS_INSTALLED: Once=         | 11       | DEFER         |
| =bi_winsize_handler_installed_p=     | =tty_winsize::HANDLER_INSTALLED: Once=   | 11       | DEFER         |
| =bi_jobctrl_handlers_installed_p=    | =tty_jobctrl::HANDLER_INSTALLED: Once=   | 11       | DEFER         |

Re-evaluate when path (b) becomes feasible (= post-Doc-122.I/J/K);
at that point the entire install path moves to elisp and the
Once + parallel-bool can be deleted together.

** §117.D.gaps.3 — Required Doc 122 grammar extensions

Summary of grammar ops missing on main =42cb1004= that gate the
remaining Tier B sweep:

- *Doc 122.I — CString grammar*: =(cstr-from-sexp STR-PTR)= →
  =*const c_char= NUL-terminated.  Owning =CString= management
  (alloc + drop) lives in the Rust shim.  Unblocks all path-
  taking syscall handlers.
- *Doc 122.J — Struct-by-value extern-call*: =(extern-call FN
  ARGS… STRUCT-ARG)= supporting =sigaction= / =termios= /
  =winsize= / =pollfd= / =stat= layouts.  Likely needs a
  =struct-make TAG SIZE ALIGN= constructor + =struct-field-set=
  / =struct-field-get= ops.  Unblocks the install / raw-mode /
  ioctl families.
- *Doc 122.K — libc constants*: signum (=SIGINT=, =SIGWINCH=,
  =SIGTSTP=, =SIGCONT=, =SIGTERM=, =SIGHUP=, =SIGQUIT=) +
  =SA_RESTART= + =TIOCGWINSZ= + =TCSANOW= + =VMIN= + =VTIME= +
  =POLLIN= + =POLLHUP=.  Could land as a generated
  =nelisp-cc-libc-constants.el= from a Rust build-script (=
  =bindgen= or a hand-rolled probe binary).  Unblocks any
  handler that names a libc constant.
- *Doc 122.X (proposed) — sexp-write-str-lossy*: =from_utf8_lossy=
  equivalent for reading a raw =*const u8= +len buffer into a
  =Sexp::Str= with U+FFFD replacement.  Unblocks the
  =bi_syscall_read_file= / =bi_syscall_readdir= alloc-then-wrap
  shape entirely.

Until at least 122.I + 122.J ship, the §117.B sweep is parked at
the four handlers in commits 5162c7b7 / a849cec2 / 0ce2446b
(quit-flag x3, write-stdout-bytes, write-stderr-line,
read-stdin-bytes).  The bool readers in classes 1, 2, 4 are
*technically* ship-able with today's grammar but are not worth
the Rust-LOC plumbing cost in isolation; they should be batched
with the corresponding =install_*= sweep so the ptr accessor +
extern + wrapper land together and amortize across multiple
swept handlers.
