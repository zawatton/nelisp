** 1.5 precedent reference (Emacs / SBCL / Clojure / Racket)

本 phase の直接 precedent は Emacs。ただし "editor-less Lisp runtime に
editor extension surface を載せる" という構図では Clojure / Racket 的な
分離も参考になる。

| 軸 | Emacs precedent | SBCL / Clojure / Racket precedent | NeLisp 方針 |
|----+-----------------+------------------------------------+------------|
| runtime と editor の一体性 | Emacs は一体型 | SBCL は editor 非内蔵、Clojure/Racket は editor binding 分離 | runtime と extension substrate を分離 |
| API source | =window.c= / =frame.c= / =keymap.c= 等 | editor API precedent は薄い | Emacs semantic を優先、実装層は分離 |
| package ecosystem 接続 | Emacs Lisp package 前提 | host runtime + tooling 分離 | 9a/9b/35+ に段階分割して導入 |

このため、本 doc は Emacs semantic precedent を採用しつつ、Doc 33 / Phase 7-11
の layered architecture に合わせて consumer/producer contract を明示する。
