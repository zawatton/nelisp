* §1 動機

** §1.1 Stage 7k = secure-hash collapse の前提崩壊

Doc 61 §6 / §4 が想定していた retire batch:

#+BEGIN_QUOTE
secure-hash dispatcher collapse — md5 / sha1 / sha224 / sha256 / sha384
/ sha512 が Rust 側で =bi_secure_hash(args, "ALGO")= 形の thin wrapper
になっているはず
#+END_QUOTE

実際は:

- 公開 primitive は =nl-secure-hash= の **1 entry のみ**
- 個別 algo (md5 / sha1 / ...) は =bi_nl_secure_hash= 関数内部の
  =match algo.as_str() { "sha1" => ..., "sha256" => ..., ... }= 分岐
- 外部に dispatcher arm として露出していないため、collapse する余地が無い

つまり Doc 61 §6 の前提は誤検知で、対象は元から無かった。grep + 関数本体
読みで確認 (=builtins.rs:3931-3983=)。

#+BEGIN_SRC emacs-lisp
;; Rust-min batch 7k (2026-05-07, Doc 65 closing batch).

* §5 残留 Rust primitive audit (= retire 井戸枯渇 確認)

builtins.rs の =install_builtins= に残っている primitives を分類:

** §5.1 Leaf primitives (= elisp 合成不可、Phase 5 lower 対象)

- *cons / aref* — =car / cdr / cons / setcar / setcdr / aref / aset /
  elt / vector / make-vector= : Sexp variant 直接操作
- *eq / equal* — =eq / equal / nelisp--ref-eq= : 同一性 / 構造再帰、
  primitive で書く以外不可能
- *predicates* — =type-of / functionp= : Sexp variant 判定 (= type-of
  自体が leaf、その上に consp / listp / symbolp 等が elisp 化済)
- *bitwise* — =nelisp--logior2 / -logand2 / -logxor2 / ash= : raw bit
  ops、ash は overflow clamp 必要
- *arithmetic* — =nelisp--add2 / -sub2 / -mul2 / / / -num-{lt,gt,le,ge,eq}2=
  : raw int / float ops
- *float / IEEE-754* — =float / exp / log / nelisp--f64-trunc / truncate /
  nelisp--format-float-body= : f64 op に直接降りる必要

** §5.2 Symbol / function table (= state 操作 primitives)

- =symbol-value / symbol-function / fboundp / boundp / defalias / fset /
  fmakunbound / makunbound / make-symbol / intern / symbol-name=
- 状態を持つテーブル (= obarray / Env 内) を直接弄るため Phase 7
  switch-over (= elisp eval が Env を直接 manage) で初めて elisp 化

** §5.3 Special forms / control flow

- =funcall / apply / eval / signal / require=
- =read / read-from-string= (Track O' Phase 2 で elisp 化予定 = Phase 7
  Stage 7.2)

** §5.4 OS surface (Doc 53〜60)

- =nelisp--syscall*= 系 26 entries (= Doc 53〜60 の Phase 4.0〜4.4)
- 全部 syscall / libc / sockaddr のため Rust から下げられない
- ただし =nelisp--syscall= 1 個だけが generic、他 25 entries は
  specialized = sockaddr serialization が elisp で書きづらいため Rust 化

** §5.5 Native time / hash / Unicode / file write

- =nl-current-unix-time / nl-secure-hash / nl-format-unix-time / nl-downcase /
  nl-upcase / nl-split-by-non-alnum / nl-write-file / nl-make-directory=
- 8 entries、全部 OS / native lib 直叩き、elisp 化不可

** §5.6 Terminal / signal infra (Doc 51 Track E〜Q)

- =terminal-raw-mode-{enter,leave} / read-stdin-byte-available /
  set-quit-flag / install-sigint-handler / install-winsize-handler /
  install-jobctrl-handlers= 等 ~14 entries
- termios / SIGINT / SIGWINCH / SIGTSTP の handler installation
- 全部 OS native、elisp 化不可

** §5.7 FFI / records / I/O sliver

- =nl-ffi-call / nl-ffi-malloc / nl-ffi-read-bytes / nl-ffi-free=
- =nelisp--make-record / -record-ref / -record-set / -record-length /
  -record-type / recordp=
- =nelisp--write-stdout-bytes / nelisp--write-stderr-line / read-stdin-bytes=
- 全部 leaf primitive、Phase 7 switch-over でも残る

** §5.8 Phase 7 で elisp 化される primitives (= 現在は Rust)

- =read / read-from-string= → Phase 7 Stage 7.2 (= elisp reader)
- =eval / funcall / apply= → Phase 7 Stage 7.4 (= elisp apply-env)
- =prin1-to-string= → Phase 7 Stage 7.1 (= elisp printer、Doc 64 設計済)
- これら 6 entries は Phase 7 で消える、本 Doc 時点では Rust に残留

** §5.9 結論 = thin wrapper retire の井戸は枯渇

§5.1〜§5.7 の category は全て /elisp 化が物理的に不可能 or 非効率/
で Rust に残らざるを得ない。§5.8 は Phase 7 で消えるが /thin wrapper
retire ではなく self-host で消す/ ため、本 Doc 50 ramp の延長線上には
無い。

つまり Doc 50 stage 7 = 7k で完結、次の retire は Phase 5 / Phase 7
着地後に再 audit。
