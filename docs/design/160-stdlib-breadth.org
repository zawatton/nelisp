* Goal

Close common-elisp library gaps on the bare standalone reader (=target/nelisp=)
so real packages drive further before hitting a =void-function=.  Driven by
discovery probes (≈55 + 70 + 50 common forms wrapped in =condition-case=) run
against the reader, then filling each gap as a guarded prelude definition
(=scripts/nelisp-stdlib-prelude.el=, =(unless (fboundp 'X) (defun/defmacro X ...))=).

All additions are pure-elisp library/compat definitions (no Rust, no dialect
primitive).  They live in the prelude, which is baked into the runtime image;
additions were kept low in string-literal density and verified after each batch
against the format/baked-string corruption symptom (=format "price=%d"= etc.).

* Notes / known limitations

- =%= (C-style integer remainder) and =/== (2-arg numeric not-equal) are now
  native dispatch arms (siblings of =mod= / =/= / ===), registered in the
  builtin name list (=reader-surface-audit= passes).  =%= is lowered to =x −
  y·trunc(x/y)= via the truncating dialect =/=, so its sign follows the
  dividend exactly like Emacs (=(% -7 2)= → =-1=); =/== negates =wf_num_eq= so
  it handles int/float/mixed.  The round-2 prelude predicates still use =mod= /
  =(not (= ...))= and are left unchanged.
- =expt= supports only integer exponents (no =pow=); float exponents error.
- =string-version-lessp= approximates with =string-lessp= (no numeric-run
  comparison).  =fround= uses round-half-up, not Emacs's round-half-even.
- =cl-letf= covers the three common place forms only (not full generalized-var
  =setf= places).
- =cl-letf= was intentionally the last gap closed; it restores correctly
  (=(cl-letf (((symbol-value 'zz) 9)) zz)= → 9, =zz= back to its prior value).
