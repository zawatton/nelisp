* §0 TL;DR

Phase 7 Phase 7 self-host (Doc 63) の最初の足場。Sexp の printer /
serializer を elisp で書き、Rust 側 builtins から =prin1-to-string= /
=princ= / =print= / =terpri= の dispatcher を撤去する。

| layer            | before                                       | after                                  |
|------------------+----------------------------------------------+----------------------------------------|
| Rust dispatch    | bi_prin1_to_string / bi_princ / bi_print / bi_terpri (4 arm) | (なし)                                 |
| Rust I/O         | =nelisp--write-stdout-bytes= / =number-to-string= / =nelisp--format-float-body= / =symbol-name= (= 残す) | 同左                                 |
| elisp 関数       | (なし)                                       | =prin1=/=prin1-to-string=/=princ=/=print=/=terpri= + helpers (7 本) |
| sexp.rs Display impl | error message format に内部使用 (= 残す)  | 同左                                   |

LOC 削減推定: Rust ~200、elisp +200。

Sexp printer は eval / reader 非依存の純粋関数 (Sexp → string)、
bootstrap dependency 最小なので Phase 7 の最初の足場として最適。

** §2.1 elisp 関数一覧 (= 5 user-visible + 3 helper = 8 本)

| =(prin1-to-string OBJECT)=            | =OBJECT= の readable な string 表現を返す              |

Internal helpers:

| function                              | 動作                                                  |
|---------------------------------------+--------------------------------------------------------|
| =(nelisp--prn-to-string OBJ ESCAPE)= | =OBJ= を string 表現にする中核 dispatcher (=ESCAPE= = readable / non-readable 切替) |
| =(nelisp--prn-string-escaped STR)=   | string literal の escape 形式 ("..." with \\ / \n / \t / \" 処理) |
| =(nelisp--prn-cons LIST ESCAPE VISITED)= | cons cell を再帰下降 (proper / dotted / circular の 3 種判定) |

** §2.2 Sexp variant 別 dispatch

#+BEGIN_SRC emacs-lisp
(defun nelisp--prn-to-string (obj escape)
  "Convert OBJ to its printed representation.
ESCAPE = t  → readable form (= prin1)
ESCAPE = nil → non-readable form (= princ)"
  (cond
   ((null obj) "nil")
   ((eq obj t) "t")
   ((integerp obj) (number-to-string obj))
   ((floatp obj)   (nelisp--format-float-body obj))
   ((symbolp obj)
    (if escape
        ;; readable: special chars (e.g. space) need backslash escape
        (nelisp--prn-symbol-escaped (symbol-name obj))
      (symbol-name obj)))
   ((stringp obj)
    (if escape
        (concat "\"" (nelisp--prn-string-escaped obj) "\"")
      obj))
   ((consp obj)
    (nelisp--prn-cons obj escape (make-hash-table :test 'eq)))
   ((vectorp obj)
    (concat "[" (mapconcat (lambda (e) (nelisp--prn-to-string e escape)) obj " ") "]"))
   ((recordp obj)
    (nelisp--prn-record obj escape))
   ;; ... CharTable / BoolVector ...
   (t (format "#<unprintable %S>" obj))))
#+END_SRC

各 variant 詳細:
- =Nil= → ="nil"=
- =T= → ="t"=
- =Int= → =(number-to-string N)=
- =Float= → 既存 =nelisp--format-float-body=
- =Symbol= → =(symbol-name SYM)=、ESCAPE=t なら special char に
  =\\= 付き
- =Str= → =\"...\"= (prin1) / そのまま (princ)、内部 =\\= =\"= =\n= =\t=
  などを escape
- =MutStr= → Str 同形 (= elisp 側からは透過)
- =Cons= → =(...)=、proper / dotted / circular を判別して再帰下降
- =Vector= → =[a b c]=
- =Record= → =#s(type-tag slot1 slot2 ...)=
- =CharTable= / =BoolVector= → 既存 Rust impl と同形
- =HashTable= は Doc 50 stage 4f-2 で Rust 撤去済 → record で表現

** §2.3 循環参照の扱い

Doc 50 stage 5b+5c で =equal= の cycle-safe 版を elisp 化済 = 同
pattern を流用:

#+BEGIN_SRC emacs-lisp
(defun nelisp--prn-cons (lst escape visited)
  "Print a cons cell.  VISITED = hash-table 'eq → label string."
  (when (gethash lst visited)
    ;; 既に visited = 循環参照
    (let ((label (gethash lst visited)))
      (if (eq label 'placeholder)
          (let ((new-label (format "#%d#" (hash-table-count visited))))
            (puthash lst new-label visited)
            new-label)
        label)))
  ;; 初回 visit = placeholder を立てる
  (puthash lst 'placeholder visited)
  (let ((parts (list "(" (nelisp--prn-to-string (car lst) escape visited)))
        (tail (cdr lst)))
    (while (consp tail)
      (push " " parts)
      (push (nelisp--prn-to-string (car tail) escape visited) parts)
      (setq tail (cdr tail)))
    (when tail
      (push " . " parts)
      (push (nelisp--prn-to-string tail escape visited) parts))
    (push ")" parts)
    (apply #'concat (nreverse parts))))
#+END_SRC

MVP では:
- 循環参照は =#1=...#1#= 表記で正しく出力
- 過深な構造は =max-lisp-eval-depth= で abort
- ERT で circular cons / circular vector を確認

** §2.4 String escape 詳細

#+BEGIN_SRC emacs-lisp
(defun nelisp--prn-string-escaped (s)
  "Return S with " / \\ / \n / \r / \t escaped per Emacs prin1 rules."
  (mapconcat (lambda (c)
               (cond
                ((eq c ?\")  "\\\"")
                ((eq c ?\\)  "\\\\")
                ((eq c ?\n)  "\\n")
                ((eq c ?\r)  "\\r")
                ((eq c ?\t)  "\\t")
                ((or (< c 32) (= c 127))
                 (format "\\%03o" c))   ; non-printable → octal
                (t (char-to-string c))))
             s ""))
#+END_SRC

** §2.5 Stream 書き出し

elisp eval の世界では =STREAM= は (1) buffer (2) function (3) marker
(4) t (= stdout) のどれか。NeLisp Stage 7.1 では実装簡素化のため:

- =STREAM= = nil → stdout (= =nelisp--write-stdout-bytes=)
- =STREAM= = function → =(funcall STREAM CHAR-OR-STRING)=
- それ以外 → =error= で abort

(buffer / marker は nemacs 側にしかないので NeLisp 単体では non-target)

** §2.6 Rust 側で残す primitive

- =nelisp--write-stdout-bytes BYTES= (既存) — 唯一の I/O 出口
- =number-to-string N= (既存) — Int → Str
- =nelisp--format-float-body F= (既存) — Float → Str
- =symbol-name SYM= (既存) — Symbol → Str

** §2.7 Rust 側削除する primitive

- =bi_prin1_to_string= 関数 + dispatch arm + install_builtins names
- =bi_princ= 同上
- =bi_print= 同上
- =bi_terpri= 同上

ただし sexp.rs::Display impl は内部使用 (= error message format) のた
め残す。ユーザー visible primitive のみ削除。

* §5 Bootstrap dependency

** §5.1 Pure function
- printer は eval / reader に非依存の pure function
- =cons= / =car= / =cdr= / =eq= / =hash-table-*= / =number-to-string= /
  =mapconcat= / =concat= など bundled elisp 既存の primitive のみで
  実装可能
- bootstrap 困難なし

** §5.2 internal Rust 使用箇所
- error message format で =prin1-to-string= を呼ぶ Rust 側コードがある:

#+BEGIN_SRC rust

  残す (= internal use)
- ユーザー visible =bi_prin1_to_string= 等は削除しても error message
  経路に影響なし
