*** §2.1.1 NlRc<T> 自前実装

現状: =Rc<RefCell<T>>= が =Sexp::Cons= 等の payload に使われる.  Rust 内部では std で動くが elisp / Cranelift IR から直触れない.

新設: =NlRc<T>= 自前 ref-counted ptr.  layout fixed:

#+begin_example
#[repr(C)]
struct NlRcInner<T> {
    refcount: AtomicUsize,  // offset 0
    value: T,                // offset 8
}

pub struct NlRc<T>(*mut NlRcInner<T>);
#+end_example

API parity with =Rc<T>=:
- =NlRc::new(value) -> NlRc<T>=
- =NlRc::clone(this) -> NlRc<T>= (= refcount += 1)
- =Drop= impl (= refcount -= 1, free if 0)
- =NlRc::deref(this) -> &T=

*** §2.1.2 NlConsBox layout pin

#+begin_example
#[repr(C)]
struct NlConsBox {
    car: Sexp,    // offset 0  (= 16 bytes: 1 byte tag @ 0, 8 byte payload @ 8)
    cdr: Sexp,    // offset 16
    refcount: AtomicUsize,  // offset 32
}
#+end_example

Sexp 自体も pin:

#+begin_example
#[repr(C, u8)]
pub enum Sexp {
    Nil,                       // tag = 0, payload = 0
    T,                         // tag = 1, payload = 0
    Int(i64),                  // tag = 2, payload = i64
    Cons(NonNull<NlConsBox>),  // tag = 7, payload = box ptr
    // ... 他 variant
}
#+end_example

Sexp Cons → NlConsBox の deref は単一 ptr load + offset access.  JIT IR で直 emit 可能 (= Doc 77 §2.3.3 emit pattern 通り).

*** §2.1.3 elisp primitive 追加

Layer 2 から NlConsBox を直接操作するための primitive:

| primitive | signature | semantics |
|-----------+-----------+-----------|
| =nl-cons-alloc CAR CDR= | =Sexp Sexp -> Sexp= | NlConsBox を heap に malloc、Sexp::Cons を返す |
| =nl-cons-car BOX= | =Sexp -> Sexp= | NlConsBox の car field を read |
| =nl-cons-cdr BOX= | =Sexp -> Sexp= | NlConsBox の cdr field を read |
| =nl-cons-set-car BOX VAL= | =Sexp Sexp -> Sexp= | car field を write |
| =nl-cons-set-cdr BOX VAL= | =Sexp Sexp -> Sexp= | cdr field を write |
| =nl-rc-inc BOX= | =Sexp -> ()= | refcount +1 |
| =nl-rc-dec BOX= | =Sexp -> ()= | refcount -1, 0 になったら drop |
| =nl-rc-count BOX= | =Sexp -> i64= | 現 refcount を read |

Rust 側 ~50 LOC の薄い shim で実装.  elisp から =nl-ffi-call= 経由で呼ばれる.

*** §2.1.4 JIT trampoline 改修

現在 =build-tool/src/jit/cons.rs::nl_jit_cons_car= は =&*arg= で =Sexp::Cons(rc)= を match、=rc.borrow().clone()= で deref.  Stage 4 後は:

#+begin_example
unsafe extern "C" fn nl_jit_cons_car(arg: *const Sexp, out: *mut Sexp) -> i64 {
    if (*arg).tag() == SEXP_TAG_CONS {
        let box_ptr = (*arg).cons_box_ptr();
        *out = (*box_ptr).car.clone();  // Sexp clone (= 16 byte memcpy + tag adjust)
        TRAMPOLINE_OK
    } else {
        TRAMPOLINE_ERR
    }
}
#+end_example

clone semantics: Sexp::Cons(box_ptr) の clone は box の refcount inc + ptr copy. 別 variant (Int / Float / Nil / T) は payload memcpy.

** §2.2 Phase B — Sexp 操作を elisp wrapper に migrate (= 2-4 週間)

Phase A 完了後、=cons= / =car= / =cdr= / =setcar= / =setcdr= の elisp wrapper を =nl-cons-*= primitive で書換:

#+begin_example
(defun cons (a b)

(defun car (cell)
  (cond ((eq cell nil) nil)

        (t (signal 'wrong-type-argument (list 'consp cell)))))
#+end_example

** §2.3 Phase C — GC を elisp に migrate (= 1-2 ヶ月)

=Drop= による auto-free は Rust 内部のみ.  elisp が manual ref counting を駆動:

#+begin_example
(defun cons (a b)
  (let ((box (nl-cons-alloc a b)))
    (nl-rc-inc a)        ; car ref +1
    (nl-rc-inc b)        ; cdr ref +1
    box))

(defun nl-cons-finalize (box)
  "cons の refcount = 0 時、car/cdr の refcount を dec して free."
  (nl-rc-dec (nl-cons-car box))
  (nl-rc-dec (nl-cons-cdr box))
  (nl-cons-free-box box))
#+end_example

Cycle detection (= cons 同士の循環参照) は elisp で Bacon-Rajan アルゴリズムを実装. ~300 LOC elisp.

Rust =Drop= impl は依然必要 (= elisp が dec 漏らした時の safety net) だが、運用上は elisp 駆動.

** §2.4 Phase D — Native compiler を elisp 化 (= 3-6 ヶ月)

Cranelift 依存を解消.  2 path:

*** §2.4.1 Path D-A: Cranelift C-style shim (= 1-2 週間、現実路線)

Cranelift Rust crate を保持しつつ薄い C API で expose:

#+begin_example
#[no_mangle]
pub extern "C" fn cl_make_module() -> *mut c_void { ... }
#[no_mangle]
pub extern "C" fn cl_emit_iadd(b: *mut c_void, a: u32, c: u32) -> u32 { ... }
// ... ~50 fn
#+end_example

elisp が =nl-ffi-call= で driving:

#+begin_example
(defun nl-jit-build-add2 (module)
  (let ((sig (cl-make-sig 2 1)))           ; 2 i64 in, 1 i64 out
    (cl-add-param sig :i64) (cl-add-param sig :i64)
    (cl-add-return sig :i64)
    (let ((func (cl-declare-function module "add2" :local sig)))
      (cl-emit-block func 'entry)
      (let ((a (cl-arg func 0)) (b (cl-arg func 1)))
        (cl-emit-iadd func a b)
        (cl-emit-return func a))
      func)))
#+end_example

Cranelift crate は依然 Rust だが「**Rust crate を呼ぶ shim**」のみが Rust core に残る.

*** §2.4.2 Path D-B: Cranelift 完全置換 (= 3-6 ヶ月、true self-host)

x86_64 / ARM64 の machine code byte sequence を elisp で直 emit、=mmap PROT_EXEC= した page に書込んで dispatch:

#+begin_example
(defun nl-emit-iadd-rax-rdx (page offset)
  ;; x86_64: 48 01 d0 = add rax, rdx
  (nl-ffi-write-bytes-at page offset (unibyte-string #x48 #x01 #xd0))
  (+ offset 3))
#+end_example

これが完成すれば nelisp は **Rust の Cranelift 依存ゼロ** で動く.  ただし register allocator / instruction selection / ELF linkage の elisp 実装が必要 (= ~5000 LOC elisp).

Phase D-A で start、Phase D-B は将来 Stage として後段化を推奨.
