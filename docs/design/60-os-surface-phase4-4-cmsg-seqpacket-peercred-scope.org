* §0 TL;DR

OS surface ライン Doc 53〜59 で deferred になっていた最後の 4 項目を 1
batch で消化する。これで NeLisp の Linux Path A surface は 「実用 modern
network stack」の覆いがほぼ完了。

| 系統          | item                                            | layer            |
|---------------+-------------------------------------------------+------------------|
| Phase 4.4.1   | =socketpair= (AF_UNIX 専用ペア生成)              | 専用 primitive   |
| Phase 4.4.2   | =sendmsg= + =recvmsg= (SCM_RIGHTS = fd passing) | 専用 primitive   |
| Phase 4.4.3   | =SOCK_SEQPACKET= (boundary-preserving UDS)       | 定数のみ         |
| Phase 4.4.4   | =SO_PEERCRED= (struct ucred = pid/uid/gid)       | 専用 primitive   |
| Phase 4.4.5   | IPv6 =scope_id= + =flowinfo= 完全 surface         | 専用 primitive   |

scope = 7 specialized primitive + 8 elisp wrapper + 約 6 定数 +
≥ 4 ERT。=socketpair=, =sendmsg=, =recvmsg= は struct/buffer/cmsg を
扱うため generic =nelisp--syscall= には乗らない。

* §1 動機

Doc 56 で AF_UNIX を入れ Doc 58 で abstract namespace を入れたが、UNIX
domain socket の核心機能である *fd passing* (SCM_RIGHTS) と *peer
authentication* (SO_PEERCRED) と *boundary-preserving message* (SOCK_SEQPACKET)
は未対応で、daemon 系のプロセス間通信 (e.g. systemd, dbus, lxc) を
書くには必須の surface が欠けていた。

加えて IPv6 で =scope_id= / =flowinfo= が常に 0 で固定されていたため、
link-local アドレス (=fe80::1%lo=) が扱えない状態だった。これも fix。

NeLisp の Lisp Machine vision では daemon / IPC 系は「OS interop fully
self-host」を目指しているので、この batch でその surface を埋める。

これらの追加で Linux Path A の OS surface ラインは「修飾的に欠けてい
たもの」がなくなり、io_uring / Path B (Darwin/Windows) / native cc
lowering の 3 つだけが Future work として残る状態になる。

*** §2.1.5/6/7 IPv6 scoped primitives

bind / connect / accept の 3 本それぞれ既存の inet6 primitive と同形だが、
=sockaddr_in6.sin6_flowinfo= と =sockaddr_in6.sin6_scope_id= を引数に
取って set / 戻す。helper =build_sockaddr_in6_with_scope= 1 本追加。

#+BEGIN_SRC rust
fn build_sockaddr_in6_with_scope(
    groups: &[u16; 8], port: u16,
    flowinfo: u32, scope_id: u32,
) -> libc::sockaddr_in6 { ... }
#+END_SRC

bind/connect は 5 引数化 (FD HOST6 PORT FLOWINFO SCOPE-ID)、accept は
1 引数のまま戻り値を 5-element list (FD HOST6 PORT FLOWINFO SCOPE-ID)
に拡張。
