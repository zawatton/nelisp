* §119.1 Premise

=build-tool/src/eval/env_helpers.rs= is 880 LOC.  Of that:

- ~430 LOC: =mirror_*= helpers (globals dictionary read/write).
  Already dispatch the hit-path to AOT (Doc 111 §111.E).  But
  the *miss-path* (= auto-vivify a new symbol-entry) is still Rust:
  =mirror_insert_new_entry= (~12 LOC) + =mirror_prepend_to_bucket=
  (~45 LOC) +  =Sexp::record / Sexp::cons= calls.

- ~250 LOC: =frame_*_rust_direct= helpers (lexical frame stack
  push/pop/bind/lookup).  Already dispatch most ops to AOT.
  Remaining Rust: =frame_stack_ensure_capacity= bootstrap path,
  =make_empty_frame_record=, =frame_bind_into= / =frame_lookup_in=
  inner-bucket walks (= called when AOT misses).

- ~200 LOC: snapshot/diff helpers (=mirror_snapshot_globals=,
  =mirror_diff_view=, =wrap_alist_cells=).  =mirror_snapshot_globals=
  is baker-only; rest are test/Closure construction paths.

* §119.2 Staged plan

** §119.A — auto-vivify path → elisp

The AOT helpers currently return `1` on hit and `0` on miss; the
caller falls back to Rust =mirror_prepend_to_bucket=.  Push the
prepend step into elisp so the helper handles both paths.

Surface change (per helper):
- =nelisp_mirror_set_value_or_insert= replaces =nelisp_mirror_set_value=
  (= absorbs the miss-path).
- Same for =_set_function=, =_install_entry=, =_set_constant=.

New elisp helpers needed:
- =nelisp_mirror_alloc_entry= (= replaces =mirror_insert_new_entry=)
- =nelisp_mirror_bucket_prepend= (= replaces =mirror_prepend_to_bucket=)

Both already have AOT grammar coverage (=record-make=,
=vector-make=, =cons-make=, =int-logand=, =mut-str-*= unused here,
=ptr-read/write-u64= for bucket-count slot).

Target: −80 LOC from the 4 wrappers + 2 helpers, −12 LOC from
=mirror_insert_new_entry= + −45 LOC from =mirror_prepend_to_bucket=
= −137 LOC.

** §119.B — frame_stack auto-vivify + bucket walks → elisp

Same pattern as §119.A, applied to the frame-stack helpers:

- =frame_stack_ensure_capacity= already moved to elisp in Doc 115
  §115.1.  Verify the wrapper is now 1-line.
- =make_empty_frame_record= (~17 LOC) → fold into elisp side of
  =frame_push_rust_direct=.
- =frame_bind_into= (~52 LOC) + =frame_lookup_in= (~33 LOC) — these
  are inner-bucket walks.  Move to elisp helpers
  =nelisp_frame_bind_into= / =nelisp_frame_lookup_in=.  The Rust
  callers already pass a =&Sexp= frame ptr that the helpers can
  consume.

Target: −110 LOC.

** §119.C — wrap_alist_cells → elisp

=Env::wrap_alist_cells= (~50 LOC) wraps a captured-env alist's
non-Cell values in fresh =NlCellRef= for closure-binding shape.
Called from =push_captured= only.

Two paths:
1. Move whole body to elisp =nelisp-env-wrap-alist-cells=.  Requires
   =cell-make= (already in AOT §122.E).
2. Keep Rust but inline (= no separate helper).  Lower priority since
   we'd save ~5 LOC of helper-call overhead.

Recommend Path 1.  Target: −50 LOC.

** §119.D — gate baker-only helpers

=mirror_snapshot_globals= + =mirror_diff_view= are =#[cfg(any(test,
feature = "image-baker"))]= already, but their bodies remain in the
production build's =env_helpers.rs= source pre-cfg.  Move to a new
=eval/env_baker.rs= file (or =cfg-gate the impl Env block).
Production cargo build (`make build`) carries 0 LOC of these.

Target: −150 LOC from the *production* compile unit; same Rust
source-LOC total (just relocated).

** §119.E — sweep + delete dead Rust helpers

After §A-D, audit env_helpers.rs for paths that no longer have callers.
Likely deletions:
- =mirror_insert_new_entry= (replaced in §119.A)
- =mirror_prepend_to_bucket= (replaced in §119.A)
- =make_empty_frame_record= (replaced in §119.B)
- =frame_bind_into= / =frame_lookup_in= (replaced in §119.B)
- =wrap_alist_cells= (replaced in §119.C)

Target: −137 + −85 + −50 = −272 LOC verified deletions.

* §119.3 Out of scope

- =impl Env= block existence.  Method dispatch must go through `&self`
  or `&mut self`, which requires a Rust impl block on a Rust struct.
  The end state is "1-line method body that delegates to AOT +
  pointer plumbing", which is the floor we can't pass.

* §119.5 Risk + mitigation

- §119.A SIGSEGV risk: Doc 124.F-blocker (NlBox Clone sweep) shows that
  AOT composed paths can crash.  Mitigate by shipping each helper
  swap individually + smoke-test before composing.

- =mirror_prepend_to_bucket= mutates =NlVector.value= via
  =with_value_mut= — this is the same pattern that crashed in §124.F
  when composed with another AOT call.  Pin investigation result
  before §A ships.

** §119.A SHIPPED log

Date: 2026-05-18.

LOC delta: =build-tool/src/eval/env_helpers.rs= 841 → 757 LOC
(= −84 LOC).  Target was −137; the gap is the per-call-site Rust
boilerplate that grew slightly when swapping each =mirror_set_*=
caller to the =_or_insert= variant (= extra =unbound_ptr= /
=Sexp::Symbol(name.into())= setup) — partially offset by the deletion
of =mirror_insert_new_entry= (12 LOC) + =mirror_prepend_to_bucket=
(45 LOC).  Net deletion in =env_helpers.rs= = −57 LOC dead code +
−27 LOC simplified call sites.

New elisp helpers shipped (= 6 new =.o= files):
- =nelisp-cc-mirror-alloc-entry.el= → =nelisp_mirror_alloc_entry=
  (= ~ =record-make= + 4 × =record-slot-set=).
- =nelisp-cc-mirror-bucket-prepend.el= → =nelisp_mirror_bucket_prepend=
  + =nelisp_mirror_prepend_install= sub-defun.  Composes
  =cons-make= + =cons-set-car= / =cons-set-cdr= (= refcount-safe
  Nil-seed pattern, mirrors =nelisp-cc-frame-bind.el=) +
  =vector-slot-set= + =sexp-write-str= (= bucket KEY is Sexp::Str
  per =mirror_iter_entries= tag-match contract) + =sexp-int-make=
  for count bump + =extern-call= to =nelisp_fnv1a=.
- =nelisp-cc-mirror-set-value-or-insert.el= →
  =nelisp_mirror_set_value_or_insert=.  Composes
  =nelisp_mirror_lookup_entry= + on-hit =record-slot-set 0= +
  on-miss =extern-call= to the two new building-block helpers.
- =nelisp-cc-mirror-set-function-or-insert.el= → slot-1 variant.
- =nelisp-cc-mirror-set-constant-or-insert.el= → slot-3 variant.
- =nelisp-cc-mirror-install-entry-or-insert.el= → 4-slot variant
  (= absorbs the legacy =mirror_install_entry= miss-path).

Surface change on the Rust side:
- =Env::mirror_set_value= / =_set_function= / =_set_constant= /
  =_install_entry= now dispatch to the =_or_insert= elisp helpers;
  no Rust fallback remains.  Each method body is a 5-10 LOC
  pre-condition + pointer setup + single =unsafe= extern call.
- Two private Rust helpers deleted outright:
  =mirror_insert_new_entry=, =mirror_prepend_to_bucket=.

Scratch vector layout: all four =_or_insert= helpers share an
11-slot =Sexp::Vector= built by the Rust safe wrapper.  Slots 0-4
hold =bucket_prepend= scratches (Nil source / inner-pair scratch /
outer-cell scratch / count-int scratch / KEY-Str scratch); slot 5
holds the =symbol-entry= tag symbol; slot 6 receives the freshly-
allocated entry record from =alloc_entry=; slots 7-10 hold the
4-tuple value-resolution from the safe wrapper (= value / function
/ plist / constant Sexp, with =unbound_marker= / =Sexp::Nil=
defaults per variant).

Smoke tests (= post-ship, =target/release/nelisp eval=):
- =(+ 1 2)= → =3=.
- =(progn (defun foo () 42) (foo))= → =42=  (= miss-path fires when
  =defun foo= installs a fresh symbol-entry via

- =(progn (setq x 100) x)= → =100=  (= =set_value_or_insert=
  miss-path for top-level =setq=).

- =(f 10)= for =(defun f (n) (if (= n 0) 1 (* n (f (- n 1)))))= →
  =3628800=  (= recursive auto-vivify under control flow).

Integration probes (= =cargo test --release --test
'elisp_cc_mirror_*'=): all 12 mirror probes green (= 4 +
=lookup_entry= / 3 + =lookup_function= / 4 + =lookup_value= / 4 +
=set_constant= / 4 + =set_function= / 5 + =set_value= / 4 +
