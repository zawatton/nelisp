* §0 TL;DR

** §0.1 What Doc 114 left unfinished

After Doc 114 Step 5 (main =2e32f16b=), the lexframe dispatch chain is:

#+begin_src
caller (eval::env::Env method)
   ↓
Phase 47-compiled .o (lisp/nelisp-cc-frame-*.el)
   ↓
nl_frame_* extern (env_lexframe_phase47_shims.rs)  ← still Rust
   ↓
NlRecord / NlVector primitive ops (Rust)
#+end_src

The shim layer (= 500 LOC) is "hybrid": elisp dispatches into the
shim, but the shim does the real work in Rust.  Per the repo's
existence purpose (= 純 elisp化, Rust 削減 唯一の評価軸), this is a
contradiction — the elisp `.o' files are thin trampolines back into
Rust.

Doc 115 closes the loop: the `nl_frame_*' implementations move into
elisp, the shim file is deleted, the dispatch chain becomes:

#+begin_src
caller (eval::env::Env method)
   ↓
Phase 47-compiled .o (lisp/nelisp-cc-frame-*.el)  ← does the actual work
   ↓
NlRecord / NlVector / NlCell / NlConsBox primitive ops (extern "C",
shared with all other Phase 47 callers)
#+end_src

** §0.2 Why this is feasible now

All required Phase 47 grammar ops + ABI helpers are already SHIPPED
on main =2e32f16b=:

| Op needed                          | Source     | Status   |
|------------------------------------+------------+----------|
| record-slot-ref / -ref-ptr / -set | Doc 111 §111.B | SHIPPED |
| record-make (via `nl_alloc_record') | Doc 111 §111.E #12 | SHIPPED |
| vector-len / -ref / -ref-ptr      | Doc 111 §111.C | SHIPPED |

No new ABI surface is required.  Doc 115 is *implementation only*.

* §1 Shim audit (= what gets rewritten)

** §1.1 The 7 shims

Source-of-truth: =build-tool/src/eval/env_lexframe_phase47_shims.rs=
@ main =2e32f16b= (501 LOC).

*** nl_frame_stack_ensure_capacity (= shim #20, ~50 LOC Rust)

*Behavior*: grow the backing vector inside `frames_record' to at
least `needed' slots.  Returns the resulting capacity, or 0 on
unbuilt mirror.

*Phase 47 ops needed*: =record-slot-ref-ptr=, =vector-len=,
=vector-slot-set= (= `nl_vector_set_slot'), =nl_alloc_vector=.

*Algorithm*:
1. Read `slots[0]' from frames_record (= backing NlVector) via
   record-slot-ref-ptr.
2. If `vector-len' ≥ needed → return current capacity.
3. Compute new capacity via capacity-doubling.
4. Allocate fresh vector via `nl_alloc_vector(new_cap)'.
5. Copy existing slots via repeated `vector-slot-set' + vector-ref.
6. Overwrite `slots[0]' via record-slot-set.
7. Return new capacity.

LOC estimate: ~60 elisp + 0 new Rust.

*** nl_frame_push (= shim #21, ~60 LOC Rust)

*Behavior*: push a fresh empty `nelisp-lexframe' record onto the
stack.  Returns 1 on push, 0 on unbuilt mirror.

*Phase 47 ops needed*: ensure_capacity (= #20, call sibling helper
via extern-call), =record-slot-ref=, =vector-slot-set=,
=record-slot-set= (for depth increment), =nl_alloc_record=,
=sexp-int-unwrap/make=.

*Algorithm*:
1. Read current `depth' (= slots[1]) and `backing' (= slots[0]).
2. Call `nelisp_frame_stack_ensure_capacity(depth + 1)' via
   extern-call.
3. Allocate fresh frame record via `nl_alloc_record(type_tag,
   slot_count)' with `nelisp-lexframe' type tag.
4. Initialize frame's slots (= 1 fast-hash-table sub-record + 16
   nil bucket vector).
5. `vector-slot-set(backing, depth, new_frame)'.
6. `record-slot-set(frames_record, 1, depth + 1)' to increment.
7. Return 1.

LOC estimate: ~80 elisp.

*** nl_frame_pop (= shim #22, ~30 LOC Rust)

*Behavior*: pop the innermost frame (= write Nil into slot
[depth-1], decrement depth).  Returns 1 on pop, 0 on empty stack.

*Phase 47 ops needed*: =record-slot-ref-ptr= for backing,
=sexp-int-unwrap/make= for depth, =vector-slot-set=,
=record-slot-set=, =sexp-write-nil=.

*Algorithm*:
1. Read current depth.  If 0 → return 0.
2. Read backing pointer.
3. =vector-slot-set(backing, depth-1, Sexp::Nil)= (= refcount drop).
4. =record-slot-set(frames_record, 1, depth - 1)=.
5. Return 1.

LOC estimate: ~30 elisp.

*** nl_frame_bind (= shim #23, ~30 LOC Rust + uses bind_into_frame)

*Behavior*: bind NAME → CELL in the innermost frame's hash table.
Returns 1 on bind, 0 on no-op.

*Phase 47 ops needed*: =record-slot-ref-ptr= (frame stack), =vector-ref-ptr=
(active frame), =record-slot-ref= (hash-table inner record), =vector-ref-ptr=
(bucket vector), FNV-1a hash via `nl_mirror_fnv1a_sexp' extern (shared
with env_mirror), =cons-make=, =cons-car/-cdr=, =vector-slot-set= (=
overwrite bucket head).

*Algorithm*:
1. Read innermost frame from `backing[depth-1]'.
2. Read `fast-hash-table' inner record (frame's slot 0).
3. Read `buckets' vector (= hash-table slot 1).
4. Compute FNV-1a hash of name → bucket index via `vector-len' mod.
5. Walk bucket cons-list, replace cell if name matches, else
   cons-prepend new `(name . cell)' pair to bucket head.
6. Return 1.

LOC estimate: ~60 elisp (longest shim).

*** nl_frame_stack_find (= shim #24, ~25 LOC + uses lookup_in_frame)

*Behavior*: walk the lexframe stack innermost-first, return pointer
to the matching pair's CDR (= cell slot), or 0 on miss.

*Phase 47 ops needed*: depth read, =vector-ref-ptr=, frame
internal walk (= same hash-table lookup as `nl_frame_bind' minus
the cons-prepend).

*Algorithm*:
1. Read depth.  For i in [depth-1, 0]:
   a. Read backing[i] = frame.
   b. Compute FNV-1a hash of name.
   c. Walk corresponding bucket cons-list checking str-eq.
   d. On hit, return pointer to pair's CDR via cons-cdr-raw (=
      `*const Sexp', as `i64' for the extern signature).
2. On miss, return 0.

LOC estimate: ~50 elisp.

*** nl_wrap_alist_cells (= shim #26, ~50 LOC Rust)

*Behavior*: walk a proper-list alist of `(symbol . value)' pairs,
construct a new alist of `(symbol . NlCell(value))', return Sexp
result via output slot.  Returns 1 on success, 0 on malformed input.

*Phase 47 ops needed*: =cons-walk= (= proper-list iteration),
=cons-car=, =cons-cdr=, =cell-make=, =cons-make=.

*Algorithm*: standard list-rebuild fold — for each pair (sym . val)
in input, emit (sym . cell-make(val)) in output.

LOC estimate: ~40 elisp.

*** nl_frame_stack_depth (= shim #19, ALREADY trivial)

Already a single =sexp-int-unwrap= of `slots[1]'.  No Rust shim
exists (= goes direct from `.o' to the Sexp byte at the
known offset).

*Status*: nothing to do; this entry counts as 0 LOC of Rust to
remove.

* §4 Risk + open questions

** §4.1 FNV-1a hash via extern-call

The `nl_mirror_fnv1a_sexp' extern stays in Rust.  It's a hot inner
loop (= reads `str.bytes' and produces a u32).  Pure-elisp
re-implementation would be slower; the existing Phase 47 grammar
doesn't have `byte-shift' / `byte-xor' ops to express FNV-1a
densely.

*Decision*: keep as Rust extern.  Doc 115 is "no Rust *logic in
shims*", not "zero extern" — primitive memory + numeric helpers
remain (per Doc 111 §111.E Group C / D classification).

If pure-elisp FNV-1a becomes desirable later, that's a separate
Phase 47 grammar extension (Doc 116+).

** §4.2 Probe coverage drift

When the per-shim probe was first written (Agent G's work), it
exercised the Rust shim.  After elisp化, the same probe exercises
the elisp path — but the probe's assertions may have implicitly
depended on Rust behaviour.

*Mitigation*: each per-shim commit must run the existing probe
unchanged (= same input, same expected output).  If the probe
fails after elisp化, that's a bug in the elisp port, not a probe
update.

** §4.3 Frame structure layout assumptions

The shims hardcode:
- `nelisp-lexframe' is a Record with `fast-hash-table' in slot 0.
- `fast-hash-table' is a Record with `bucket-count' (slot 0),
  `buckets' (slot 1, Vector), `size' (slot 2).
- `frames_record' is a Record with `backing' (slot 0, Vector),
  `depth' (slot 1, Int).

Pure-elisp versions must match exactly.  Source-of-truth is the
current shim file + `lisp/nelisp-lexframe.el'.

*Decision*: each elisp impl source includes a top-of-file comment
listing the layout assumptions (= maintenance footprint), and a
=defconst= for the slot indices.

** §4.4 Refcount discipline

`vector-slot-set' / `record-slot-set' / `cell-set-value' all
delegate to refcount-aware Rust externs (= `nl_vector_set_slot'
clones+drops, etc.).  Pure-elisp impls do not need to manage
refcounts explicitly — that's already encapsulated.
