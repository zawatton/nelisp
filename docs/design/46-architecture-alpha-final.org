** 5.3 launcher 経路

| client request | bin/anvil dispatch | spawn target | tool surface |
|----------------+---------------------+--------------+--------------|
| =mcp serve= (default) | Emacs path | =emacs --batch + anvil.el= | 41 tools (headless profile) |
| =mcp serve --strict-no-emacs= | Rust path | =anvil-runtime= binary | tools registered in Rust |
| =mcp serve --no-emacs= (bundled present) | Rust path | =$ANVIL_HOME/bin/anvil-runtime= | 同上 |

** 5.4 path dep / subprocess の握手

  - anvil-runtime crate → nelisp-runtime crate: =[dependencies] nelisp-runtime = { path = "../nelisp-runtime" }=
  - anvil.el (Emacs) → nelisp-XXX libraries: =(require 'nelisp-XXX nil 'noerror)= + fboundp guard
  - anvil.el (Emacs) → NeLisp interpreter: =anvil-eval--nelisp= が =nelisp eval EXPR= subprocess を spawn

NeLisp は Rust crate level でも Elisp library level でも *anvil の知識を持たない*。
