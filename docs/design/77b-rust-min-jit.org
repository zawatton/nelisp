* §0 TL;DR

Doc 77 §0 articulate した perf 課題 (= 1.4-1.6x のみ達成、target 5-20x 未達) は依然有効. 解 = "extern \"C\" boundary 撚廃 + IR 直 emit". ただし Doc 77 §3 の Stage 2-5 が *Rust core を +1000-1500 LOC 拡大してから -2000 LOC 削除* する path で、途中 Stage で大量の Rust が一時的に積み上がる.

Doc 77 (b) rewrite (= 本 Doc) の articulation: *strategy / wiring / IR 記述を elisp に移し、Rust は最小骨格 (Cranelift bridge + extern "C" trampolines + DSL interpreter) に留める*. Doc 76 (OS surface elisp 化) と同じ Rust-min 哲学を JIT に適用.

期待 net delta:
- 既存 jit/ Rust core 3163 LOC → ~1220 LOC (-1940 LOC, aggressive 推定)
- 移管先 elisp ~600-800 LOC 増 (= DSL rules + strategy wrapper)
- *Rust core net -1640〜-1800 LOC*, elisp net +600-800 LOC, *合計 -1000 LOC 程度*
- perf goal は変えず (= extern "C" 撚廃で 5-10x speedup) — strategy が elisp でも JIT 自体は依然 Cranelift native

** §1.3 elisp で再現可能な範囲 (= 本 Doc の核心)

*完全に Rust 必須 (= Cranelift API access、mmap、extern "C" boundary)*:
- =JITBuilder= / =JITModule= / =FunctionBuilder= の API 呼出
- =Box::leak= (executable page を process 寿命に固定)
- =unsafe extern "C" fn nl_jit_*= trampoline (= Sexp の =Rc<RefCell<...>>= 直アクセス)
- =std::mem::transmute= で fn pointer を typed extern fn に cast

*elisp で十分*:
- registry の populate (= "eq → JIT entry, car → JIT entry")
- inline rule の strategy 選択 (= "BoolVector aref を inline する/しない")
- IR 構造の記述 (= block / op / branch を data-driven で表現)
- =lowered_*= の arity check + JIT call + dispatch fallback wrap
- bench harness 全部
- env-toggleable filter

*data-driven (= elisp が描き、Rust が解釈実行)*:
- =declare_eq_inline= / =declare_length_with_inline_nil= / =declare_binop= 等の IR 構築 fn
  → elisp DSL でルール記述、Rust DSL interpreter が Cranelift IR call を emit

** §2.2 DSL syntax (= elisp s-expr で IR 記述)

s-expr 1 個 = 1 JIT 関数. 詳細は §A "DSL Reference" 参照.

#+begin_example
(jit-rule
 :name "nelisp_jit_eq_inline" :linkage local
 :params ((a ptr) (b ptr)) :returns i64
 :imports (("nl_jit_pred_eq" (ptr ptr) i64))
 :blocks
 ((block entry
    (= same (icmp eq a b))
    (brif same same-blk load-blk))
  (block same-blk
    (= one (iconst i64 1))
    (return one))
  (block load-blk
    (= at8 (load i8 a 0)) (= bt8 (load i8 b 0))
    (= at  (uextend i64 at8)) (= bt (uextend i64 bt8))
    (= teq (icmp eq at bt))
    (brif teq match-blk diff-blk))
  (block diff-blk (= z (iconst i64 0)) (return z))
  (block match-blk
    (= isi (icmp-imm eq at (tag SEXP_TAG_INT)))
    (brif isi int-blk slow-blk))
  (block int-blk
    (= ai (load i64 a 8)) (= bi (load i64 b 8))
    (= eq1 (icmp eq ai bi)) (= r (uextend i64 eq1))
    (return r))
  (block slow-blk
    (= r (call "nl_jit_pred_eq" a b)) (return r))))
#+end_example

5 jit-rule (= eq / length / car / cdr / aref) を elisp に書けば Doc 77 §3 Stage 2 の "call-site JIT entry" がカバーする領域がほぼ達成.

** §2.3 Rust interpreter (= dsl.rs ~330 LOC)

#+begin_example
struct Rule { name, linkage, params, returns, imports, blocks }
enum Op { Load, Store, Uext, Iconst, Icmp, IcmpImm, Bin, Unary, Call }
enum Term { Brif, Jump, Return }

pub(super) fn build_rule(module: &mut JITModule, rule: &Rule) -> FuncId;
fn parse_rule(sexp: &Sexp) -> Rule;
fn ty(t: Ty) -> types::Type;
fn cc(s: &Symbol) -> IntCC;

#+end_example

walks blocks twice:

** §2.4 elisp registry (= lisp/nelisp-jit-strategy.el)

#+begin_example
(defconst nelisp--jit-rules
  '((eq         . #s(jit-rule :name "nelisp_jit_eq_inline" ...))
    (car        . #s(jit-rule :name "nelisp_jit_car"       ...))
    (cdr        . #s(jit-rule :name "nelisp_jit_cdr"       ...))
    (length     . #s(jit-rule :name "nelisp_jit_length"    ...))
    (aref       . #s(jit-rule :name "nelisp_jit_aref"      ...))
    (aset       . #s(jit-rule :name "nelisp_jit_aset"      ...))
    (elt        . #s(jit-rule :name "nelisp_jit_elt"       ...))
    (cons       . #s(jit-rule :name "nelisp_jit_cons"      ...))
    (setcar     . #s(jit-rule :name "nelisp_jit_setcar"    ...))
    (setcdr     . #s(jit-rule :name "nelisp_jit_setcdr"    ...))
    (nelisp--add2 . #s(jit-rule :name "nelisp_jit_add2"   ...))
    ;; ... 12 arith ops
    (nelisp--syscall . #s(jit-rule :name "nelisp_jit_syscall" ...))
    (nelisp--syscall-supported-p . #s(jit-rule ...))))

(defconst nelisp--jit-env-toggleable
  '(nelisp--syscall nelisp--syscall-supported-p))

(defun nelisp--jit-bootstrap ()
  "Called by Rust startup before Box::leak.  Iterates over
   `nelisp--jit-rules' and calls into Rust:
     (nl-jit-build-rule rule-spec) -> registers JIT entry
   Returns alist of (NAME . FN-PTR) for try_lower."
  (mapcar (lambda (cell)
            (cons (car cell)
                  (nl-jit-build-rule (cdr cell))))
          nelisp--jit-rules))
#+end_example

Rust startup hook: =UnifiedJit::build= 内で elisp =nelisp--jit-bootstrap= を =Env= 経由で呼出 → 結果 alist を =lower_entries= に injest.

** §2.5 lowered_* の strategy も elisp 化

Pre-Doc-77b の lowered_eq:

#+begin_example
fn lowered_eq(args: &[Sexp], _env: &mut Env) -> Result<Sexp, EvalError> {
    crate::eval::builtins::require_arity("eq", args, 2, Some(2))?;
    let v = (jit().eq)(&args[0] as *const _, &args[1] as *const _);
    Ok(if v != 0 { Sexp::T } else { Sexp::Nil })
}
#+end_example

(defun eq (a b)

      nil t))
#+end_example

* §A DSL Reference

** Type vocabulary

=i8= / =i64= / =ptr= (= i64 が ABI 上同一).

** Op grammar

| form | semantics |
|------+-----------|
| =(= V (load TY PTR OFFSET))= | TY 値を [PTR + OFFSET] から load して V に bind |
| =(store TY VAL PTR OFFSET)=  | VAL (TY 型) を [PTR + OFFSET] に store |
| =(= V (uextend TY INNER))= | INNER (smaller TY) を TY に zero-extend |
| =(= V (iconst TY LIT))= | TY 型の immediate. LIT = 整数 or =(tag SYMBOL)= |
| =(= V (icmp CC A B))= | i64 比較 → i8 (0/1).  CC = eq/ne/slt/sle/ult/ule/sgt/sge/ugt/uge |
| =(= V (icmp-imm CC A LIT))= | A vs immediate LIT |
| =(= V (iadd A B))= 等 | iadd / isub / imul / ishl / sshr / bor / band / bxor |
| =(= V (call HELPER A B ...))= | imported helper を call、戻り値 i64 を V に bind |

** Terminator

| form | semantics |
|------+-----------|
| =(brif COND THEN ELSE)= | COND が i8 / i64 / bool で truthy なら THEN block へ jump |
| =(jump BLK)= | unconditional jump |
| =(return V)= | V を function return |

** Tag references

=(tag SYMBOL)= は Rust 側 const table (=TagTable=) を引く.  対応:

- =SEXP_TAG_NIL= → 0
- =SEXP_TAG_T= → 1
- =SEXP_TAG_INT= → 2
- =SEXP_TAG_FLOAT= → 3
- =SEXP_TAG_SYMBOL= → 4
- =SEXP_TAG_STR= → 5
- =SEXP_TAG_MUT_STR= → 6
- =SEXP_TAG_CONS= → 7
- =SEXP_TAG_VECTOR= → 8
- =SEXP_TAG_CHAR_TABLE= → 9
- =SEXP_TAG_BOOL_VECTOR= → 10
- =SEXP_TAG_CELL= → 11
- =SEXP_TAG_RECORD= → 12

** Block scope rule

各 block 内の =(= V ...)= で bind した V は same block + dominator chain で参照可.  cross-block phi 必要時は jump arg として渡す: =(jump BLK V1 V2)= → =(block BLK (V1-PARAM ...) ...)= の予定 (= 初期版 b.1 では未対応、必要になった Stage で追加).
