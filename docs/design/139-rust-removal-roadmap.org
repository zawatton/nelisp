* TL;DR

The repo no longer needs Rust *to build* — `make standalone-reader` =
`emacs --batch -l nelisp-standalone-build`, zero cargo.  The ~18.7K Rust LOC
that remains is *not a build dependency*; it survives as four things:

1. *Test oracle* — the Rust reader/evaluator is the cross-impl reference the
   81 probe tests + 13 ERT files compare elisp output against.  (Most of
   build-tool's LOC is this test harness, NOT a production interpreter.)
2. *Release artifact* — current tarballs bundle the Rust `nelisp` binary +
   `libnelisp_runtime.a`/cdylib.
3. *ABI drift gate* — `make sexp-abi-check` diffs elisp layout constants
   against Rust `const` values.
4. *Substrate kernel* — `nelisp-runtime` (~1K LOC syscall stub + seed
   loader) + optional syscall/sqlite extension crates.

Removing Rust = *decoupling these four consumers* (mostly S/M effort) +
*finishing the in-flight nelisp-sys substrate cutover* (Docs 124/125/126/134/
135, the real remaining elisp work).  The eval-core Rust is already ~75%
demolished; this roadmap finishes the job and tears down the crates.

*Key reframing*: build-tool's bulk LOC is the *cross-impl test harness*
(81 `tests/*.rs` probe files = 14.2K lines), not a production interpreter.
The actual Rust *src* (`src/eval/mod.rs` evaluator/reader/builtins) is
heavily minified (~1.5K equiv) and is itself a *dispatch shell that delegates
to the compiled-elisp `.o` objects* via `elisp_cc_spike` FFI.  The Rust
reader/eval/special-forms/builtins/nlboxes subsystems are *SUPERSEDED* — they
exist only as the Rust side of the test oracle.

** (4) ABI GATE — effort L (hard blocker)
- `make sexp-abi-check` → `cargo build ... --bin sexp-abi-emit` →
  `./target/release/sexp-abi-emit` prints Rust `ABI_EXPORT` layout constants,
  diffed against `lisp/nelisp-sexp-layout.el` `nelisp-sexp--abi-export`.
- Decouple by *promoting the elisp constants to canonical*: make the ERT
  suite `nelisp-sys-sexp-abi-test.el` (already runs under Emacs alone, no
  Rust) the *sole* gate, OR emit the same `NAME=VALUE` table from a tiny C
  `offsetof`/`sizeof` probe.  Requires a deliberate "elisp is ground truth"
  decision.

* Strategic fork (decide in Stage 0)

Two ways to reach zero-Rust; both end identically, differ in path:

- *Path A — finish nelisp-sys cutover (Docs 133/134/135)*: keep a green
  `cargo build` with *shrinking* Rust at every step until the crate is empty,
  then delete.  Preserves the cross-impl test oracle longest.  Slower.
- *Path B — aggressive decouple-then-delete*: stop maintaining the Rust
  build; decouple verify/release/CI/ABI onto the standalone+ERT path
  (Stages 1–3), Phase-47-replace only the ~10 load-bearing symbols + the
  substrate gaps actually used (Stage 4), then delete all crates (Stage 6).
  Faster, aligned with "Emacs builds it"; cost = lose the Rust oracle, so
  ERT/probe coverage must be re-homed onto the elisp impl first.

Recommendation: *Path B for tooling (Stages 1–3 now, cheap, high-leverage),
Path A's mechanism (nelisp-sys) for the load-bearing substrate (Stage 4),
since that elisp work is already in flight.*

* Staged teardown plan

- *Stage 0 — Baseline & decision* (S):
  - Confirm current `cargo build --bin nelisp` state at HEAD (green/red).
  - Verify which load-bearing symbols the `standalone-eval` (full evaluator)
    link actually needs vs already-elisp-stubbed.
  - Pick Path A vs B per area; record in this doc.
  - *Gate*: decision recorded → proceed.

- *Stage 1 — Decouple VERIFY/CI* (S):
  - verify-*.sh: `target/release/nelisp` → `target/nelisp`;
    `cargo test` → ERT.  Delete/lint-ify the cargo CI job.
  - *Gate*: green CI with no cargo invocation in verify/smoke.

- *Stage 2 — Decouple RELEASE/ARTIFACTS* (M):
  - Tarball builders ship the standalone binary; drop Rust runtime
    cdylib/staticlib/`nelisp` binary.  Redirect release workflow.
  - *Gate*: a release tarball built + soak-passed with zero Rust artifacts.

- *Stage 3 — Decouple ABI GATE* (L):
  - Make `nelisp-sexp-layout.el` canonical; `nelisp-sys-sexp-abi-test.el`
    (ERT) becomes the sole drift gate; retire `sexp-abi-emit.rs`.
  - *Gate*: `make sexp-abi-check` (or its ERT successor) green without cargo.

- *Stage 4 — Replace load-bearing substrate in elisp* (L, = Docs 117/123/124/
  125/126/134/135):
  - Port the ~10 `#[no_mangle]` helpers (tty statics, counters, f64/MutStr)
    to Phase-47 `.o`.
  - Port mmap_jit/clear_icache/jit_write_protect + seed loader (Doc 126).
  - *Gate*: standalone-eval links with no Rust-provided symbols.

- *Stage 5 — Optional extension crates* (M):
  - `syscall-process`: already orphaned by the JIT `syscall-direct` path →
    delete.  `fileio`/`filenotify`: finish dir-enum/inotify elisp or drop.
  - `sqlite-rs`: decide keep-as-Optional vs port FFI vs drop
    `packages/nelisp-sqlite/`.
  - *Gate*: each remaining crate has a green elisp path or a documented drop.

- *Stage 6 — Physical crate teardown* (M):
  - Delete `build-tool/` (incl. `build.rs`, `tests/*.rs` once re-homed to
    ERT), `nelisp-runtime/`, `nelisp-runtime-cli/`, all `nelisp-syscall-*`,
    `nelisp-sqlite-rs/`.  Remove the workspace `Cargo.toml`/`Cargo.lock`.
  - *Gate*: `rg --type rust` returns nothing; full test/verify/release green
    on the elisp path; README + CLAUDE.md updated.

* Risks / watch-items

- *Losing the cross-impl oracle*: the 81 probe tests are the main correctness
  net.  Before deleting build-tool's tests, the equivalent assertions must be
  re-homed as ERT against the elisp impl, else regressions go silent.
- *Hidden load-bearing symbols*: Stage 0 must enumerate the *full* evaluator
  link's externals, not just the reader's.
- *Bootstrap is still Emacs*: zero-Rust ≠ zero-Emacs.  Emacs-less is Doc 137's
  separate track (self-host interpreter).  This roadmap reaches "zero Rust,
  Emacs-hosted build"; Emacs-independence is downstream.
- *Optional features default-ON*: process/fileio/filenotify/sqlite crates are
  feature-gated but default ON; flipping defaults off is a cheap intermediate
  step to surface real consumers before deletion.
- *Rust LOC hard rule*: per user policy, no stage may *add* Rust; every step
  is delete-Rust / add-elisp (nelisp-sys vehicle guarantees this).
