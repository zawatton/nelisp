** §1.3 "完全削除" が aspirational である理由

*** §1.3.1 bootstrap chicken-and-egg

=Env::new_global= は =STDLIB_SOURCES= 内 elisp ファイルを順次
=reader::read_all= で parse.  elisp reader (=nelisp-stdlib-reader.el=)
自身も STDLIB_SOURCES に含まれるため、その install *前* の parse
は Rust reader でしか出来ない.

→ =reader/lexer.rs= / =reader/parser.rs= / =reader/mod.rs= は bootstrap
path として削除不可.

*** §1.3.2 =Sexp= value type の共有

=reader/sexp.rs= (= 671 LOC) は =Sexp= enum を定義する.  Rust 全体で
**runtime value 型** として使われており、reader 専用ではない:

#+begin_example
crate::reader::sexp::Sexp が以下から import される:
  - eval/* (= 全 file)
  - jit/* (= access / arith / cons / predicate / syscall)
  - bridge/*
  - image.rs
#+end_example

→ =reader/sexp.rs= は reader cleanup の scope 外、Rust 値表現として
継続使用.  本 Doc 71 では削除候補に含めない.

*** §1.3.3 build-tool driver の Rust reader 直接利用

| caller | 用途 | 必須度 |
|--------+------+--------|
| =eval_str= / =eval_str_all= | テスト + REPL 入力 | Rust API として必須 |
| =Env::new_global= | bootstrap | 必須 (= chicken-and-egg) |
| =image::compile_elisp_to_image= | AOT bake (image.rs) | 必須 |
| =bridge::bootstrap_self_host= | Phase 8 self-host bridge | Phase 8 別 layer |
| =bi_read= | =(read STRING)= builtin | post-bootstrap 移行可 |
| =bi_read_all_from_string= | =(nelisp--read-all-from-string STRING)= | post-bootstrap 移行可 |
| cargo unit tests | fixture 構築 | テスト便利、移行 cost 高 |

→ post-bootstrap path から Rust reader 直接利用を消せるのは
=bi_read= と =bi_read_all_from_string= の 2 つ.  bootstrap / image /
bridge / tests は据え置き.

** §1.4 結論: Stage 7.6 の現実的 scope

* §5 Risks

** §5.1 elisp reader の未対応 syntax

=nelisp--read-from-string-impl= が Rust reader と完全 parity でない
syntax がある可能性.  Stage 7.2.c (= round-trip property test) で
検証されているはずだが、edge case が残るかも.

mitigation: cargo serial 全件 PASS で empirical 検証.  fail なら
elisp impl の bug を特定 → fix.

** §5.2 perf regression

=bi_read= は =eval_arg_list= 後の dispatch + =apply_function=
overhead が乗る.  小さな string を頻繁に read する場合に subtle.

mitigation: bench で計測、3x 超なら Phase 5 lower entry 追加検討.

** §5.3 bridge / image との二重管理

=bridge/= と =image.rs= は依然 Rust reader 直接利用.  本 Stage 7.6
の scope 外.  将来 AOT bake (= Doc 63 §3.3) で =image.rs= が elisp
reader を使う形に移行する可能性あり、その時 Stage 7.7+ として再検討.
