#+TITLE: Doc 137 — AOT-interpreter self-hosting (AOT path, zero Rust)
#+AUTHOR: zawatton + Claude
#+DATE: 2026-05-31

* Goal
Complete *self-hosting*: a standalone =nelisp-cc= binary (no Emacs, no Rust)
that rebuilds any =.el= — including the compiler/linker themselves and
=nelisp-cc= itself — to native ELF, and does so using NeLisp-native
multi-threaded parallelism (Stage 4).

After this, Emacs is needed *exactly once* (Stage 2, the one-time bootstrap
of =nelisp-cc=); every build thereafter is Emacs-free.

* Strategy decision: interpreter-in-native (Strategy A)
Two ways to reach an Emacs-free build:

| | A: interpreter-in-native | B: compile compiler.el itself |
|--+--------------------------+-------------------------------|
| native binary embeds | runtime + *interpreter loop* | the whole AOT-compiled toolchain |
| compiler.el kept as | *source* (read + eval'd) | must become AOT-dialect |
| blocker | grow interpreter to full Elisp | rewrite 14k lines into the 98-op subset |

*Chosen: A.* The reader already exists (=nelisp-cc-reader-lexer/parser.el=,
~95% complete), the eval core exists (=nelisp-cc-eval-inner.el= + the
=nelisp-cc-evalport-*= cluster), growth is incremental + testable, and we
reuse the existing assets instead of making 14k lines AOT-compilable.
Both strategies need the same runtime growth, so Stage 1 work is identical
either way — no risk in starting on A.

* Gap ledger (2026-05-31 recon)
** Already present, AOT-dialect, compilable today
- Reader: lexer + parser (strings, chars =?x=, vectors =[]=, quote/backquote/
  unquote, =#'=, radix ints, dotted pairs; only =#[bytecode]= missing).
- Eval core: 11 special forms (=if let let* setq while lambda function quote
  progn condition-case unwind-protect=) + apply dispatch + ~61 builtins.
- Arena allocator, cons/vector/symbol/record alloc, =syscall-direct=,
  =atomic-fetch-add= / =atomic-compare-exchange= (→ the multi-thread path).
- Dormant macros in =nelisp-stdlib-eval-special.el=: =cond when unless and or
  dolist dotimes push pop prog1 cl-defun= (defined, not yet activated).

** What the 14k-line compiler.el actually demands (the bar)
catch/throw x303 · plist-get x437 · get x764 · nth x577 · mapcar x85 ·
hash-tables(equal) x32 · format x42 · pcase x31 · cl-loop x18 · cl-labels x15.
Crucially *string-match only x1* and *cl-defstruct = zero* (plist-in-vector
structs) — the dreaded regexp engine is a near non-issue; no defstructs to model.

* Progress (2026-06-01) — Wave 1/2 + the THREE confirmed self-host blockers
After M1-M8 + the compiler fix (committed 1849377b), two parallel waves landed
(committed e8aa1d3d, 19bc2b6e):
- *Wave 1*: macro-expansion CACHING (in-place AST rewrite → re-expansion cost 0,
  ~45-50% per-call cut, fib-via-cond ceiling 21→25) + breadth foundation (fix
  =(car nil)= segfault + tag-aware =eq=; primitives; the 7 prims that let the
  repo =nelisp-stdlib-*.el= + =nelisp-cl-macros.el= load AS-IS as elisp preludes).
- *Wave 2*: =pcase= + =cl-loop=(list clauses) + =cl-labels= + =condition-case=
  TRAPPING all work (the real =nelisp-cl-macros.el= / =nelisp-pcase.el= load as
  preludes). The interpreter is now a near-complete Elisp.
*Self-host (Stage 3/4) is blocked on THREE orthogonal architectural items, each
real eval-core / reader / compiler work — NOT build patches (confirmed by 5
independent analyses):*
1. *Escape-copying GC.* The bump arena never frees; per-call leak (~17-50 KB)
   is unbounded. Form-boundary reclamation PROTOTYPED → flat ~12.4 MB for a
   stream of homogeneous pure forms (886,000× per-form cut), gates green — but
   UNSAFE for general code: when a form installs a definition/closure that
   escapes into the persistent mirror, resetting the region dangles it. A real
   fix needs deep-copy-on-escape in =nelisp_cons_construct= / clone helpers /
   the mirror-install path (lisp/ source).
2. *Reader form-count ceiling* (~8,200 top-level forms / the 256-slot parse
   pool) — the GOLDEN binary SIGSEGVs at parse time too; reader/parser work.
3. *Recursion depth* — rec_max 2000 / native C stack; no TCO. Deep single
   computations need TCO or an explicit eval stack.
Known latent interpreter bugs surfaced (for the eval-core owner): the =apply=
op returns 0; macro closures calling an external helper get corrupted when
defined mid-prelude (cl-loop needs a final redef workaround); cl-loop numeric+
accumulator clause unsupported; no error-subtype hierarchy.

** Stage 3 — self-host  (PATH PROVEN, breadth remaining)
PROVEN: a stdlib macro (=cond=) defined in elisp via the =defmacro= prelude and
loaded from a file works → the whole stdlib can be loaded as elisp preludes.
REMAINING (the mountain): add the builtins compiler.el/linker use (nth/assq/memq/
mapcar/plist-get/get/vector ops/logand/ash/…), load the macro prelude, and the
HARD blockers =pcase=/=cl-loop=/=cl-labels=. Then =nelisp-cc= loads compiler.el +
static-linker.el + asm + elf-write and rebuilds any =.el= → native, zero Emacs.
Acceptance: rebuilds =target/nelisp-standalone-eval= cmp-identical to the Emacs build.

*** Stage 3 gap — MEASURED (2026-05-31)
*BLOCKER #0 (the real barrier): no memory reclamation.* The arena is a bump
allocator that never frees (=nl_dealloc_bytes= is a no-op) and each eval call
allocates ~hundreds of KB (env/mirror cloning). =fib(9)= exhausted the old 16 MiB
arena; raising it to 1 GiB reaches =fib(15)= but =fib(20)= still dies. compiler.el
= millions of eval calls → would exhaust any fixed arena. *Self-host needs a GC or
a region/mark-release allocator* (with deep-copy of escaping results) — a major
phase, not just "add primitives". (Two lesser ceilings, NOT self-host blockers:
=rec_max= guard, set to 2000; native C stack ~2500 levels. The compiler recurses
by AST-nesting depth = tens, far under both.)
*Breadth gap (after BLOCKER #0):* ordered plan —
1. ~10 trivial tag-check/list glue builtins (consp/stringp/symbolp/integerp/
   vectorp/listp/atom/zerop/aref/aset/make-vector).
2. Load =lisp/nelisp-stdlib-list.el= + =-plist-str.el= as elisp preludes — unlocks
   nth/nthcdr/append/assq/memq/reverse/nreverse/mapcar/plist-get/-put in one shot
   (covers nth×591 + plist-get×251 + … of compiler.el). Needs only the 7
   primitives consp/aref/signal/fboundp/stringp/vectorp/copy-sequence first.
3. =append= → fixes the prelude macros prog1/prog2/pop/dolist (their expanders call append).
4. =signal= wired into the catch/throw stash (trappable errors; ×345 in compiler.el).
5. =backquote= reader support (=`= / =,= / =,@=; ×306 in compiler.el + asm) — a
   reader-parser extension; hard, and a hard prerequisite to even READ compiler.el.
6. no-op stub macros: defconst/defvar/define-error/require/provide/declare-function.
7. =pcase= / =cl-loop= / =cl-labels= (HARD) — load the real =lisp/nelisp-cl-macros.el=
   / =nelisp-pcase.el= as preludes once the primitive+backquote floor exists.
Realistic first toolchain target = static-linker.el (steps 1-4,6; one pcase site
desugarable to cond, modest backquote). compiler.el additionally needs backquote
+ the cl/pcase prelude. ALL gated behind BLOCKER #0.

** Stage 4 — NeLisp-native multi-threaded build
=clone(2)= + futex via =syscall-direct=, work-stealing over the unit list with
=atomic-fetch-add=; per-unit compile dominates → real speedup (the Emacs
multi-process path was startup-bound, ~no speedup — see worklog 2026-05-30).

* Acceptance criteria
- Stage 1: each Mn has a byte-level / exit-code test (see =make standalone-eval-test=).
- Stage 3: =nelisp-cc build= produces a binary =cmp -l=-identical to the Emacs build.
- Stage 4: wall-clock build time scales with thread count on a many-unit set.

* Invariants (do not regress)
- Zero =.rs= / cargo / rustc. Every layer is pure elisp compiled by AOT.
- Rust LOC must never increase (it is being driven to zero).
- The existing 11-special-form eval core + reader stay byte-stable; growth is additive.
