* 0. Purpose

The flat-arena cold-loader (scripts/nelisp-flat-arena-probe.el, boot-wiring 5–14,
merged 439ac529) proves the dump→file→load→relocate→root-recovery MECHANISM, but
only as a READ-ONLY VERIFY (loads into a scratch =alloc-bytes= buffer; "live
runtime untouched").  The remaining work to make it usable (fast cold-load of a
loaded heap, ~9.2x vs .repl replay) is the production INSTALL: a fresh process
that loads an image into its REAL arena, relocates against the real base, installs
the EvalCtx roots, and skips the normal bootstrap.  This doc fully scopes that.

Motivation: the full nemacs bundle (=./bin/nemacs --driver=nelisp=, .repl replay)
cold-loads in 30–66 min and is OOM-prone (~11–15 GB RSS); package-breadth driving
needs the bundle (cl-lib/seq/pcase/macroexp/...).  A flat-arena image of the
loaded heap would load fast and unblock breadth — IF a one-time dump can be
created (the dump still needs ONE slow load; vendor-core subset is lighter).

* 2. Image format (=bf_arena_dump_image_to_file=, header 64 B)

| off | field | source |
|-----+-------+--------|
| 0  | magic 1179407692 ("FLAT") | literal |
| 8  | slen (chunk-0 region bytes) | cursor − 1024 |
| 16 | isz (interned region bytes, 8-aligned) | abase+832 .. abase+840 |
| 24 | tlen (relocation-table entries) | nl_fa_roots + bf_arena_mr_chunks |
| 32 | gbox image-offset (globals root) | nl_img_off of env+8 |
| 40 | fbox image-offset (frames root)  | nl_img_off of env+40 |
| 48 | ubox image-offset (unbound root) | nl_img_off of env+72 |
| 56 | intern-region base (split-loader name-ptr fix) | abase-relative |

File body = [header][reloc table: tlen × u64][chunk-0 region: slen][interned
region: isz].  Reloc entry F: =image[F] += newbase= (chunk-0-relative pointers).
The interned region's open-addressing name-buffer ptrs (slot+8) live INSIDE the
interned region, so =nl_fa_field= (chunk-0-only) does NOT table them; they retain
absolute OLD pointers and must be fixed by base-delta on load (see the dump
comment + =bf_arena_load_split_verify=).

EvalCtx (env) layout (gdb-verified, from the dump + line ~2682):
- env reached via published safepoint ctx: =env = ptr-read-u64 (ctx+64)= when
  =ptr-read-u64 ctx 0 != 0=.
- globals box @ env+8, frames box @ env+40, unbound box @ env+72.

* 3. The remaining work: production INSTALL

** 3.1 New builtin =bf_arena_boot_install (args out)= (adapt =bf_arena_boot_load_verify=)

=bf_arena_boot_load_verify= already: opens file, reads header, reads table+image
into a SCRATCH =alloc-bytes img=, applies the reloc table (=image[f] += img=),
reconstructs a rootslot (tag 12, ptr = img+goff), verifies it is a valid Record.

INSTALL差分:
1. Load the chunk-0 region into the REAL arena chunk-0 data area (=sstart=, i.e.
   =arena_base + 0x400=) instead of a scratch buffer; load the interned region
   into the real interned region (=ib=).
2. Relocate against the REAL base delta (=newbase = sstart_new − slen_origin_base=),
   not the scratch base.  Apply the table AND the interned-region base-delta fix
   (mirror =bf_arena_load_split_verify=).
3. Seed arena chunk metadata (cursor/used/chunk-count at offsets 0x2c0.., see the
   =nelisp-standalone--arena-chunk-*-offset= consts) so the arena is well-formed
   for subsequent allocation (the image cursor = slen+1024).
4. INSTALL the EvalCtx roots: set env+8 = sstart+goff (globals Record),
   env+40 = sstart+foff, env+72 = sstart+uoff.  (Where -1 = immediate, rebuild
   inline per =bf_arena_image_root_verify=.)

** 3.2 New driver mode =exec-arena-image IMAGE FORM...=

In =nelisp-standalone--reader-driver-source= (line ~9708):
- Add matcher =nl_cstr_eq_exec_arena_image= for ="exec-arena-image"= (alongside
  the existing =nl_cstr_eq_exec_runtime_image= etc., ~lines 155–162 / 9866–9868).
- Add it to the mode-dispatch chains (3 sites: ~310–316, ~10021–10055,
  ~10689–10693).
- Handler: =nl_arena_init= (reserve real arena) → =bf_arena_boot_install(IMAGE)=
  (loads heap + installs env roots) → SKIP the normal bootstrap/prelude load (the
  image already contains the prelude + everything) → eval FORM through the normal
  eval path with the installed env → print.

Key invariant: the baked prelude must NOT be re-installed on top of the image
(the image already has it).  The handler replaces the boot, it does not augment.

* 4. Concept-proof plan (incremental, branch spike/flat-arena-boot-install)

1. *Scratch value-survival* — *DONE 2026-06-23*.  New builtin
   =nelisp--arena-value-survival PATH SYM= (=bf_arena_value_survival=): loads the
   image into scratch (c0 + ir), applies the split relocation (per
   =bf_arena_load_split_verify=), rebuilds the globals Record root (tag 12,
   box = c0+goff), and looks up SYM via =nelisp_mirror_lookup_entry= +
   =record-slot-ref= slot 0 (= the value cell; only mirror-lookup.o/entry is
   linked, so composed here, not =nelisp_mirror_lookup_value= which is not
   linked).  VERIFIED on target/nelisp: =(set 'ct 42)= → dump → survival → 42;
   strings ("hello"), conses ((1 . 2)), symbol lists ((a b c)) all survive; an
   unbound symbol misses → nil.  *Relocation correctness for arbitrary reachable
   state is now proven*; only the real-arena INSTALL (§3) remains.
2. *Real-arena install* — *DONE 2026-06-23, via the boot-wiring MARKER path (not the
   planned =exec-arena-image= driver mode)*.  The spike owner's boot-wiring already
   ships the real-arena install as a BOOT step gated by marker-file existence:
   - =nl_cold_load_arena= (build script ~10461): if =(nl_cold_marker_cpath)=
     (=/tmp/nelisp-cold-image.bin=) opens, loads the chunk-0 region into the REAL
     arena data area (=base+1024=) + interned region (=base+832=), applies
     =nl_cold_reloc= (table) + =nl_cold_reloc_intern= (interned base-delta), zeroes
     tail, clears marks, seeds the cursor (=base+0 = 1024+slen=, =base+840 = ib+isz=),
     pushes the next GC far, returns 1.  Runs BEFORE the boot alloc.
   - =nl_cold_overwrite_globals= (~10502): installs the globals Record into the
     EvalCtx (env+8 = tag-12 box at =ds+goff=).
   - Driver boot (~10456): =(_cl (nl_cold_load_arena))= then
     =(if (< _cl 0) <normal-boot> (nl_cold_overwrite_globals globals))=; the cold
     path SKIPS the baked prelude (the image already carries user state).
   *VERIFIED end-to-end* on =target/nelisp= (2026-06-23): process-1
   =(defun cf (x) (* x x)) (set 'k 5) (set 'lst '(10 20 30))
   (nelisp--arena-dump-image-to-file MARKER)= (image 47.9 MB) → process-2 boots
   with the marker present and evaluates =(list (fboundp 'cf) (cf 9) k lst)= →
   =(t 81 5 (10 20 30))=.  *A user-defined defun survives the cold-load AND is
   callable in the LIVE runtime; value + cons-list state survives.*  Removing the
   marker restores normal boot (=(+ 40 2)= → 42).
   - ✅ *Caveat FIXED in increment 4a (commit f781ce31)*: originally =(floor 7 2)= → 7
     on cold-load (native) vs 3 on normal boot (A1 prelude override); =funcall= also
     returned 7, and =(symbol-function 'floor)= had reverted to the native subr.  Root
     cause was NOT native dispatch but the boot order: the driver ran
     =nelisp-standalone--reader-install-builtins-forms= UNCONDITIONALLY at ~line 10505,
     AFTER =nl_cold_overwrite_globals= (10502) had installed the cold image's mirror.
     install-builtins re-registers every native builtin into the mirror, overwriting
     the image's entries — including the prelude overrides of those ~60 builtins — back
     to the bare native subr.  User symbols (=cf=) have no builtin entry, so they were
     never touched (which is why defuns survived but floor reverted).  *Fix*: gate the
     install-builtins splice on =(< _cl 0)= (normal boot only), mirroring the prelude
     gates.  The cold image already carries all native builtins + their overrides; the
     builtins' native code pointers are stable across processes (same static binary,
     same load base) and the override closures are relocated inside the image, so all
     remain callable.  *Verified*: cold =(floor 7 2)=3=, =(funcall #'floor 7 2)=3=,
     =(cf 9)=81=, =(+ 40 2)=42=, =(* 6 7)=42=, mapcar/eq/mod/truncate identical to
     normal boot.  No full-prelude re-run needed — that path is unsafe (free/reuse of
     image blocks → SIGSEGV, per the spike owner's prelude-skip comments).
3. *vendor-core subset*: dump cl-lib+cl-generic+pcase+macroexp+subr-x+seq (lighter
   than the full bundle, avoids OOM), boot with the marker + drive seq-filter etc.
   Fold in the native-override re-apply fix above so the subset is faithful.
4. *Full bundle*: one-time slow dump (still 30–66 min, OOM risk — run off-peak /
   high-RAM), then fast marker cold-load for breadth.

* 5. Risks / caveats

- The install writes the REAL arena before the eval loop runs — a relocation or
  metadata-seed error → SIGSEGV on the first variable lookup.  Debug with the
  scratch value-survival (step 1) FIRST to isolate relocation correctness from
  install correctness.
- Interned-region name-ptr base-delta (slot+8) is the most error-prone part
  (=bf_arena_load_split_verify= is the reference).
- This is the spike owner's active area (boot-wiring 5–14); coordinate to avoid
  duplicating the install they may already be drafting.
- The one-time dump still needs ONE full load; the flat-arena win is on RELOAD,
  not the first dump-creation.  The host OOM on the first load is orthogonal and
  unsolved (vendor-core subset mitigates by being lighter).
