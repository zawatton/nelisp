* §0 TL;DR

** §0.1 静的リンカ (static linker) とは何か

static linker = 1 個以上の **compiled unit** (= Doc 92 macro assembler
の output、各 unit は section 群 + symbol table + relocation entry を
持つ memory representation) を入力に取り、

1. 各 unit 内の =EXTERN= 参照 (= 未解決 symbol) を、同 unit または他
   unit 内の =DEFINED= symbol に解決
2. 全 unit の同種 section を結合 (= =.text= 同士、=.rodata= 同士、…)
3. 各 relocation entry が指す byte 位置を、解決済 symbol address で
   patch
4. 結合後の section 群を Doc 91 ELF writer に渡し、最終 ELF binary を
   出力

= 「複数 =.o= を 1 個の実行可能 ELF にまとめる」古典的 link 工程の
elisp 版。

責務の core:
1. *symbol table merge* = 各 unit の symbol を 1 つの global table に
   統合、重複 (= 同名 GLOBAL DEFINED) を error / 同名 WEAK は preference
   rule で resolve
2. *section combiner* = =.text= / =.rodata= / =.data= / =.bss= を unit
   順に concat、各 unit 内の符号には base offset を加算
3. *relocation pass* = =R_X86_64_PC32= / =R_X86_64_64= / =R_X86_64_PLT32=
   の 3 type を最低限 support、各 entry の =r_offset= 位置の bytes を
   target address (= symbol value + addend) で patch
4. *output emit* = Doc 91 =nelisp-elf-write-binary= に section bytes +
   resolved symbol table + 残 relocation (= dynamic 用、本 doc は空) を
   渡す

** §0.2 elisp で書ける根拠

linker は **pure-functional symbol arithmetic** の問題:

- 入力 = unit list (= elisp data structure of section + symbol +
  reloc plist)
- 出力 = 結合後 section bytes (= byte vector) + resolved symbol table
  (= alist) + entry symbol address
- 副作用 = なし (= file I/O は Doc 91 writer 側のみ)
- 算術 = 32-bit / 64-bit signed arithmetic、bit-mask、PC-relative
  delta = elisp 整数で素直に表現可能 (Emacs 27+ で bignum 安全)

elisp で =logand= / =logxor= / =ash= / =mod= / =unibyte-string= /
=substring= / =aref= / =aset= が動けば実装可能。性能は §6.7 risk として
articulate するが correctness 上の障害なし。

** §0.3 なぜ必要か (= dlsym bridge との対比)

現状: nelisp-cc が emit する trampoline call site (= =CALL nl_jit_cons_make=)
は **runtime に dlsym で解決** されている。この path:

#+begin_src text
   compile time:                       runtime:
   [nelisp-cc emit `CALL ?']    -->    [nelisp-cc-runtime resolve-symbol]
   (= relocation 未適用)               (= dlsym(RTLD_DEFAULT, "nl_jit_..."))
                                       --> CALL rel32 patch
                                       (= elisp 側で page bytes を mutate)
#+end_src

これは **JIT** path = page を生成して即実行する pattern。
=cargo build= が出力する =./nelisp= binary 自体の「Rust extern fn の
解決」は **system linker (rustc → ld → -rdynamic flag)** が行う。

Phase 47 = 0 Rust runtime 完遂時:

- =./nelisp= は **elisp toolchain によって生成** される
- system linker (= rustc + ld) は invoke できない (= Cargo workspace
  自体を消す前提)
- → elisp が **AOT で symbol を resolve、relocation を patch、最終 byte
  stream を出力** する役目を担う = 本 doc の static linker

代替なし: elisp が複数 unit を結合して 1 binary にする手段は (a) ld、
(b) lld、(c) gold、(d) 自前 emit の 4 択。(a)-(c) は Phase 47 の "0
Rust / toolchain 不要" goal を否定。よって (d) elisp 自前 emit が唯一
の path。

** §1.4 「writer 不在」「linker 不在」の現状 articulate

Doc 91 §1.4 で articulated 通り、Rust 側に独立した ELF writer は存在せず
system linker (rustc / ld) に完全 delegate。Linker 側も同様に **Rust 側
に独立した static linker は存在しない**:

- =cargo build= → =rustc= → =ld= chain が relocation を解決
- =dlsym_bridge.rs= は **runtime symbol resolution の 1 path (= JIT
  page の CALL fixup)** であって AOT linker ではない
- nelisp-cc-runtime の Stage 81.4 path も runtime fixup table の consumer
  = AOT linker ではない

よって本 doc が articulate する static linker は「**Rust source からの
port ではなく完全新規実装**」。参考 = GNU binutils =ld= の =ldmain.c= +
=bfd/elf64-x86-64.c= の =elf_x86_64_relocate_section=、LLVM lld の
=ELF/Driver.cpp=、Plan 9 mk =8l= の linker stage、tcc の =tccelf.c=。

;; unit = output of Doc 92 assembler for one source compilation
(:name        "callee.o"          ; debug-only label

               ;; R-OFFSET    = section 内の patch byte 位置
               ;; SYM-NAME    = 解決対象 symbol (= 同 unit / 別 unit)
               ;; R-TYPE      = :pc32 / :abs64 / :plt32 / ...
               ;; R-ADDEND    = signed addend
               ...))
#+end_src

** §2.4 top-level orchestration =nelisp-link-units= (~50 LOC)

#+begin_src elisp
(defun nelisp-link-units (units output-path &optional opts)
  "Statically link UNITS (= compiled-unit plists from Doc 92 assembler)
and emit ELF executable to OUTPUT-PATH via Doc 91 writer.
OPTS plist: :entry-sym (default \"_start\") :base-addr (default
#x400000)."
  (let* ((entry-sym (or (plist-get opts :entry-sym) "_start"))
         (base-addr (or (plist-get opts :base-addr) #x400000))
         (layout (nelisp-link--compute-layout units base-addr))
         (combined (nelisp-link--combine-sections units layout))
         (symtab (nelisp-link--merge-symtabs units layout))
         (relocated (nelisp-link--apply-relocs units combined symtab layout))
         (entry-addr (nelisp-link--lookup-entry symtab entry-sym)))
    (nelisp-elf-write-binary
     output-path
     (append relocated
             (list :symbols (nelisp-link--export-symtab symtab)
                   :relocs nil
                   :entry-sym entry-sym
                   :entry-addr entry-addr)))))
#+end_src

** §2.5 symbol table merge =nelisp-link--merge-symtabs= (~40 LOC)

責務:
1. 全 unit の =DEFINED= (= GLOBAL / LOCAL / WEAK) symbol を 1 個の hash
   table に集約、各 symbol の絶対 vaddr = =base-addr + section-base +
   offset=
2. =EXTERN= 参照を 2nd pass で解決、無ければ error
3. 重複規則 = GLOBAL+GLOBAL → error、GLOBAL+WEAK → strong wins、
   WEAK+WEAK → first wins
4. LOCAL は unit-scoped、key = =UNIT-NAME.SYM-NAME= で mangling

#+begin_src elisp
(defun nelisp-link--merge-symtabs (units layout)
  "Build global hash keyed on symbol-name (or mangled local), value =
plist (:vaddr V :size SZ :section S :binding B :type T :unit U).  Two
passes: insert DEFINED, verify EXTERN."
  (let ((tab (make-hash-table :test 'equal)) (errors '()))
    (dolist (unit units)
      (let* ((uname (plist-get unit :name))
             (sec-base (gethash (cons uname :sections) layout)))
        (dolist (sym (plist-get unit :symbols))
          (cl-destructuring-bind (sn sk off sz binding type) sym
            (when (memq binding '(:local :global :weak))
              (let* ((key (if (eq binding :local)
                              (format "%s.%s" uname sn) sn))
                     (vaddr (+ (plist-get layout :base-addr)
                               (plist-get sec-base sk) off))
                     (existing (gethash key tab))
                     (entry (list :vaddr vaddr :size sz :section sk
                                  :binding binding :type type :unit uname)))
                (cond ((null existing) (puthash key entry tab))
                      ((and (eq binding :global)
                            (eq (plist-get existing :binding) :global))
                       (push (list :multiply-defined key uname
                                   (plist-get existing :unit)) errors))
                      ((and (eq binding :global)
                            (eq (plist-get existing :binding) :weak))
                       (puthash key entry tab)))))))))
    (dolist (unit units)
      (dolist (sym (plist-get unit :symbols))
        (cl-destructuring-bind (sn _sk _off _sz binding _type) sym
          (when (and (eq binding :extern) (not (gethash sn tab)))
            (push (list :unresolved-external sn (plist-get unit :name))
                  errors)))))
    (when errors (signal 'nelisp-link-error (list :symbol-merge errors)))
    tab))
#+end_src

** §2.6 relocation pass =nelisp-link--apply-relocs= (~80 LOC)

責務: 各 unit の =:relocs= entry を走査、patch target byte 位置を結合
後 section 内 offset に翻訳、symbol vaddr で計算、target bytes を mutate。

支援 R-TYPE 最小 set (= Phase 47 scope、=S= = symbol vaddr / =A= =
addend / =P= = patch site の絶対 vaddr):

| keyword  | ELF code (r_info)   | computes      | patch size |
|----------+---------------------+---------------+------------|
| =:pc32=  | =R_X86_64_PC32=  = 2 | =S + A - P=   | 4 byte LE  |
| =:abs64= | =R_X86_64_64=    = 1 | =S + A=       | 8 byte LE  |
| =:plt32= | =R_X86_64_PLT32= = 4 | =S + A - P=   | 4 byte LE (= static link で =:pc32= 同等) |

#+begin_src elisp
(defun nelisp-link--apply-relocs (units combined symtab layout)
  "Walk each UNIT's :relocs and patch COMBINED section bytes."
  (let ((bvs (list :text (plist-get combined :text)
                   :rodata (plist-get combined :rodata)
                   :data (plist-get combined :data)))
        (base-addr (plist-get layout :base-addr)))
    (dolist (unit units)
      (let* ((uname (plist-get unit :name))
             (sec-base (gethash (cons uname :sections) layout)))
        (dolist (rel (plist-get unit :relocs))
          (cl-destructuring-bind (sk r-off sym-name r-type r-addend) rel
            (let* ((S (or (plist-get (gethash sym-name symtab) :vaddr)
                          (signal 'nelisp-link-error
                                  (list :reloc-unresolved sym-name uname))))
                   (sec-off (plist-get sec-base sk))
                   (P (+ base-addr sec-off r-off))
                   (bv (or (plist-get bvs sk)
                           (signal 'nelisp-link-error
                                   (list :reloc-bad-section sk))))
                   (patch-pos (+ sec-off r-off)))
              (pcase r-type
                ((or :pc32 :plt32)
                 (let ((d (- (+ S r-addend) P)))
                   (nelisp-link--check-rel32-range d sym-name)
                   (nelisp-link--patch-le32 bv patch-pos d)))
                (:abs64 (nelisp-link--patch-le64 bv patch-pos (+ S r-addend)))
                (_ (signal 'nelisp-link-error
                           (list :reloc-unsupported-type r-type)))))))))
    (append bvs (list :bss-size (plist-get combined :bss-size)))))

(defun nelisp-link--check-rel32-range (d sym)
  (unless (and (>= d (- (ash 1 31))) (< d (ash 1 31)))
    (signal 'nelisp-link-error (list :rel32-overflow sym d))))

(defun nelisp-link--patch-le32 (bv pos v)
  (dotimes (i 4) (aset bv (+ pos i) (logand (ash v (- (* i 8))) #xff))))

(defun nelisp-link--patch-le64 (bv pos v)
  (dotimes (i 8) (aset bv (+ pos i) (logand (ash v (- (* i 8))) #xff))))
#+end_src

** §2.7 section combiner =nelisp-link--combine-sections= (~30 LOC)

責務: 全 unit の同種 section を unit 順に concat、unit 毎 base offset を
=layout= に記録。

#+begin_src elisp
(defun nelisp-link--combine-sections (units layout)
  "Concat .text/.rodata/.data per unit, sum .bss.  Stores per-unit base
offset in LAYOUT.  Returns plist (:text BV :rodata BV :data BV :bss-size N)."
  (let ((text (make-vector 0 0)) (rodata (make-vector 0 0))
        (data (make-vector 0 0)) (bss 0))
    (dolist (unit units)
      (let* ((uname (plist-get unit :name))
             (secs (plist-get unit :sections)))
        (puthash (cons uname :sections)
                 (list :text (length text) :rodata (length rodata)
                       :data (length data) :bss bss) layout)
        (setq text   (vconcat text   (plist-get secs :text)))
        (setq rodata (vconcat rodata (plist-get secs :rodata)))
        (setq data   (vconcat data   (plist-get secs :data)))
        (cl-incf bss (or (plist-get secs :bss-size) 0))))
    (list :text text :rodata rodata :data data :bss-size bss)))

(defun nelisp-link--compute-layout (units base-addr)
  (let ((h (make-hash-table :test 'equal)))
    (puthash :base-addr base-addr h)
    (puthash :units (mapcar (lambda (u) (plist-get u :name)) units) h)
    h))
#+end_src

* §4 Testing strategy

** §4.1 layered test

| layer | 目的                                | tool                          |
|-------+-------------------------------------+-------------------------------|
| L0    | reloc handler unit (= patch math)   | ert (pure elisp, fixed bytes) |
| L1    | symtab merge                         | ert (synthetic units)         |
| L2    | section combine                      | ert (concat invariant)        |
| L3    | full link round-trip (single unit)   | ert + Doc 91 writer + =readelf= |
| L4    | multi-unit cross-reference link      | ert + =readelf -r= sanity     |
| L5    | exec test (= 実 process 起動)        | ert + =call-process=          |
| L6    | =gas + ld= baseline と bytewise diff | shell + =cmp -l=              |

L6 = "system linker と elisp linker の output が **modulo timestamp /
build-id** で一致する" 究極証明。

** §4.2 round-trip test (= 単一 unit)

最小 unit = =_start= が =B8 3C 00 00 00= (=MOV EAX, 60= = =SYS_exit=) +
=48 31 FF= (=XOR RDI, RDI=) + =0F 05= (=SYSCALL=) の 10 byte =.text=、
symbol = =((_start :text 0 10 :global :func))=、reloc 0。

ert は: =nelisp-link-units= で binary 出力 → =readelf -h= で
=ELF64 / X86-64 / Type:.*EXEC= を =string-match-p= で照合 → =call-process=
で実行 → =should (= rc 0)=。=unwind-protect= で temp file 削除。

** §4.3 multi-unit cross-reference test (= 真の linker test)

本 doc の真価は **複数 unit 間 symbol 解決**。最小 reproduction =
"caller.o" の =_start= が CALL で "callee.o" の =callee_fn= を呼び、
callee は =XOR EAX, EAX; RET= で 0 を返し、caller が =SYS_exit(0)= で
正常終了する pattern:

- caller =.text= = =E8 00 00 00 00= (CALL placeholder) + =B8 3C 00 00 00=
  (=MOV EAX,60=) + =48 31 FF= (=XOR RDI,RDI=) + =0F 05= (=SYSCALL=)
- caller :symbols = ((_start :text 0 15 :global :func) (callee_fn nil 0 0
  :extern :func))
- caller :relocs = ((:text 1 "callee_fn" :pc32 -4))
- callee =.text= = =31 C0 C3= (=XOR EAX,EAX; RET=)、:symbols =
  ((callee_fn :text 0 3 :global :func))

linker は =callee_fn= を解決、CALL の 4 byte placeholder を =callee_fn
vaddr - (CALL site + 4)= で patch。exec → exit code 0 (= ert
=call-process= で =should (= rc 0)=)。

** §4.4 system-linker baseline diff (= L6)

同 2 unit を =gas + ld -static -e _start= で link した baseline ELF を
git に commit、elisp linker output と =cmp -l= で diff。timestamp /
=.note.gnu.build-id= section を =objcopy --remove-section= で mask した
上で **bytewise identical** を期待。

実態としては section ordering / =.shstrtab= ordering / alignment 詳細で
微差が出る可能性高 (= ld は決定論的だが convention は実装依存)。よって
本 doc は「**modulo section ordering identical**」を目標とし、=readelf
-S= の section 内容を sort してから diff する形に緩和する。

** §4.5 corpus

minimal ~5 sample 結合を test corpus として固定:

1. 単一 unit、=exit(0)= (= §4.2)
2. 2 unit cross-call、callee = =return 0= (= §4.3)
3. 3 unit cross-call (= caller → middle → callee 鎖)
4. =.rodata= absolute reference (= =MOV RAX, [rip+offset]= で string
   literal pull)
5. =.data= mutation (= initialized global incrementing)

各 sample で gas + ld baseline を git に commit (= regression lock)、
elisp linker output と periodic diff (= L6)。

* §6 Risk / unknowns

** §6.1 PIE / PIC support

問題: 本 doc は ET_EXEC + 固定 vaddr =0x400000= 前提。modern Linux
distro は ASLR enabled = ET_DYN (PIE) + GOT/PLT machinery を要求する場合
あり。PIE 化に必要 = =.got= / =.got.plt= section emit + =R_X86_64_GOTPCREL=
真解決 + =R_X86_64_RELATIVE= dynamic reloc emit (= load 時に dynamic
linker が patch、本 doc scope 外) + =PT_DYNAMIC= segment emit。

緩和: Phase 47 = ET_EXEC 維持、modern distro でも ET_EXEC は exec 可能
(= ASLR 無視されるが exec 自体は OK)。PIE 化は Phase 48+ 候補。判断:
medium risk、将来 PIE 要件発生で linker LOC ~+200 増加可能性。

** §6.2 Dynamic library linking

libc / libm / libpthread を dynamic link する path は本 doc で **完全
廃止**。Phase 47 方針 = pure syscall binary (Doc 94 inline syscall scope)。
libm の =strtod= / =sin= / =cos= 等は elisp 実装で置換、=malloc= /
=free= は elisp GC 経由、threading は elisp single-thread。判断: Phase
47 全体で解決済前提、本 doc は static link only に絞れば risk なし。

** §6.3 Debug info (DWARF)

=gdb= / =addr2line= 等の tool は =.debug_info= / =.debug_line= section
を読む。本 doc は **DWARF emit せず** = =gdb= で "no symbols"。但し
=.symtab= / =.strtab= は emit する = =nm= で symbol 表示可能。Doc 91
writer の DWARF 拡張 (Doc 91 §7.5 risk) で後続 doc 対応。判断: low-med
risk、scope 外。

** §6.4 LTO / dead-code elimination

本 doc は section 全体保持 = unused function も全部 binary 内。LTO は
inter-unit call graph + 到達性 + section GC が必要。Phase 47 完遂後の
最適化 phase で対応、minimum viable は size 度外視で OK。判断: low risk、
scope 明示で済。

** §6.5 PC32 range overflow

=R_X86_64_PC32= = signed 32-bit displacement (±2 GB)。=.bss= / =.rodata=
肥大時 hit 可能性。§3.5 で range check + error throw articulated、越えた
場合 =R_X86_64_64= fallback だが instruction encoding 変更要 (= Doc 92
concern)。判断: monitor 要、Phase 47 binary < 10 MB 想定で当面 OK。

** §6.6 Symbol mangling collision / weak symbol semantics

LOCAL mangle key = =UNIT.SYM=、unit 名 duplicate で collision。caller
契約で unique 保証、本 doc 内 duplicate 検出 → error。WEAK + GLOBAL は
strong wins、WEAK + WEAK は first wins (§2.5)。GNU ld と LLVM lld の微差
は ert edge case で固定。判断: low risk、契約 + ert で済。

** §6.7 elisp 性能 (= ~1 MB binary link speed)

100 unit × 各 100 reloc = 10,000 reloc walk + patch ≈ 10 sec オーダー
(= ~1 ms / reloc)。緩和 = =gethash= 多用 + byte vector pre-alloc +
=aset= mutate + 必要なら C =reloc-patch-many= escape hatch。Phase 47
binary 数 (~50 unit 想定) では問題なし、肥大時 optimization 要。判断:
medium risk。

** §6.8 Risk summary 表

| risk                              | severity | mitigation                                  |
|-----------------------------------+----------+---------------------------------------------|
| §6.1 PIE / PIC                    | medium   | Phase 47 ET_EXEC 維持、PIE は Phase 48+    |
| §6.2 dynamic lib linking          | low      | Phase 47 全体方針で解決済                  |
| §6.3 DWARF debug info             | low-med  | scope 外、後続 doc                          |
| §6.4 LTO / DCE                    | low      | scope 外、後続 doc                          |
| §6.5 PC32 range overflow          | medium   | range check + R_X86_64_64 fallback         |
| §6.6 mangling / weak              | low      | 契約 + ert edge case                        |
| §6.7 elisp 性能                   | medium   | hash + byte vector pre-alloc + escape C    |

*top 2 risks for implementer*:

1. *§6.1 PIE / PIC* = modern distro で hardened kernel 強制時に Phase 47
   binary が exec 不能になる 1 アーキ越境問題、ET_EXEC 維持で当面 OK
   だが distro 監視要
2. *§6.7 elisp 性能* = 10,000 reloc で 10 sec オーダ、Phase 47 build
   time 全体で許容範囲かは benchmark 要
