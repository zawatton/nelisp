* §0 TL;DR

** §0.1 Why this doc exists

`build-tool/src/eval/rc_primitives.rs' (464 LOC) hosts the elisp-
facing refcount + GC primitives backing the Bacon-Rajan cycle
collector skeleton in `lisp/nelisp-stdlib-gc.el'.  Every primitive
in the file follows the same three-step shape:

1. Sexp unwrap (= match on `Sexp::Cons(r) => ...` etc.).
2. *Atomic memory op on the refcount slot* (= `fetch_add(1,
   Relaxed)` / `fetch_sub(1, Release)`).
3. Sexp wrap (= `Ok(Sexp::Int(new_count))` / `Ok(Sexp::Nil)`).

Step 2 is the only "real algorithm".  Steps 1 + 3 are mechanical
ABI boilerplate already established by Doc 99/100/101.  Doc 122.E
(= `atomic-fetch-add' + `ptr-read-u64' grammar ops) is the gate:
once it shipped, Step 2 can be written as a one-line elisp body.

Per the Doc 79 v6 Track L pattern (= refcount mutation goes through
a per-box typed pointer at the call site), each rc primitive
materializes the *NlConsBox refcount slot address* = `box_ptr +
REFCOUNT_OFFSET` and feeds it to `atomic-fetch-add'.  The offset
is a `#[repr(C)]`-pinned constant (= 64 for NlConsBox = 2 *
`size_of::<Sexp>()` = 2 * 32) so the elisp body needs no extern
hop for the address arithmetic.

** §1.1 Per-macro feasibility table

| Stage  | Symbols                                              | Feasibility | Risk             |
|--------+------------------------------------------------------+-------------+------------------|
| §123.A | `bi_nl_rc_inc_strong`                                | TRIVIAL     | Race vs SeqCst   |
| §123.B | `rc_dec_no_drop` + `bi_nl_rc_dec_strong`             | EASY        | Saturating sub   |
| §123.C | `bi_nl_rc_strong_count` + `bi_nl_rc_kind`            | EASY        | Tag dispatch     |
| §123.D | `bi_nl_rc_payload_ptr` + `bi_nl_gc_walk_children` +   | MEDIUM      | Refcount-aware  |
|        | `bi_nl_gc_buffered_decs` + helpers                   |             | clone for kids  |
| §123.E | `bi_nl_rc_alloc` + `bi_nl_rc_dealloc` +              | HARD        | Defer to Doc 125 |
|        | `bi_nl_gc_finalize`                                  |             | (alloc/dealloc)  |

*Risk notes*:

- §123.A: §122.E `atomic-fetch-add' uses SeqCst, Rust's
  `rc_inc_raw' uses Relaxed.  SeqCst ⊃ Relaxed (= stronger ordering
  is always valid for an inc-only operation); the cost is a single
  `mfence' on x86_64 which is acceptable for the rc-bump fast path.
  Doc 122.E.2 may add a `Relaxed' variant if profiling shows it.
- §123.B: `saturating_sub` is a single `cmp + cmov`; expressible as
  `(if (= prev 0) 0 (- prev 1))' in elisp.
- §123.C: `Acquire` load — §122.E ships `ptr-read-u64` with no
  explicit ordering (= compiler defaults are sufficient for a
  read-only load of an atomic slot that no writer is racing with).
- §123.D: requires §122.E `ptr-read-u64' for the car/cdr pointer
  walks + cons-make to materialize the result list.  Refcount
  bumping for the cloned kids reuses §123.A's atomic-fetch-add.
- §123.E: `bi_nl_rc_alloc' allocates a fresh NlConsBox via the Rust
  `alloc::alloc` global allocator.  AOT has no alloc grammar
  op as of Doc 122; this is Doc 125 (= alloc/dealloc handlers).
  §123.E ships *after* Doc 125 lands.

* §2 Stage breakdown

** §2.1 §123.A — `nelisp_rc_inc` (simplest macro elisp化)

- New file: `lisp/nelisp-cc-rc-inc.el` (= ~10 elisp LOC).
- Body:
  #+begin_src elisp
  (defun nelisp_rc_inc (box-ptr)
    (atomic-fetch-add (+ box-ptr 64) 1))
  #+end_src
  - `box-ptr' = raw `*const NlConsBox' as i64 (= the same pointer
    `NlConsBoxRef::as_ptr' returns to `bi_nl_rc_payload_ptr').
  - `+ box-ptr 64' = REFCOUNT_OFFSET (= 64 bytes = 2 *
    `size_of::<Sexp>()`).  Constant verified at the §122.E layout
    assert in `nlrc.rs:292`.
  - `atomic-fetch-add' returns the *pre-add* i64 value in rax.
- New extern: =extern "C" fn nelisp_rc_inc(box_ptr: *mut i64) -> i64=
  declared in `build-tool/src/lib.rs::elisp_cc_spike`.
- Probe: `build-tool/tests/elisp_cc_rc_inc_probe.rs' (≥ 3 cases):
  1. Single inc on a refcount=1 slot returns 1 (pre-add) and writes 2.
  2. Multiple inc accumulates monotonically.
  3. Atomic ordering check — concurrent inc from 2 host threads
     converges to N1+N2 total (no lost updates).
- Verification: existing `rc_primitives.rs::tests::rc_prim_inc_dec_balance`
  still passes (= the Rust `bi_nl_rc_inc_strong` still calls
  `rc_inc_raw'; this stage only adds the elisp counterpart, does
  not swap the dispatch yet).  §123.B swaps the dispatch.
- LOC: +10 elisp / +5 Rust extern wrapper / -0 Rust (= no removal
  yet; §123.B removes the Rust body after the swap lands).

** §2.6 §123.F — dispatch sweep (Rust algorithmic bodies → elisp kernels)

- *SHIPPED 2026-05-18*: swaps the Rust algorithmic bodies of the 4
  `bi_nl_*' primitives whose elisp kernels SHIPPED in §123.A-C for
  thin AOT dispatches.  This is where Rust LOC actually
  disappears — §123.A-C only landed the kernels + probes alongside
  the existing Rust bodies.
- Swapped primitives:
  - `bi_nl_rc_inc_strong' → `crate::elisp_cc_spike::rc_inc' (§123.A
    kernel).  Pre-add i64 return discarded (= Sexp surface stays
    Nil).
  - `bi_nl_rc_dec_strong' → `crate::elisp_cc_spike::rc_dec' (§123.B
    kernel).  Pre-sub i64 returned by the kernel; the Rust shim
    applies `saturating_sub(1)' that the deleted `rc_dec_no_drop'
    helper performed inline so the Sexp surface still reports the
    *new* count (= 0 floor preserved).
  - `bi_nl_rc_strong_count' → `crate::elisp_cc_spike::rc_strong_count'
    (§123.C kernel).
  - `bi_nl_rc_kind' → `crate::elisp_cc_spike::rc_kind' (§123.C
    kernel).
- *Deleted helper*: `rc_dec_no_drop' free fn (= the
  `fetch_sub(1, Release) + saturating_sub' inline helper).  Only
  caller was `bi_nl_rc_dec_strong' itself; grep across `build-tool/
  src/' confirms no external callers.
- *Kept*: `NlConsBoxRef::rc_inc_raw' (method, NOT a free fn) —
  also used by `cons_primitives.rs:113'.  The elisp dispatch in
  `bi_nl_rc_inc_strong' uses `NlConsBoxRef::as_ptr' + the elisp
  kernel directly instead of this method, but the method survives
  for the cons-primitive caller.
- *Verification*:
  - All 10 existing `eval::rc_primitives::tests::*' pass unchanged
    (= behavior preserved at the Sexp surface).
  - 4 elisp kernel probes (`elisp_cc_rc_{inc,dec,kind,strong_count}_probe')
    pass unchanged.
  - `cargo test --release --lib': 733 passed (= baseline maintained).
  - `cargo test --release --tests -p nelisp-build-tool': 1048 passed
    (= above 1006 baseline).
- *LOC delta*: `rc_primitives.rs' 464 → 481 (= +17 net, but: ~25 Rust
  algorithmic lines deleted vs. ~42 lines of dispatch + ample
  inline doc-comments documenting the §123.F migration).  The
  Doc 99/100/101 swap pattern documents the *kernel* in elisp and
  the *contract* in the Rust shim; the doc-overhead is paid once,
  Rust algorithmic surface area is gone.
