** §2.3 Bootstrap circularity assessment

| substrate sym | 依存 | 提供方法 |
|---------------+------+----------|
| =cond= macro | =if= (special form) + =defmacro= (= 自己起動 boot) | =fset 'cond (cons 'macro ...)= で初期 install |
| =when= / =unless= macro | =if= + =progn= (special form) + =defmacro= | 同上 |
| =defmacro= macro 自体 | =fset= + =cons= (Rust builtin) + =lambda= (special form) | =nelisp-stdlib-eval-special.el= 49-62 の bootstrap pattern を流用 |
| =defun= macro | =fset= + =lambda= + =defmacro= (上で定義済) | 同 file 後半に order |
| =signal-helper= 関数 | =signal= (Rust builtin) + =cons= + =quote= | 直接 =fset= で install |

循環は無い: =if= は special form で常時 available、=defmacro= は self-bootstrap pattern が =nelisp-stdlib-eval-special.el= 49-62 で実証済み、=signal= は Rust builtin で常時 available.

唯一 risk: =cond= 内で使われる =null= (= =(eq x nil)= の elisp wrapper) は =nelisp-stdlib.el= で定義される.  substrate では =null= を使わず inline で =(if x ... ...)= を書く (= or =(eq x nil)= 直書き).  既存 =nelisp-stdlib-eval-special.el= の =cond= macro body は =null= を使っているが、これは現 STDLIB_FILES[1] で =null= は Rust builtin (= まだ shadowed されてない) として reach 可能だった経緯による.  substrate では =null= を Rust builtin 経由で reach できるかどうかを確認 (= =builtins.rs:79-87= "null shadowed by elisp 2026-05-06 batch 6d" 記述 — Rust 側 =null= は既に削除済の可能性 = =substrate= では =eq x nil= 直書きが安全).

** §3.3 implementation skeleton (= 抜粋)

#+begin_src emacs-lisp
;;; nelisp-jit-substrate.el --- Pre-stdlib expression substrate (Doc 80) -*- lexical-binding: t; -*-

;;; Code:

;; ---- 1. defmacro self-bootstrap (lifted from nelisp-stdlib-eval-special.el 49-62) ----
(fset 'defmacro
      (cons 'macro
            (cons (lambda (name args &rest body)
                    (let* ((lambda-form (cons 'lambda (cons args body)))
                           (qname (cons 'quote (cons name nil)))
                           (inner-cons (cons 'cons (cons lambda-form (cons nil nil))))
                           (outer-cons (cons 'cons
                                             (cons (cons 'quote (cons 'macro nil))
                                                   (cons inner-cons nil)))))
                      (cons 'progn
                            (cons (cons 'fset (cons qname (cons outer-cons nil)))
                                  (cons qname nil)))))
                  nil)))

;; ---- 2. when / unless / cond ----
(defmacro when (test &rest body)
  (cons 'if (cons test (cons (cons 'progn body) (cons nil nil)))))

(defmacro unless (test &rest body)
  (cons 'if (cons test (cons nil (cons (cons 'progn body) nil)))))

(defmacro cond (&rest clauses)
  (if (eq clauses nil)
      nil
    (let* ((clause (car clauses))
           (rest (cdr clauses))
           (test (car clause))
           (body (cdr clause)))
      (if (eq body nil)
          (cons 'let
                (cons (cons (cons '--nl-cond-tmp (cons test nil)) nil)
                      (cons (cons 'if
                                  (cons '--nl-cond-tmp
                                        (cons '--nl-cond-tmp
                                              (cons (cons 'cond rest) nil))))
                            nil)))
        (cons 'if
              (cons test
                    (cons (cons 'progn body)
                          (cons (cons 'cond rest) nil))))))))

;; ---- 3. defun (after defmacro) ----
(defmacro defun (name args &rest body)
  (let ((lambda-form (cons 'lambda (cons args body)))
        (qname (cons 'quote (cons name nil))))
    (cons 'progn
          (cons (cons 'fset (cons qname (cons lambda-form nil)))
                (cons qname nil)))))

;; ---- 4. null / not ----
(defun null (x) (eq x nil))
(defun not (x) (eq x nil))

;; ---- 5. signal helpers ----
(defun nelisp--signal-wrong-type (expected got)
  (signal 'wrong-type-argument (cons expected (cons got nil))))

(defun nelisp--signal-arith (data)
  (signal 'arith-error data))

;; ---- 6. type-tag thin alias ----
(defun nelisp--type-tag (x) (type-of x))

(provide 'nelisp-jit-substrate)
;;; nelisp-jit-substrate.el ends here
#+end_src

* §5 risk + mitigation

** §5.1 Bootstrap circularity

** §5.2 Performance regression

Doc 62 Phase 5 / Doc 76 bench history を Stage 80.3 / 80.4 完了直後に re-run、specific bench tagged regression > 5% なら =strategy.rs= 残し戻す revert option を保持.

** §5.3 Bytecode dependency

=nelisp-baker.rs= の =compile_elisp_to_image= は新 file をそのまま bake できることを確認.  pre-stdlib の elisp file は既に nelisp-jit-strategy.el で前例があり、Tier 1 special form + Rust builtin only の制約下で問題なく compile されている.
