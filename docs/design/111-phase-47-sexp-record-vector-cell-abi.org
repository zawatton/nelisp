* §0 TL;DR

** §0.1 What Doc 101 leaves unfinished

Doc 101 covers Sexp::Cons, Sexp::Symbol, Sexp::Str — the variants
needed to walk bucket alists (= Cons chains) and compare keys
(= Str byte equality).  But the structures the buckets *live
inside* are:

- =Sexp::Record(NlRecordRef)= — the =nelisp-env= / =fast-hash-table=
  / =symbol-entry= / =nelisp-lexframe= / =nelisp-lexframe-stack=
  records.  Without Record read ops, no elisp helper can find the
  bucket vector to start walking.
- =Sexp::Vector(NlVectorRef)= — the bucket array (= =buckets= slot
  of =fast-hash-table=) + the lexframe-stack BACKING vector.  Without
  Vector read ops, no helper can index into the bucket array by
  hash.
- =Sexp::Cell(NlCellRef)= — the write-through cell that backs
  let-bindings (= every entry in a lexframe is a =Sexp::Cell=).
  Without Cell read+write ops, =setq= inside a closure can't update
  the let-binding's slot.

These three variants are the "structural backbone" that env_mirror
+ env_lexframe walk; Doc 111 supplies the ABI to walk them at
native speed.

** §0.2 The collection-payload question

Doc 100 (Sexp::Int) was atomic — payload inline, no indirection.
Doc 101 (Sexp::Cons) was boxed — payload behind one pointer, fixed-
size NlConsBox.  Doc 111 (Record / Vector / Cell) is boxed *with
variable-size interior*:

- =NlRecord= holds a =Vec<Sexp>= (= variable-length slots array).
- =NlVector= holds a =Vec<Sexp>= (= variable-length elements).
- =NlCell= holds a single =Sexp= (= fixed-size, simplest of the
  three).

The =Vec<Sexp>= header layout (ptr / cap / len at offsets 0 / 8 /
16) is the same =Vec= we already documented in Doc 101 §7.  Doc 111
extends that to the Sexp-element case: =Vec<Sexp>= means each
element is a 32-byte =Sexp= slot, so element indexing requires a
=lea= for the base pointer + =N * 32= offset arithmetic.

* §1 Background

** §1.1 Where env_mirror / env_lexframe use Record / Vector / Cell

Concrete walks in =build-tool/src/eval/env_mirror.rs= +
=env_lexframe.rs= (post-Phase-8 split):

*Record walks* — finding the bucket vector inside the env record:

#+begin_src rust
  // env_mirror.rs::mirror_lookup_entry, simplified:
  let env_rec = match env_record { Sexp::Record(r) => r, _ => return None };
  let ht_rec = match env_rec.slots.get(0)? { Sexp::Record(r) => r, _ => return None };
  let buckets = match ht_rec.slots.get(1)? { Sexp::Vector(v) => v, _ => return None };
#+end_src

Three Record dereferences: env → fast-hash-table → buckets-slot.
Each is a Vec indexed read.

*Vector walks* — indexing into the bucket array:

#+begin_src rust
  let h = mirror_fnv1a(name);
  let idx = (h & (bucket_count - 1)) as usize;
  let bucket = buckets.value.get(idx)?;
#+end_src

One indexed read into a =Vec<Sexp>=.

*Cell write-through* — closure setq:

#+begin_src rust
  // env_lexframe.rs::wrap_alist_cells (cell construction at bootstrap):
  let cell = match &inner.cdr {
      Sexp::Cell(_) => inner.cdr.clone(),
      v => Sexp::Cell(FrameCell::new(v.clone())),
  };
#+end_src

#+begin_src rust
  // setq via captured cell (= eval::env::set_value at lexical hit):
  unsafe { cell.set_value(value.clone()) };
#+end_src

Read of the cell's current value (via =cell.value.clone()=) + write
of a new value (via =set_value=) are both needed.

** §1.3 Element indexing in Vec<Sexp>

Reading the N-th Sexp from a =Vec<Sexp>= header at offset O of an
NlXxx box:

#+begin_src text
  mov rax, qword ptr [box + O + 0]    ; rax = vec.ptr (= Sexp array base)
  imul rcx, rsi, 32                   ; rcx = N * 32 (= byte offset)
  add rax, rcx                        ; rax = ptr to N-th Sexp
#+end_src

For an out-of-bounds read the caller must compare =N < len= first
(via =mov rcx, [box + O + 16]= for the len field) — Phase 47's
=if= form handles the dispatch.

Reading the *length*:

#+begin_src text
  mov rax, qword ptr [box + O + 16]   ; rax = vec.len
#+end_src

Doc 100 §5.1 + Doc 101 §5.1 — same pattern.  Doc 111 §111.A

* §2 ABI design

** §2.1 Sexp::Record read ops

Three new value-producing grammar forms:

*1. =(record-type-tag H SLOT)= — read the type_tag Sexp into a slot*

Loads =NlRecordRef= from =[H + 8]=, then copies 32-byte =type_tag=
from =[record + 0]= into the caller-provided slot.  Two SIMD copies
(same pattern as Doc 101 §2.1 =cons-car=):

#+begin_src text
  mov rsi, [rdi + 8]                ; rsi = NlRecord*
  movdqu xmm0, [rsi + 0]
  movdqu xmm1, [rsi + 16]
  movdqu [rdx + 0], xmm0
  movdqu [rdx + 16], xmm1
  mov rax, rdx                      ; return slot pointer
#+end_src

*2. =(record-slot-count H)= — read the slots Vec's length*

The slots Vec's length is at offset 32 + 16 = 48 of NlRecord.

#+begin_src text
  mov rsi, [rdi + 8]                ; rsi = NlRecord*
  mov rax, qword ptr [rsi + 48]     ; rax = slots.len
#+end_src

*3. =(record-slot-ref H N SLOT)= — copy the N-th slot into SLOT*

Composed: locate slots.ptr at offset 32 + 0 = 32, multiply N by 32,
copy 32-byte Sexp:

#+begin_src text
  mov rsi, [rdi + 8]                ; rsi = NlRecord*
  mov rcx, qword ptr [rsi + 32]     ; rcx = slots.ptr (Sexp* base)
  imul r10, rsi, 32                 ; r10 = N * 32  (N in rsi by ABI)
  add rcx, r10                      ; rcx = ptr to N-th Sexp
  movdqu xmm0, [rcx + 0]
  movdqu xmm1, [rcx + 16]
  movdqu [rdx + 0], xmm0
  movdqu [rdx + 16], xmm1
  mov rax, rdx
#+end_src

*Const additions* (= =lisp/nelisp-sexp-layout.el=):

#+begin_src elisp
  (defconst nelisp-nlrecord--offset-type-tag       0)
  (defconst nelisp-nlrecord--offset-slots-vec      32)
  (defconst nelisp-nlrecord--offset-slots-ptr      32)   ; = slots-vec + 0
  (defconst nelisp-nlrecord--offset-slots-capacity 40)   ; = slots-vec + 8
  (defconst nelisp-nlrecord--offset-slots-length   48)   ; = slots-vec + 16
  (defconst nelisp-nlrecord--offset-refcount       56)
  (defconst nelisp-nlrecord--size                  64)
#+end_src

** §2.2 Sexp::Vector read ops

Three ops paralleling §2.1:

*1. =(vector-len H)= — read the value Vec's length*

NlVector.value.len lives at offset 0 + 16 = 16 of the NlVector box.

#+begin_src text
  mov rsi, [rdi + 8]                ; rsi = NlVector*
  mov rax, qword ptr [rsi + 16]     ; rax = value.len
#+end_src

*2. =(vector-ref H N SLOT)= — copy the N-th element into SLOT*

#+begin_src text
  mov rsi, [rdi + 8]                ; rsi = NlVector*
  mov rcx, qword ptr [rsi + 0]      ; rcx = value.ptr (Sexp* base)
  imul r10, rsi, 32                 ; r10 = N * 32  (N in rsi by ABI)
  add rcx, r10                      ; rcx = ptr to N-th Sexp
  movdqu xmm0, [rcx + 0]
  movdqu xmm1, [rcx + 16]
  movdqu [rdx + 0], xmm0
  movdqu [rdx + 16], xmm1
  mov rax, rdx
#+end_src

*3. =(vector-ref-ptr H N)= — return raw =*const Sexp= to N-th element*

Used internally by bucket walks that don't need to copy.  Returns a
pointer the caller may immediately re-dereference (= no clone, no
refcount touch).

#+begin_src text
  mov rsi, [rdi + 8]                ; rsi = NlVector*
  mov rcx, qword ptr [rsi + 0]      ; rcx = value.ptr
  imul r10, rsi, 32                 ; r10 = N * 32
  lea rax, [rcx + r10]              ; rax = &vec[N]
#+end_src

*Const additions*:

#+begin_src elisp
  (defconst nelisp-nlvector--offset-value-vec      0)
  (defconst nelisp-nlvector--offset-value-ptr      0)
  (defconst nelisp-nlvector--offset-value-capacity 8)
  (defconst nelisp-nlvector--offset-value-length   16)
  (defconst nelisp-nlvector--offset-refcount       24)
  (defconst nelisp-nlvector--size                  32)
#+end_src

** §2.3 Sexp::Cell read + write-through ops

The cell's =value= Sexp lives at offset 0 of NlCell.  Read + write
are the only ops needed; =NlCell::set_value='s safety contract
(= drop-then-write semantics in =build-tool/src/eval/nlcell.rs=)
must be respected.

*1. =(cell-value H SLOT)= — copy the cell's current value into SLOT*

#+begin_src text
  mov rsi, [rdi + 8]                ; rsi = NlCell*
  movdqu xmm0, [rsi + 0]
  movdqu xmm1, [rsi + 16]
  movdqu [rdx + 0], xmm0
  movdqu [rdx + 16], xmm1
  mov rax, rdx
#+end_src

*2. =(cell-set-value H VAL-PTR)= — replace the cell's value*

Drops the old value (= calls =nl_drop_sexp= for box-tagged old) then
writes the new 32-byte Sexp from =[VAL-PTR + 0]= into =[cell + 0]=.

Drop dispatch table: read the cell's current tag at =[cell + 0]=,
conditionally call =nl_rc_dec= for boxed variants (= same table
Doc 101 §2.6 documents).  Then copy the new 32 bytes in.

#+begin_src text
  mov rsi, [rdi + 8]                ; rsi = NlCell*
  ; --- conditional drop of old value ---
  movzx rax, byte [rsi + 0]         ; rax = old.tag
  ; ... dispatch on box-variant tag, conditional call to nl_rc_dec ...
  ; (templated by the same emit table Doc 101 §2.6 generates)
  ; --- write new value ---
  movdqu xmm0, [rdx + 0]
  movdqu xmm1, [rdx + 16]
  movdqu [rsi + 0], xmm0
  movdqu [rsi + 16], xmm1
  ; --- conditional incref on new value ---
  movzx rax, byte [rsi + 0]
  ; ... dispatch on box-variant tag, conditional call to nl_rc_inc ...
  mov rax, rdi                      ; return H (= cell pointer)
#+end_src

The conditional drop+incref machinery is non-trivial; Doc 111 §3.D
defers the full implementation to a sub-stage that ships after the
read-only ops have proven the read path.

*Const additions*:

#+begin_src elisp
  (defconst nelisp-nlcell--offset-value     0)
  (defconst nelisp-nlcell--offset-refcount  32)
  (defconst nelisp-nlcell--size             40)
#+end_src

** §2.5 Worked example — mirror_lookup_entry replacement

Current Rust (= =build-tool/src/eval/env_mirror.rs::mirror_lookup_entry=):

#+begin_src rust
  pub(crate) fn mirror_lookup_entry(
      env_record: &Sexp,
      name: &str,
  ) -> Option<NlRecordRef> {
      let env_rec = match env_record { Sexp::Record(r) => r, _ => return None };
      let ht_rec = match env_rec.slots.get(0)? { Sexp::Record(r) => r, _ => return None };
      let bucket_count = match ht_rec.slots.get(0)? { Sexp::Int(n) => *n as u32, _ => return None };
      let buckets = match ht_rec.slots.get(1)? { Sexp::Vector(v) => v, _ => return None };
      let h = mirror_fnv1a(name);
      let idx = (if bucket_count & (bucket_count - 1) == 0 {
          h & (bucket_count - 1)
      } else {
          h % bucket_count
      }) as usize;
      let bucket = buckets.value.get(idx)?;
      let mut cur = bucket;
      while let Sexp::Cons(c) = cur {
          if let Sexp::Cons(pair) = &c.car {
              if let Sexp::Str(k) = &pair.car {
                  if k == name {
                      if let Sexp::Record(r) = &pair.cdr {
                          return Some(r.clone());
                      }
                  }
              }
          }
          cur = &c.cdr;
      }
      None
  }
#+end_src

Phase 47 elisp source (= §111.E rewrite):

#+begin_src elisp
  (defun nelisp_mirror_lookup_entry (env-ptr name-ptr result-slot)
    ;; env-ptr:     *const Sexp pointing at Sexp::Record(`nelisp-env')
    ;; name-ptr:    *const Sexp pointing at Sexp::Str(name)
    ;; result-slot: *mut Sexp 32-byte slot for the symbol-entry Record
    ;;              (or Sexp::Nil if absent).
    (let* ((ht-slot-ptr (record-slot-ref-ptr env-ptr 0))      ; *const Sexp -> ht record
           (bc-slot-ptr (record-slot-ref-ptr ht-slot-ptr 0))  ; *const Sexp -> bucket-count Int
           (bucket-count (sexp-int-unwrap bc-slot-ptr))
           (buckets-slot-ptr (record-slot-ref-ptr ht-slot-ptr 1))  ; *const Sexp -> buckets Vector
           ;; FNV-1a hash inline (Doc 101 §2.2 str-byte-at + arithmetic)
           (h (nelisp_fnv1a_bytes name-ptr))
           ;; power-of-2 fast-path (bucket-count & (bucket-count - 1) == 0)
           (idx (if (= 0 (logand bucket-count (- bucket-count 1)))
                    (logand h (- bucket-count 1))
                  (mod h bucket-count)))
           (bucket-ptr (vector-ref-ptr buckets-slot-ptr idx))
           (cur-ptr bucket-ptr))
      ;; Walk the cons chain (= Doc 101 §2.1 cons-cdr-raw-from-box loop)
      (while (and (not (= 0 (sexp-payload-ptr cur-ptr)))
                  ;; ... pair match logic ...)
        ;; ... compare *(pair.car) bytes against name-ptr bytes ...
        ;; ... if equal, copy *(pair.cdr) into result-slot, return ...
        (setq cur-ptr (cons-cdr-raw cur-ptr)))
      ;; Not found — write Sexp::Nil into result-slot.
      (sexp-write-nil result-slot)))
#+end_src

Body relies on Doc 100 (=sexp-int-unwrap=) + Doc 101 (=cons-cdr-raw=
+ =sexp-payload-ptr= + str byte access) + Doc 111 (=record-slot-ref-ptr=
+ =vector-ref-ptr=).  Pure elisp, no Rust on the call path except
the extern shim that triggers the call.

The =record-slot-ref-ptr= op is a §111.B addition I didn't list
above — it returns a raw =*const Sexp= to the slot (= same pattern
as =vector-ref-ptr= §2.2.3).  Useful when the caller will
immediately re-dereference into another Record / Vector without
needing to copy the intermediate Sexp.

Add to §111.B grammar:

#+begin_src text
  (record-slot-ref-ptr H N)
    mov rsi, [rdi + 8]                ; rsi = NlRecord*
    mov rcx, qword ptr [rsi + 32]     ; rcx = slots.ptr
    imul r10, rsi, 32                 ; r10 = N * 32   (N from rsi)
    lea rax, [rcx + r10]              ; rax = &slots[N]
#+end_src

** §2.6 ELF / linker mechanics

Same as Doc 101: read ops compile to direct memory access with no
relocations.  §111.D Cell set ops need =call nl_rc_dec= +
=call nl_rc_inc= for refcount maintenance — extern-call via §100.A
grammar, PLT32 relocations resolved by the linker.

§111.E uses =call mirror_fnv1a= (= reuse the existing Rust helper)
for the FNV-1a hash temporarily; a future elisp implementation of
FNV-1a is plausible (= Doc 101 §2.2 already exposes =str-byte-at=)
but adds elisp complexity.  §111.E first ship can use the Rust
helper; a follow-up swap can elisp-ify =mirror_fnv1a=.

* §5 Risks / unknowns

** §5.1 Vec<Sexp> stride invariant

Each element is 32 bytes (= =SEXP_SLOT_SIZE=).  The =imul r10, rsi,
32= constant must match.  Mitigation: =const_assert!= already
asserts this in Doc 100 §100.B; reuse the same constant.

** §5.2 Refcount safety in =cell-set-value=

The drop+write pattern (= drop_in_place then write) requires the
old value's destructor to complete before the new value is written.
The Phase 47 emit must respect this ordering even when the compiler
reorders instructions for ILP.

*Mitigation*: emit a memory fence (= or call into a Rust =unsafe fn=
helper that does the drop_in_place + write) before the new write.
The current =NlCell::set_value= Rust impl uses =std::ptr::drop_in_place=
+ =std::ptr::write= sequentially; the Phase 47 emit reuses the same
helper via =call nl_cell_set_value= rather than re-implementing the
unsafe sequence inline.

This reduces §111.D ambition: =cell-set-value= becomes a thin wrapper
around an existing Rust helper, not a from-scratch elisp emit.
Slightly less pure-elisp but markedly safer.

** §5.3 Vec<T> capacity-doubling on push

=env_lexframe::frame_push_rust_direct= currently grows the backing
vector when full.  Phase 47 =vector-push= grammar requires either:
- An =nl_vec_push_sexp= Rust helper (= ~15 LOC).
- Or capacity-aware allocation in elisp (= ~50 LOC of arithmetic +
  =call nl_alloc_vec=).

§111.E will pick when it gets there; Doc 111 §111.A-D doesn't
require a decision yet.

** §5.4 env_mirror / env_lexframe rewrite scope creep

The 12-helper list in §111.E §3.E is the *minimum*; if =install_stage0=
or =register_extern_builtin= also need to compile-and-link an elisp
helper, the list grows.

*Mitigation*: audit at §111.E ship time (= NOT now); fold any
discovered additions into the same checklist + grow the LOC delta
accordingly.

** §5.5 String layout drift recurrence

Same as Doc 101 §5.2.  Doc 111 inherits the =rust-toolchain.toml=
pin + the =sexp_abi_assert.rs= =const_assert!= for =size_of::<String>=.
