** §2.2 lexer (~lisp/nelisp-stdlib-reader.el~ §1)

state machine は char-index loop で実装:

#+BEGIN_SRC emacs-lisp
(defun nelisp--read-tokenize (string)
  "Tokenize STRING into a list of token plists.
Each token is (:type TYPE :value VALUE :pos POS :line LINE :col COL).
TYPE is one of: lparen rparen lbracket rbracket
                quote backquote comma comma-at function-quote
                dot int float str symbol sharps-paren."
  (let ((len (length string))
        (i 0) (line 1) (col 1)
        (tokens nil))
    (while (< i len)
      (let ((c (aref string i)))
        (cond
         ;; whitespace
         ((memq c '(?\s ?\t ?\n ?\r ?\f))
          (when (eq c ?\n) (setq line (1+ line) col 0))
          (setq i (1+ i) col (1+ col)))
         ;; line comment
         ((eq c ?\;)
          (while (and (< i len) (not (eq (aref string i) ?\n)))
            (setq i (1+ i)))
          ;; do not consume the newline; outer loop handles it
          )
         ;; opener / closer
         ((eq c ?\() (push (nelisp--read-token 'lparen "(" line col) tokens)
                     (setq i (1+ i) col (1+ col)))
         ;; ... rest of dispatch ...
         (t (nelisp--read-error "unexpected character" line col)))))
    (nreverse tokens)))
#+END_SRC

15+ token type、各 type に専用の sub-tokenizer:

| token type     | tokenizer                                            |
|----------------+------------------------------------------------------|
| =int=          | =nelisp--read-tok-int= (= 10進 / #x / #o / #b)       |
| =float=        | =nelisp--read-tok-float= (= 1.5 / 1e3 等)            |
| =str=          | =nelisp--read-tok-string= (= "..." with \\ / \n 等)  |
| =symbol=       | =nelisp--read-tok-symbol= (= alnum + - _ . :)        |
| =quote-family= | =nelisp--read-tok-prefix= (= ' ` , ,@ #')             |
| =sharps-paren= | =nelisp--read-tok-sharps= (= #s()                  |

数値 vs symbol disambiguation は lookahead 1 char で判定 (= "1.5" は
float、"1.5.x" は symbol、"1abc" は symbol — Rust lexer の
=is_symbol_char= 関数を elisp で再実装)。

** §2.3 parser (~lisp/nelisp-stdlib-reader.el~ §2)

recursive descent、token list を消費:

#+BEGIN_SRC emacs-lisp
(defun nelisp--read-parse-one (tokens)
  "Parse exactly one form from TOKENS.
Returns (cons FORM REMAINING-TOKENS) or signals on error."
  (when (null tokens) (nelisp--read-error "unexpected EOF"))
  (let ((tok (car tokens)) (rest (cdr tokens)))
    (pcase (plist-get tok :type)
      ('int    (cons (plist-get tok :value) rest))
      ('float  (cons (plist-get tok :value) rest))
      ('str    (cons (plist-get tok :value) rest))
      ('symbol
       ;; nil/t are recognized here (= 1 source of truth)
       (let ((name (plist-get tok :value)))
         (cond
          ((string= name "nil") (cons nil rest))
          ((string= name "t")   (cons t rest))
          (t (cons (intern name) rest)))))
      ('quote          (nelisp--read-parse-prefix 'quote rest))
      ('backquote      (nelisp--read-parse-prefix 'backquote rest))
      ('comma          (nelisp--read-parse-prefix '\, rest))
      ('comma-at       (nelisp--read-parse-prefix '\,@ rest))
      ('function-quote (nelisp--read-parse-prefix 'function rest))
      ('lparen         (nelisp--read-parse-list rest))
      ('lbracket       (nelisp--read-parse-vector rest))
      ('sharps-paren   (nelisp--read-parse-record rest))
      ('dot            (nelisp--read-error "unexpected ." tok))
      ('rparen         (nelisp--read-error "unexpected )" tok))
      ('rbracket       (nelisp--read-error "unexpected ]" tok)))))
#+END_SRC

主 sub-parser:

| sub-parser                      | 役割                                                |
|---------------------------------+----------------------------------------------------|
| =nelisp--read-parse-list=       | =(...)= = proper list / dotted pair                 |
| =nelisp--read-parse-vector=     | =[...]= = vector literal                            |
| =nelisp--read-parse-record=     | =#s(...)= = record literal (Doc 52 stage 4b 参照)  |
| =nelisp--read-parse-prefix=     | quote-family の =(quote X)= / =(function X)= 等 desugar |

dotted pair handling: parse-list は token を消費しながら element を
集積、=dot= token 検出時に「次の 1 form を cdr として読み、=)= を要求」
の特殊 path に分岐。

** §2.4 entry surface

3 個の user-visible function:

| function                              | 動作                                          |
|---------------------------------------+-----------------------------------------------|
| =(read-from-string STRING &optional START END)= | 1 form と consumed-end を cons で返す |
| =(read &optional STREAM)=             | STREAM (= string OR symbol OR fn) から 1 form 読む |
| =(read-all-from-string STRING)=       | 全 top-level form を list で返す (= load 用)  |

~read~ の STREAM = symbol/function は host Emacs 互換 (= 1 char 取得
function)。bootstrap subset では string と nil (= ~standard-input~) を
support、symbol/function は別 phase。

** §2.5 bootstrap entry primitive

Rust 側に残す ~read-from-string~ shim:

#+BEGIN_SRC rust
// build-tool/src/eval/builtins.rs
"read-from-string" => bi_read_from_string(args, env),

fn bi_read_from_string(args: &[Sexp], env: &mut Env) -> Result<Sexp, EvalError> {
    require_arity("read-from-string", args, 1, Some(3))?;
    // Stage 7.2 takeover: if elisp-side `nelisp--read-from-string-impl'
    // is defined, dispatch to it.  Otherwise fall back to Rust reader
    // (= bootstrap before stdlib-reader.el is loaded).
    if env.has_function("nelisp--read-from-string-impl") {
        return env.call_function("nelisp--read-from-string-impl", args.to_vec());
    }
    // Bootstrap fallback: Rust reader directly.
    let s = args[0].as_string_owned().ok_or_else(|| ...)?;
    let form = read_str(&s).map_err(|e| EvalError::Reader(e))?;
    Ok(Sexp::cons(form, Sexp::Int(s.len() as i64)))
}
#+END_SRC

このパターンは Doc 49 の elisp pivot で確立済 (= Rust 側 dispatcher が
elisp 関数の存在を確認、有れば call、無ければ Rust 実装に fallback)。

*** 2026-05-08 改判断 (= Doc 71 / Stage 7.6 設計時に明確化)

* §6 共通原則

- *parallel path 期間 minimal*: Stage 7.2.b → 7.2.d を素早く回す
  (= Rust + elisp 両 reader が動く期間が長引くと test infra が冗長化)
- *round-trip を gate 化*: Stage 7.2.c で自動化、以降の primitive
  追加 / Sexp variant 追加で必ず round-trip pass を確認
- *bootstrap で host Emacs reader を借用しない*: Doc 47 では host
  Emacs ~read~ を呼んでいたが本 Doc では完全に自前 lexer + parser
  に切替、host 依存を撤去
- *primitive 追加を最小限*: §2.6 通り既存 12 primitive で reader が
  完結、新規追加は不要 (= Rust min 原則)
