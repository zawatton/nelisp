** §1.2 monotonic shrink rule + precursor exception articulation

per-commit rule: *全 commit で Rust net LOC が non-positive でなければ
ship しない*.  例外 articulation:

- *atomic primitive + consumer ship*: primitive 追加と consumer 配線を
  同 commit に折り畳むことで dead-code (= zero-callsite) 状態を回避.
  Doc 79 v4 Stage C.4-atomic / Doc 79 v6 Stage C.3 sub-stage 5.4.a /
  Doc 79 v7 Stage 5.3.a で実例.
- *bundle 評価*: per-commit 緩和ではなく *prereq stage ship を先取り
  する bundle 評価*.  Doc 75 §3.7 Phase 9 + Stage 8.4 prereq bundle で
  -1,524 net、Doc 81 §4.3 Stage 81.x + Phase 7.1.6.a bundle で
  cumulative net -415 Rust 等.

precursor 例外 = *primitive +X / consumer atomic carry* のみ許容.
shim 中間 +800 LOC のような hybrid path (= Doc 78 D-A) は user directive
#4 (= 100% 純 elisp commitment) と方向矛盾で reject.
