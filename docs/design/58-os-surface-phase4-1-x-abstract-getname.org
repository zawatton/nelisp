* §0 TL;DR

Doc 56 Phase 4.1 で AF_UNIX (filesystem path) + AF_INET6 (sockaddr_in6) を ship。
本 Doc は user-deferred だった (4.1.1) AF_UNIX abstract namespace と (4.1.2)
getsockname / getpeername を *socket family を完全に使い切る* ところまで
まとめて積む。

scope = 8 専用 primitive + 8 elisp wrapper + 4 ERT。

| 区分              | 名前                                              | 提供層        |
|-------------------+---------------------------------------------------+---------------|
| AF_UNIX abstract  | bind-unix-abstract / connect-unix-abstract       | 専用 primitive |
| getsockname (×3)  | -inet / -inet6 / -unix                            | 専用 primitive |
| getpeername (×3)  | -inet / -inet6 / -unix                            | 専用 primitive |

設計指針 (= Doc 53〜57 継承): *Rust 追加最小化*。

- AF_UNIX abstract は Linux 限定 (sun_path[0]=NUL prefix) で、elisp string に
  NUL を埋め込むのは扱いが面倒なので、=NAME= 引数を 「NUL を含まない string」
  として受け、Rust 側で NUL prefix と socklen 計算を行う形にする。
- getsockname / getpeername は family が socket 作成時に確定しているので、
  elisp 呼び側がどの family の sockaddr を期待するかは既知。専用 primitive
  3 + 3 で scope 限定 (= AF_INET / AF_INET6 / AF_UNIX) として ship、unknown
  family は scope 外。
- SCM_RIGHTS (fd-passing via sendmsg / recvmsg) は struct msghdr / cmsghdr の
  full surface が要るので別格の設計 pass、Doc 60 候補 (本 Doc では out)。

* §1 動機

- *AF_UNIX abstract*: Linux で systemd / dbus / Wayland 系 IPC は abstract
  socket を当たり前に使う。filesystem path だと cleanup race / 残骸
  socket file の問題、abstract は kernel が auto-clean。NeLisp の elisp
  IPC を modern Linux native pattern に揃える。
- *getsockname*: =bind(0)= で kernel が選んだ port を後から取り出す
  ergonomics。Phase 4 で `port=47291` 直書きしていた boilerplate を
  `port=0 + getsockname` に置き換えられる。
- *getpeername*: AF_UNIX の credentials 確認 (= peer pid via SO_PEERCRED は
  Phase 4.x、まずは "誰が繋がってきたか" を AF_INET / AF_INET6 で取れる line
  だけ作る)。

* §2 設計

** §2.1 Rust 追加

builtins.rs:

- =install_builtins()= names + =dispatch()= に *8 名追加*:
  - =nelisp--syscall-bind-unix-abstract= / =-connect-unix-abstract=
  - =nelisp--syscall-getsockname-inet= / =-getsockname-inet6= / =-getsockname-unix=
  - =nelisp--syscall-getpeername-inet= / =-getpeername-inet6= / =-getpeername-unix=
- 関数 8 本実装。abstract 系は既存の =build_sockaddr_un= / =parse_sockaddr_un_peer=
  helper を re-use しつつ NUL prefix encoding を別 helper にくくり出す。
- getsockname / getpeername は libc::getsockname / getpeername を呼び、
  Doc 55 / 56 で既に書いた =parse_in6_addr_groups= / =parse_sockaddr_un_peer= を
  reuse して family 別の Sexp に変換。

*** §2.1.1 abstract socket helper

#+BEGIN_SRC rust
fn build_sockaddr_un_abstract(name: &str)
    -> Result<(libc::sockaddr_un, libc::socklen_t), EvalError>
{
    // sun_path[0] = NUL、sun_path[1..1+name.len()] = name bytes
    // socklen = sun_path offset + 1 + name.len()
    // NAME に NUL byte が含まれる場合は EvalError::Internal.
}
#+END_SRC

*** §2.1.2 getsockname / getpeername のシェル

unix:

#+BEGIN_SRC rust
fn bi_syscall_getsockname_unix(args: &[Sexp]) -> Result<Sexp, EvalError> {
    // FD → string path (anonymous = "", abstract = "\0name" になるが
    // NUL を含む string は elisp 側では扱いにくいので abstract 検出時は
    // Sexp::cons(Sexp::Symbol("abstract"), Sexp::Str(name)) を返す
