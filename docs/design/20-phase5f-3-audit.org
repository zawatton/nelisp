* §3 §2.5 quick win = =org-read-outline-tree=

** 3.1 spec

入力: =path= (string、.org file 想定)
出力: =:path :total-headlines :tree=
=:tree= = nested list of plists、各 entry =(:level :title :line :children [...])=

例:
#+begin_src text
input: /path/to/file.org
output:
  :tree
    [(:level 1 :title "TL;DR" :line 5 :children [])
     (:level 1 :title "現状"   :line 10
      :children
        [(:level 2 :title "main HEAD" :line 11 :children [])
         (:level 2 :title "tests"     :line 16 :children [])])
     ...]
#+end_src

** 3.2 既存 =file-outline= との差

| | file-outline | org-read-outline-tree |
|---+--------------+-----------------------|
| input | 任意 (.el / .org / .md) | .org のみ (validation) |
| output | flat list | hierarchical tree |
| 用途 | quick scan | section navigation |

file-outline は flat list (level + title + line) を返すが、entries 間の
親子関係は推測必要。本 tool は parser side で stack を回して
=:children= を埋め、call 側がツリー UI に直接食わせられる shape にする。

** 3.3 実装 sketch (pure regex、~50 LOC)

#+begin_src elisp
(defun nelisp-tools--org-parse-tree (lines)
  "Convert flat (level title line) entries into nested tree."
  (let ((stack (list (list :level 0 :children nil))))
    (dolist (entry lines)
      (let ((lvl (plist-get entry :level)))
        (while (>= (plist-get (car stack) :level) lvl)
          (let ((done (pop stack)))
            (setf (plist-get (car stack) :children)
                  (append (plist-get (car stack) :children)
                          (list (plist-put done :children
                                          (or (plist-get done :children) nil)))))))
        (push (append entry (list :children nil)) stack)))
    ;; flush remaining
    ...))
#+end_src

** 3.4 ERT カバー

- =org-read-outline-tree-flat-3-headlines= (level 1 だけの平坦)
- =org-read-outline-tree-nested-2-levels= (parent / child 関係)
- =org-read-outline-tree-rejects-non-org-extension= (.md → user-error)
