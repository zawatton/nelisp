| invariant                                                                                          | rationale                                                |
|----------------------------------------------------------------------------------------------------+----------------------------------------------------------|
| =make-frame= return value = unique frame-id (integer or symbol)                                    | Doc 34 §2.11 LOCKED invariant 維持                        |
| =frame-width= / =frame-height= = stub mode で 80x24 固定、real backend で actual size               | Doc 34 §2.11 LOCKED invariant 維持                        |

| =delete-frame= = stub では registry 更新のみ、real backend で =display-spec-frame-destroy= 呼び出し | Doc 34 §2.11 LOCKED invariant 維持                        |

*swap-in protocol* (Phase 11.A SHIPPED 時 procedure):

1. =nelisp-display-init= 呼び出し時に backend symbol 受領 (=tui= / =gtk= /

2. backend symbol が =stub= 以外なら =display-spec-frame-handle= 構築 path

3. 既存 stub mode で create された frame は *残置 + frame-id 再利用不可*
   (= backend 切替は process restart 推奨、=user signoff 案件=)
4. ERT regression: Doc 34 §2.11 の "stub backend で全 frame/window API
   ERT 緑" を Phase 11.A real backend でも *同 ERT が緑* で確認 (=
   double-check pattern)

** 2.3 glyph matrix (= Phase 11.B core data structure)

=glyph-matrix= = 各 window の出力 buffer に相当する 2D char grid + face
attribute mapping。Emacs の =struct glyph= / =struct glyph_row= /
=struct glyph_matrix= に対応:

#+begin_src elisp
;; 基本 struct (cl-defstruct で実装想定)
(cl-defstruct nelisp-display-glyph
  char           ;; codepoint (integer)
  face-id        ;; realized face index (integer)
  width          ;; glyph width in columns (1 for ASCII, 2 for CJK)
  composition-p  ;; nil or composition reference
  display-spec)  ;; nil or display property override

(cl-defstruct nelisp-display-glyph-row
  glyphs         ;; vector of nelisp-display-glyph
  used           ;; integer = active glyph count (≤ vector length)
  hash           ;; row hash for diff propagation
  start-pos      ;; buffer position at row start
  end-pos)       ;; buffer position at row end

(cl-defstruct nelisp-display-glyph-matrix
  rows           ;; vector of nelisp-display-glyph-row
  width          ;; column count
  height         ;; row count
  window         ;; owning window handle
  dirty-set)     ;; bitset of dirty row indices
#+end_src

*invariant*:
- =glyph-matrix= は *window-private*、frame レベルで集約は redisplay-driver
  が担当
- =hash= field で *差分 redraw* (= 前回 frame と同 hash の row は backend
  draw call skip)
- =dirty-set= は =text-property= 変更 / buffer modify / window resize で
  set される

** 2.4 face / display attribute system

Phase 11.B =face-realize= の design:

#+begin_src elisp
;; defface (= source spec、user editable)
(defface my-highlight
  '((((class color) (background dark)) :foreground "yellow")
    (((class color) (background light)) :foreground "blue")
    (t :inverse-video t))
  "Highlight face for my-mode.")

;; realized face (= backend specific cache、face-id で参照)
(cl-defstruct nelisp-display-realized-face
  face-id              ;; backend internal index
  foreground-color     ;; backend specific color object
  background-color     ;; backend specific color object
  font                 ;; backend specific font object
  attribute-mask)      ;; bitmask of active attributes
#+end_src

*realize protocol*:
1. backend init 時 =default-face= を realize (= face-id 0 reserved)
2. =defface= は spec のみ register、realize は *first use* で lazy
3. =face-attribute= を呼ぶと cache miss → realize → cache hit に遷移
4. backend swap (= =gtk= → =tui=) で *全 realized face cache invalidate*
5. =frame-parameter= の background-mode 変更で *該当 frame の realized face
   再 evaluate*

** 2.5a degrade contract (= capability 未対応時 fallback rule、T117 C.1 反映)

§2.5 capability matrix は "各 backend がどの capability を declare するか" の
statement、本節 §2.5a で *application 側 portable な degrade contract* を
LOCK する:

#+begin_src elisp
;; 全 backend は未対応 capability への API 呼び出しに対して
;; display-spec-unsupported condition を signal する
(define-error 'display-spec-unsupported
  "Display capability not supported by current backend")

;; condition data plist:
;;   :capability  symbol  — 未対応 capability symbol
;;   :api         symbol  — 呼び出された API name
;;   :backend     symbol  — current backend symbol

;; application 側 portable fallback pattern:
(condition-case err
    (display-image (find-image "icon.png"))
  (display-spec-unsupported
   ;; backend が image-* capability 未対応、text fallback
   (insert "[icon]")))

;; 簡易 check (= API 呼び出し前 guard):
(if (display-spec-capability-p 'image-kitty-graphics)
    (display-image (find-image "icon.png"))
  (insert "[icon]"))
#+end_src

*degrade contract invariant (LOCKED for Phase 11.A)*:
1. 未対応 capability への API 呼び出しは *=display-spec-unsupported=*
   condition を signal (= silent no-op / nil return / crash は禁止)
2. condition data plist = =:capability= symbol + =:api= symbol + =:backend=
   symbol の 3 件
3. application は =condition-case= で capture し portable fallback path へ
4. 簡易 check は =display-spec-capability-p= で API 呼び出し前 guard、ホット
   path 性能優先で = pre-check が default、condition-case は exception
5. =display-spec-unsupported= は backend 切替後の stale capability cache 検出
   にも用いる (= cached capability list と actual backend declared list の
   mismatch も同 condition で error 化)
