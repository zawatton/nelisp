* §0 TL;DR

Doc 53 Phase 1 (Minimal-5) + Doc 54 Phase 3 (Core-12) の上に Posix-30 = subprocess
+ network + multiplexing 第一弾を一括 ship。これで NeLisp の OS surface は
~12 + 11 = 23 wrapper~ となり、=fork → execve → wait4 / TCP server / TCP client /
poll-based event loop= が elisp で書ける line に到達する。

scope = 11 wrapper / 6 syscall_nr 追加 / 6 専用 primitive。

| 区分              | 名前                                   | 提供層        |
|-------------------+----------------------------------------+---------------|
| subprocess        | fork                                   | generic       |
| subprocess        | kill                                   | generic (済)  |
| subprocess        | getpid                                 | generic (済)  |
| subprocess        | execve                                 | 専用 primitive |
| subprocess        | wait4                                  | 専用 primitive |
| network (AF_INET) | socket                                 | generic       |
| network (AF_INET) | listen                                 | generic       |
| network (AF_INET) | bind                                   | 専用 primitive |
| network (AF_INET) | connect                                | 専用 primitive |
| network (AF_INET) | accept                                 | 専用 primitive |
| multiplexing      | poll                                   | 専用 primitive |
| setsockopt(int)   | setsockopt-int (= SO_REUSEADDR 等)     | 専用 primitive |

設計指針 (= Doc 54 §0 を継承): *Rust 追加を最小化する*。

- =fork=, =socket=, =listen= は int 引数のみで完結 → =syscall_nr()= の symbol map
  拡張だけで対応 (1 行 each)。=kill= / =getpid= は Doc 53 で既登録済。
- =execve= / =wait4= / =bind=/=connect=/=accept= / =poll= は char-array buffer
  ないし struct out-buffer / in-buffer が必要 → 専用 primitive。
- =setsockopt= は int-valued option 限定で 1 本 (= SO_REUSEADDR 等) — TIME_WAIT
  耐性 + ergonomics で必要、struct linger / struct timeval は Phase 4.x。

Path A (Linux) のみ。Path B (Darwin/Windows) fallback は Phase 1.1 と同じく
deferred (個別 nl-ffi-call 拡張は積み残し)。

* §1 動機

Doc 53 で =nelisp--syscall= 基盤、Doc 54 で fileio + JIT loader 用の Core-12 が
着地した。本 Phase は Lisp Machine vision (= memory =project_lisp_machine_vision_95pct=)
の "subprocess + network も elisp 化" に対応する。

実用 use case:

- *subprocess*: fork + execve + wait4 + pipe (Doc 54) + dup2 (Doc 54) で elisp
  から子プロセスを完全制御。=make-process=(将来) の elisp 実装ベース。
- *TCP server / client*: socket + bind + listen + accept + connect で
  =make-network-process=(将来) を elisp 駆動。
- *event loop*: poll で複数 fd を多重化。Phase 9d の process substrate に組み込む。

将来 (Phase 4.1 / 4.2): AF_UNIX / IPv6 / sigaction / pidfd / select の追加候補。

* §2 設計

** §2.1 Rust 追加 (最小化)

builtins.rs:

- =syscall_nr()= の symbol map に *6 件追加*: =fork= / =socket= / =listen= / =wait4=
  / =setpgid= / =getppid= (=wait4= は専用 primitive 経由で叩くが将来の汎用呼出も
  許す)。実コストは 6 行。
- =install_builtins()= names + =dispatch()= に *6 名追加*:
  =nelisp--syscall-execve= / =-wait4= / =-bind-inet= / =-connect-inet=
  / =-accept-inet= / =-poll=
- 関数 6 本実装 (= 全て =#[cfg(target_os = "linux")]= gate、非 Linux は EvalError stub)

*** §2.1.1 =bi_syscall_execve PATH ARGV ENVP=

#+BEGIN_SRC rust
fn bi_syscall_execve(args: &[Sexp]) -> Result<Sexp, EvalError> {
    // args[0] = path (string)
    // args[1] = argv (list of strings)
    // args[2] = envp (list of strings)
    // CString[]::Vec → *const c_char[]::Vec (NULL-terminated)
    // libc::execve は成功時 return しない、失敗時 -1 → -errno を返す。
}
#+END_SRC

*** §2.1.2 =bi_syscall_wait4 PID OPTIONS=

#+BEGIN_SRC rust
fn bi_syscall_wait4(args: &[Sexp]) -> Result<Sexp, EvalError> {
    // args[0] = pid (int)        ; -1 = any child
    // args[1] = options (int)    ; 0 / WNOHANG / WUNTRACED
    // status: int out-buffer; rusage: NULL (Phase 4 では不要)
    // 成功: Sexp::cons(Sexp::Int(child_pid), Sexp::Int(status))
    // 失敗: Sexp::Int(-errno)
}
#+END_SRC

*** §2.1.3 =bi_syscall_bind_inet FD HOST-INT PORT=

AF_INET 専用。HOST-INT = 32-bit IPv4 host byte order
(e.g., 127.0.0.1 = =0x7F000001=)、PORT = 16-bit host byte order。
Rust 側で =htonl= / =htons= → struct =sockaddr_in= 構築 → libc::bind 呼出。

#+BEGIN_SRC rust
fn bi_syscall_bind_inet(args: &[Sexp]) -> Result<Sexp, EvalError> {
    // 成功 0 / 失敗 -errno
}
#+END_SRC

*** §2.1.4 =bi_syscall_connect_inet FD HOST-INT PORT=

bind とほぼ同形 (libc::connect)。同上。

* §3 Sub-phase 分割

** Phase 4 (本 Doc, scope) — Posix-30 第一弾

11 wrapper + 6 syscall_nr 追加 + 6 専用 primitive + 6-8 ERT。

** Phase 4.1 (将来) — AF_UNIX + IPv6 拡張

=bi_syscall_bind_unix= / =-connect-unix= / =-accept-unix= 追加。
AF_INET6 sockaddr_in6 対応。

** Phase 4.2 (将来) — シグナル系

=sigaction= / =sigprocmask= / =sigpending= の追加。signal handler は
elisp callback を Rust 側で受けるため =Env= に signal queue 追加が必要 →
別格の設計 pass (= memory ~feedback_nelisp_minimize_rust~ に従い、
queue は elisp dynamic var に置けるかも Phase 4.2 で検討)。

** Phase 4.3 (将来) — select / pidfd / inotify

非同期 I/O 周り。poll より新しい multiplexer 群。
