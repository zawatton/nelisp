* 使い方

Phase 0 Q3 (audit 深さ: 精読 + 論文読み込み) に基づき、NeLisp 設計に
load-bearing な論文の要約を本ディレクトリに蓄積する。

** ファイル命名規約

=<author>-<year>-<slug>.org=

例: =rhodes-2008-sbcl.org=, =bolz-2009-meta-tracing.org=

** stub フォーマット

各 paper stub は以下の structure:

- citation (author, year, venue, DOI/URL)
- access status (unread / reading / summarized)
- core contribution (3-5 bullet)
- NeLisp relevance (Phase 番号 + 設計質問)
- reading notes (精読時に埋める)
- open questions (自問)
- cite-back (NeLisp docs の中でこの paper を引用した箇所リスト)

** 読み方方針

- 1 本あたり 4-8h、週末研究 *2-3 本/月*
- 各 paper stub の "reading notes" に実読時に追記
- NeLisp docs (=../00-05.org=) から引用した際は cite-back セクション更新
