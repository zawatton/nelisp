** 3.1 MVP subset 選定基準 (historic, Phase 7.4.3)

- *JIS X 0208 punctuation* (row 1, 0x8140-0x819E): 句読点・括弧・記号
- *Fullwidth digits / latin* (row 3, 0x824F-0x829A)
- *Hiragana* (row 4 全, 0x829F-0x82F1)
- *Katakana* (row 5 全, 0x8340-0x8396)
- *代表的 kanji ~20 字* (亜・漢・字 etc, partial table validation 用)
- *CP932 NEC special* (0x8740-0x879F): ①-⑳ + ㍉ etc
- *CP932 IBM extension* (0xFA40-0xFA70 一部): ⅰ-ⅹ etc
- *EUC-JP X 0208 punctuation + hiragana + katakana + 露西亜文字*
  (= 同等カバレッジ確保)
- *X 0212 supplementary kanji 64 字* (row 0x10 / 0x24 representative)

** 3.2 full table classification (Phase 7.4.5)

Phase 7.4.5 generator は CP932.TXT entries を以下の range で classify:

| domain          | byte ranges                                                | count |
|-----------------+------------------------------------------------------------+-------|
| JIS X 0208 base | 0x8140-0x84BE + 0x889F-0x9FFC + 0xE040-0xEAA4              |  6879 |
| CP932 ext (NEC) | 0x8740-0x879C                                              |    83 |
| CP932 ext (IBM) | 0xED40-0xEEFC + 0xFA40-0xFC4B                              |   762 |
| ASCII (skipped) | 0x00-0x7F                                                  |   128 |
| JIS X 0201 kana | 0xA1-0xDF                                                  |    63 |

ASCII + JIS X 0201 katakana は =nelisp-coding.el= が algorithm で直接
処理するため table に格納しない。

EUC-JP X 0208 entries は SJIS X 0208 から =SJIS → JIS row/cell= 変換で
algorithmic に派生 (CP932 vendor extension は EUC-JP 等価コードを持た
ないため除外)。JIS X 0212 は =JIS0212.TXT= から直接 parse、=EUC-INT=
形 ((row|0x80) << 8 | (cell|0x80)) に正規化。

* §4 golden SHA-256 hash

** 4.1 計算方法

#+BEGIN_SRC elisp
(secure-hash 'sha256
             (concat
              (prin1-to-string nelisp-coding-shift-jis-x0208-decode-table)
              (prin1-to-string nelisp-coding-cp932-extension-decode-table)
              (prin1-to-string nelisp-coding-euc-jp-x0208-decode-table)
              (prin1-to-string nelisp-coding-euc-jp-x0212-decode-table)))
#+END_SRC

実装は =nelisp-coding-jis-tables--canonical-bytes= +
=nelisp-coding-jis-tables-verify-hash= を参照。

** 4.3 mismatch 時の対応

CI で =make test= が =nelisp-coding-jis-tables-verify-hash-passes= で
落ちた場合の root cause:

1. generator regenerated tables but forgot to update the golden hash
   constant
   → fix: copy computed hash from `(secure-hash …)' into the defconst
2. accidental table edit (rebase / merge silent rewrite)
   → fix: revert table content, hash auto-validates
3. Emacs `prin1-to-string' output format change between versions
   → unlikely but possible; investigate `print-length' / `print-level' /
     `print-circle' / `print-quoted' bindings; pin them in
     `nelisp-coding-jis-tables--canonical-bytes' if needed
