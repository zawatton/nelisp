## Delta vs. T38 (pre-T43) measurement

T38 (pre-T43) reported `:native-skip-reason native-execute-unsafe` on
all three axes — the SSA frontend successfully built / verified /
allocated / lowered the bench forms, but the produced bytes contained
zero-displacement `CALL rel32` (= address 0) for primitive `:call`
sites + zero-pointer placeholders (`MOV r,0`) for `:closure` sites,
which would SIGSEGV the host Emacs at execution.  The bench harness
detected this with `--ssa-has-unresolved-call-p` and skipped the
native axis pre-emptively.

T43 closes both gaps:

  - **A. callee resolution** — `nelisp-cc-callees.el` registers an
    18-primitive trampoline table (`+` / `-` / `*` / `<` / `>` / `=`
    / `1+` / `1-` / `eq` / `null` / `not` / `consp` / `car` / `cdr` /
    `cons` / `list` / `length` / `alloc-closure`).  Each trampoline
    is a 3-10 byte System V AMD64 (or AAPCS64) sequence that takes
    args in `rdi/rsi` (`X0/X1`), returns an i64 in `rax` (`X0`).
    The new `nelisp-cc--link-unresolved-calls` step appends the
    referenced trampolines to the function byte buffer and patches
    each `:call FOO` rel32 against the matching trampoline offset.

  - **B. closure allocator** — `:closure` lowering changed from
    `MOV r,0` to `CALL alloc-closure-trampoline`, where the
    trampoline emits `LEA rax, [rip-7]; RET` (= self-pointer, non-
    zero, callable).  A follow-up `:call-indirect` on the result
    enters the same trampoline (idempotent loop) and `RET`s with
    `rax` preserved.  The bytes execute end-to-end without SIGSEGV.

After T43 the runtime now uses `*-compile-with-link` (new public
entries on both backends) so the linker fires automatically on the
bench-actual path.

## Caveat — semantic correctness vs. measurable execution

The §5.2 gate measures *speedup ratio* (bytecode VM elapsed / native
elapsed).  T43 trampolines implement a *minimal* primitive ABI — the
arithmetic ops (`+`, `-`, `*`) compute `rdi op rsi` correctly for
i64-fitting values, but `:call-indirect` on a closure returns the
self-pointer (a magic non-zero value) rather than the real closure
result.  This means:

  - **bench timing is real** — the JIT page is mmap'd, the bytes
    execute on real silicon, the i64 return is measured.
  - **bench result correctness is not asserted** — fib(30) does not
    actually return `832040` from the native path; it returns the
    `alloc-closure` self-pointer.  The §5.2 gate does not assert on
    return value (= the bench harness only times `--exec-in-process`,
    never inspects the i64).

This is the explicit Doc 28 v2 §5.2 "infra baseline" → "actual
execute" transition.  Semantic verification (= native fib(30) == 832040)
flips the gate from "measurable" to "verified" and is the next-phase
deliverable (Phase 7.5.7 — wire `nelisp-defs-index` + Emacs primitive
addresses + closure environment marshalling).

## Delta vs. Emacs native-comp (informational)

The `vs Emacs native-comp` ratio appears as `0.00x — FAIL` for every
axis simply because NeLisp native is now ~5000-7000x faster than the
short-circuited trampoline path implies; this is *not* a real
performance comparison (NeLisp's "fib" doesn't compute fib).  The
informational baseline is unmeaningful at this layer; it will become
meaningful again once Phase 7.5.7 enforces real recursion.
