## Key result — pipeline ON 経路 SIGSEGV 解消

T161 で発覚した backend 2 gap = T162 の deliverable:

### Gap 1 fix — synthetic callee resolution

lambda-lift で `:closure` instr が dropped、inner SSA 関数が orphan 化。
backend `--collect-inner-functions` が `:closure` 経由のみ walk するため
lifted body は emit されず → `callee:fib` fixup が address 0 → SIGSEGV。

**修正**:
1. `cl-defstruct nelisp-cc--ssa-function` に `lifted-inners` slot 追加
2. `lift-pass` で drop closure 前に `(lifted-name . inner-fn)` を caller の slot に push
3. `--lift-resolve-name` で closure の `:letrec-name` meta を preferred (= T161 が rewrite した `:call :fn fib` の name と一致させる)
4. `--compile-and-link` で `closure-inners + lifted-inner-entries` を merge
5. Step 4 で `inner-name` が `(memq callees)` の時 `callee:<NAME>' alias label を bind
6. `--embed-trampolines` の fixup 変換 guard を緩めて synthetic callee も含める

### Gap 2 fix — rec-inline snapshot original body (partial)

`--clone-blocks` の `(blocks callee)` snapshot が clone 時の状態 = self-rec で grown。
複数 splice で previously-cloned blocks を re-clone → exponential growth (2 → 22 → 766)。

**修正**: `--clone-blocks` に `&optional original-snapshot` 追加、`--rec-inline-pass` driver で
splice 前の caller blocks を 1 度 snapshot して全 splice に thread。

⚠️ **ただし**: rec-inline の depth>0 動作で `:jump-with-no-successor` error が発生する未解決の
CFG link issue あり。本 T162 では `nelisp-cc-pipeline-rec-inline-depth-limit` default を
0 維持 (= rec-inline 無効) で ship、Phase 7.7.5+ で rec-inline CFG link path の
investigate + fix を別 task で。

## Why fib didn't reach 30x

depth=0 = rec-inline unroll なし → recursive call CALL/PUSH/POP overhead が消えない。
fib(30) は recursive call を ~2 million 回叩く (= O(2^30 / 1.6))、その call overhead が
dominate。depth>0 で unroll すれば inline 効果 + base case fold で speedup 可能性高い、
が rec-inline CFG link bug が未解決のため depth=0 安全構成で ship。

期待 (Phase 7.7.5+ rec-inline CFG fix 後):
- depth=2: fib 16-22x (= ~50% 上乗せ)
- depth=4: fib 25-30x (= ~150% 上乗せ、GO 達成可能性)
