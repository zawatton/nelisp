## NeLisp 7+A-E anvil-corpus probe

For each module, the harness reads every top-level form via
`nelisp-reader-read-all` then attempts to evaluate the
"interpret-pure" subset (= forms whose head symbol is registered
in `nelisp-special-forms--dispatch` and is not a side-effect
form like `defcustom`).

| module        | forms-total | forms-read | dispatch-resident | evaluable-attempted | evaluable-pure |
| ------------- | ----------- | ---------- | ----------------- | ------------------- | -------------- |
| anvil-server  | 0           | 0          | 0                 | 0                   | 0              |
| anvil-defs    | 68          | 68         | 0                 | 0                   | 0              |
| anvil-state   | 32          | 32         | 0                 | 0                   | 0              |
| **total**     | **100**     | **100**    | **0**             | **0**               | **0**          |

`anvil-server` `forms-total` = 0 because `nelisp-reader-read-all`
errors out on the first malformed string-escape it encounters
(see Phase 7+G section below); the other two files round-trip 100%
through the reader.

`dispatch-resident` = 0 across 100 forms is the M3 readiness gap:
**every** anvil top-level form is a `defun`, `defvar`, `defconst`,
`define-error`, `defcustom`, `defgroup`, `require`, `provide`,
`cl-defstruct`, `cl-defun`, `cl-defmacro`, or `defmacro` — none of
which appear in the Phase 7+B-E dispatch table.  The synthetic
probe above shows the *evaluator core* works; the missing piece
is the **registration-form glue layer** that wires top-level
`defun` & friends to populate `nelisp-special-forms--functions`
/ `--globals` / `--specials` (and emit a corresponding host-side
binding so subsequent `funcall (function FN)` resolves).

## findings (= 7+G future feature backlog)

### F1 -- Phase 7+F-followup-1: top-level registration forms

- *names*: `defun`, `defvar`, `defconst`, `defmacro`, `define-error`,
  `defcustom`, `defgroup`, `defface`, `cl-defstruct`, `cl-defun`,
  `cl-defmacro`, `require`, `provide`
- *what*: register handlers in `nelisp-special-forms--dispatch`
  that populate the appropriate global table (`--functions` for
  `defun` / `cl-defun`; `--globals` for `defvar` / `defconst`)
  and either delegate to host (`require` / `provide` /
  `define-error` / `defcustom`) or expand to lower-level
  primitives (`cl-defstruct`).
- *gain*: lifts anvil-corpus `dispatch-resident` from 0/100 to
  ~100/100 (= every top-level form becomes M3-eligible).
- *size*: ~150-300 LOC + 10-20 ERT.

### F2 -- Phase 7+G string-escape extension

- *names*: `\(`, `\)` and likely `\,` `\;` etc. inside `"..."`.
- *what*: extend `nelisp-reader--read-string-escape`
  (`src/nelisp-reader.el:220`) to accept escape characters that
  produce themselves verbatim in the output (Emacs reader behaviour).
- *gain*: lifts anvil-server.el `forms-read` from 0 to ~250+ and
  unblocks the M3 corpus probe on the full 3-module set.
- *size*: ~30-50 LOC + 4-6 ERT.

### F3 -- Phase 7+F-followup-2: function-call dispatch when head is host fboundp

After F1, top-level `defun` will register the function but the
`(call-host-fn ...)` body inside `defun`s will need
`nelisp-special-forms--eval-call` to fall back to `(apply
(symbol-function head) vals)` when `head` is host-`fboundp` and
not in `--functions`.  The current implementation in
`nelisp-special-forms.el:466-495` does call `--function-of` which
already has this fallback (lines 171-178), so F3 may turn out to
be a no-op once F1 lands; this entry is a placeholder that the
M3 follow-up tasking should re-evaluate after F1.

## next-step suggestion (top 3)

1. **Phase 7+F-followup-1 — top-level registration forms**.
   Wire `defun` / `defvar` / `defconst` / `defmacro` /
   `define-error` / `defcustom` / `defgroup` / `cl-defstruct` /
   `cl-defun` / `cl-defmacro` / `require` / `provide` handlers
   into `nelisp-special-forms--dispatch`.  Largest single jump
   in M3 readiness (= 0/100 -> ~100/100 anvil-corpus dispatch
   coverage).  Sized ~150-300 LOC + 10-20 ERT.  No new module
   required; pure dispatch table extension.

2. **Phase 7+G — string-escape extension in `nelisp-reader`**.
   Accept `\(` / `\)` (and verify `\;` `\,` `\.` etc.) in
   `--read-string-escape`.  Unblocks anvil-server.el reader
   probe.  Sized ~30-50 LOC + 4-6 ERT.

3. **anvil-server hardening — return-shape contract regressions**.
   T97-FOLLOWUP-1 was already fixed between T97 and T157, but
   the anvil-server `string-or-nil` return contract surface
   (`anvil-server.el:1066-1078`) is still narrow; widening it
   to accept plists / hash-tables (via JSON encode) would let
   future `tools/call` round-trips through the M3 path return
   structured data without per-handler `prin1-to-string` glue.
   Out-of-scope for M3 itself; in-scope for M4 / M5
   orchestration tasks.
