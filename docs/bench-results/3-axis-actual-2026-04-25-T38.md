## Delta vs. T37 (pre-T38) measurement

T37 (pre-T38) reported `:native-skip-reason native-compile-failed` on
all three axes â€” the SSA frontend raised
`nelisp-cc-unsupported-form :head while` /
`:unknown-opcode closure 0` *at compile time* on every bench form,
so the native pipeline never got past the SSA build stage.

T38 lifts the *compile-time* blocker:

  - `letrec` is now lowered (`--lower-letrec`, allocate-then-fill
    two-pass scheme).
  - `funcall` (closure target) is now lowered (`--lower-funcall`,
    routes via the existing `:call-indirect` opcode).
  - `while` is now lowered (`--lower-while`, header / body / exit
    blocks with explicit back-edge).
  - x86_64 backend handles `:call-indirect` (`CALL [rax]`, `FF /2`)
    and `:closure` (placeholder MOV r,0).
  - arm64 backend handles `:call-indirect` (`BLR X16`,
    `0xD63F0200`) and `:closure` (placeholder MOVZ Xd, #0).

After T38 the SSA build, linear-scan allocator, and backend codegen
all run cleanly on the bench-actual forms â€” confirmed by
`nelisp-cc-pipeline-fib-30-builds-and-allocates` ERT (test 33).

The remaining gate blocker is Phase 7.5 *callee resolution*:

  - `:call` instructions to `+` / `-` / `<` etc. carry
    `:unresolved t` and emit `CALL rel32` with displacement 0 â€” the
    Phase 7.5 patcher will rewrite these against
    `nelisp-defs-index` to point at the live primitive definitions.
  - `:closure` materialisation currently produces a zero pointer
    (placeholder MOV r,0); a runtime closure allocator (Phase 7.5)
    must replace this with a function-pointer + capture-array
    address.
  - `:call-indirect` consumes the closure value as the callee
    address, so an unresolved closure â†’ CALL [0] = SIGSEGV.

The bench harness now pre-scans the SSA function for unresolved
`:call` / `:call-indirect` / `:closure` instructions (helper
`--ssa-has-unresolved-call-p`) and signals
`nelisp-cc-bench-actual-unsafe-bytes` to skip the actual exec loop.
This converts an Emacs-process-killing SIGSEGV into a recoverable
skip with a documented `:native-skip-reason`.
