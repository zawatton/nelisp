* 2. Phase 1 — Elisp interpreter in Elisp

** 1.1 subset 定義

Phase 1 subset は =../03-architecture.org= §7 で LOCKED (2026-04-22):
- 14 系の special form + =cond/and/or/when/unless/defconst/prog1/prog2/catch/
  throw/macroexpand-all= 込み (計 22 form)
- 40 builtin (cons / arithmetic / string / symbol / print)
- macro expander with closure
- 除外 (Phase 2 以降): buffer / marker / overlay / text property / advice /
  process / regex / native-comp interop / thread / hash-table / float /
  bignum / dynamic module

本 doc は release / timeline / governance に集中。subset の source of truth
は =03-architecture.org= §7.1 / §7.3、判断変更時はそちらを更新する。

** 1.2 成功条件

- =(nelisp-eval '(fib 30))= が正しい値を返す
- =(nelisp-eval '(let ((x 10)) (defun f () x) (f)))= で lexical /
  dynamic 両 semantics が正しい
- macro = (defmacro unless (c &rest body) `(if ,c nil ,@body)) が動く
- =(nelisp-read "(+ 1 2)")= → =(+ 1 2)= で reader が働く
- ERT test suite 50+ 本、すべて pass

* 3. Phase 2 — self-host

** 2.1 bootstrap loop

#+begin_src text
  cycle 0: host Emacs が nelisp.el を load
  cycle 1: nelisp が nelisp.el を eval (nelisp が nelisp を動かす)
  cycle 2: cycle 1 の nelisp が nelisp.el を eval (出力一致 = fixpoint)
#+end_src

** 2.2 成功条件

- cycle 1 と cycle 2 の output が bit-identical
- host Emacs と nelisp で 50+ ERT test が両方 pass
- self-host した nelisp で Phase 1 test suite が pass

** 2.3 リスク

- Speed: self-host 直後は 100x-1000x 遅い前提で OK
- Memory: cycle 2 で OOM しないか確認

* 4. Phase 3 — bytecode VM + GC + fiber

- =nelisp-bytecode.el= (= Emacs bytecode に近い設計、ただし再設計許容)

* 5. Phase 4 — concurrency (actor primary)

=04-concurrency.org= §7 の locked 方針に従う:

- *Phase 4 primary*: C actor 実装 (=nelisp-actor.el=, =nelisp-chan.el=)
  - shared-nothing actor + mailbox + channel
  - =nelisp-spawn= / =nelisp-send= / =nelisp-receive= / =nelisp-chan-*=
  - buffer-as-actor の実験
- *Phase 4b (optional)*: STM を experimental layer として追加
  (=nelisp-stm.el=、transactional buffer の PoC)
- 成功条件: anvil 痛み解消 test case 3 件から 1+ が PoC 動作
  (=02-scoping.org= §0 完遂テスト と整合)
- 学術論文 (ELS short paper 等) は Phase 4 成功後に opt-in で検討

* 6. Phase 5 — 3 core module Elisp 化 + AI path 完遂 = *NeLisp v1.0*

=02-scoping.org= §0 の v1.0 完遂 3 条件を達成:

- =nelisp-buffer.el= (gap buffer in Elisp、AI text manipulation の核)
- =nelisp-event.el= (event loop dispatcher、AI interactive の基盤)

- =nelisp-thread.el= (真 OS thread、Phase 5 後半、C 層 GC/signal 改修同時)
- mini-Emacs として起動 (raw terminal、AI-critical path で完動)
- *Stage A 完遂* — anvil.el 系 Stage B/C を乗せる substrate 準備完了
